## Why

当前完成前验证阶段发现前端真实联调行为与角色权限和预约菜单预期不一致：普通用户初始化时会请求管理员看板接口，进入预约菜单时会立即请求会议室列表。该修订需要在验证产物中固化回归要求，并在生成前端中避免无权限或非用户触发的接口请求，降低 403 噪音和不必要的接口负载。

## What Changes

- 普通用户启动/进入非管理页面时不得请求 `/api/hx-meeting/v1/admin/dashboard`。
- 仅管理员角色可加载管理员看板数据，且加载前必须先确认当前用户角色。
- 进入预约菜单 `/booking` 时不默认请求 `/api/hx-meeting/v1/rooms`；只有用户点击“查询可用会议室”后才发起会议室查询。
- 更新完成前验证阶段的 E2E 联调用例、测试用例摘要和测试报告，覆盖上述回归点。
- 不变更后端 API 契约；不新增外部依赖。

## Capabilities

### New Capabilities
- `verification-network-guardrails`: 约束完成前验证中前端网络请求的角色门控和用户触发时机，覆盖普通用户不得触发管理员接口、预约页面不得自动查询会议室。
