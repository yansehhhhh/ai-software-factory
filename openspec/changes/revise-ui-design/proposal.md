## Why

当前 workflowRunId `6b74f1b8-1725-4755-877f-ea35e6436020` 的 UI 设计阶段收到修订操作，但用户尚未填写具体反馈。需要建立一个受控 OpenSpec 变更，用于后续在明确允许范围内修订会议室预约系统 UI 阶段产物，避免影响产品需求、架构、数据库或业务代码。

## What Changes

- 创建 `revise-ui-design` UI 设计阶段修订变更，覆盖“移动端应用：会议室预约系统”的现有 UI 原型交付包与 UI 设计规范。
- 后续 `/opsx:apply` 仅允许调整：`docs/移动端应用：会议室预约系统/UI原型/` 下的设计稿、组件库、交互说明、响应式适配产物，以及 `docs/移动端应用：会议室预约系统/产品设计/03-UI设计规范/UI-Design-Spec.md`。
- 在缺少具体反馈时，默认以一致性、移动端可用性、可读性、状态覆盖、交互说明完整性和文件引用准确性为修订目标。
- 不新增跨阶段产品需求，不修改前后端代码、接口、数据库、测试或非 UI 阶段文档。

## Capabilities

### New Capabilities
- `ui-design-revision`: 定义 UI 设计阶段产物的受控修订能力，包括 UI 原型、组件库、交互说明、移动端适配和 UI 设计规范的一致性修订约束。

### Modified Capabilities

无。

## Impact

- 影响范围：`docs/移动端应用：会议室预约系统/UI原型/` 与 `docs/移动端应用：会议室预约系统/产品设计/03-UI设计规范/UI-Design-Spec.md`。
- 不影响范围：`apps/`、`packages/`、`tests/`、产品需求文档、架构设计、数据库设计、运行时逻辑和其他阶段产物。
- 风险控制：本变更仅创建 OpenSpec 修订定义；实际阶段产物修订需在 `/opsx:apply` 中按任务逐项执行并校验变更路径。
