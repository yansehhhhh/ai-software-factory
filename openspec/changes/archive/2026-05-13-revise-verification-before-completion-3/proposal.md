## Why

完成前验证阶段收到用户缺陷反馈：会议室管理新增入口当前不可真实创建，底部导航切换页面偶发空白仅显示标题，通知进入详情页白屏报错。这些问题会阻断移动端会议室预约系统的核心验收路径，需要在当前阶段修订中闭环修复并补充验证证据。

## What Changes

- 修复会议室管理“新增”交互，从样式/演示模式改为真实可用的新增会议室表单与后端持久化调用。
- 修复底部导航切换导致页面空白的问题，保证首页、预约、我的预约/通知、违规、管理等主要页面在多次切换后稳定渲染正文内容。
- 修复通知列表进入详情页白屏报错的问题，提供真实可访问的通知详情页面或等价详情呈现，并处理缺失数据/异常状态。
- 补充完成前验证 E2E 与测试报告，覆盖上述缺陷回归路径，验证前后端真实联调而非 mock。

## Capabilities

### New Capabilities
- `hx-meeting-defect-regression`: 覆盖会议室管理新增、底部导航稳定切换、通知详情访问三类缺陷回归验收要求。

### Modified Capabilities

## Impact

- 允许修改范围内的生成前端：`generated/HX-Meeting/frontend`，重点涉及管理会议室页面、底部导航/布局、通知列表与详情路由。
- 允许修改范围内的生成后端：`generated/HX-Meeting/backend`，如缺少会议室新增或通知详情接口，需要补齐对应真实接口与数据持久化。
- 允许修改范围内的测试产物：`docs/移动端应用：会议室预约系统/测试/`，补充 E2E 用例、测试用例说明与验证报告。
- 不引入破坏性 API 变更；现有真实联调路径与 API 前缀 `/api/hx-meeting/v1` 保持兼容。
