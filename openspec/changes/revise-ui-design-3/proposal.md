## Why

当前 UI 设计中的预约页面未满足“会议室占用看板直接选会议室与时间段”的交互预期，用户需要在同一页面快速判断会议室占用状态并完成可预约时段选择。此次修订用于明确预约页面的看板式选择规则，降低误选、跨会议室选择和超时长预约风险。

## What Changes

- 将预约页面核心展示调整为会议室占用看板，以“会议室行 + 时间格列”的形式呈现可用、已占用、已选状态。
- 支持用户在看板中直接选择会议室与时间段，选择结果自动关联下方预约表单。
- 限制看板选择只能发生在同一会议室行内，且最多连续 4 格（2 小时）。
- 禁止跨行、跳格、跨占用块选择，已占用时间格不可选。
- 看板下方保留补充表单，仅填写会议主题、参会人等信息；参会人使用多选框控件。
- 移除预约页面中的文字版预约规则提示，改由交互状态和禁用态表达规则。
- 更新允许范围内的 UI 原型与 UI 设计规范，不涉及业务代码、接口或数据库变更。
