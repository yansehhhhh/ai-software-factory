## Context

HX-Meeting 当前 UI 设计规范和原型产物以“响应式 Web + PC 端看板 + 移动端补充”为主要表达，其中 `index.html`、`desktop.svg`、组件库和交互说明均包含明显 PC 优先内容。用户反馈要求取消 PC 设计图，改为移动端 H5，并同步修改 HTML 原型、组件库等 UI 阶段产物。

本次修订仅允许修改：

- `docs/# 会议室预定系统（HX-Meeting） ### 核心业务/UI原型/`
- `docs/# 会议室预定系统（HX-Meeting） ### 核心业务/产品设计/03-UI设计规范/UI-Design-Spec.md`

因此设计策略应保持现有目录结构不变，以“重定位、替换内容、弱化/移除 PC 交付语义”为主，而不是新增平台运行时代码或重建文档体系。

## Goals / Non-Goals

**Goals:**

- 将 UI 阶段基准从 PC/响应式 Web 调整为移动端 H5。
- 将主 HTML 原型改为手机画布/移动端交互结构，覆盖核心业务路径。
- 将组件库改为移动 H5 组件系统，包含移动导航、卡片、底部抽屉、时间段选择、移动表单、状态标签、提示反馈等。
- 修订 UI 设计规范，使页面清单、布局规范、组件规范、交互规范和原型映射均与移动端 H5 一致。
- 对原有 PC 产物采取降级或替换策略，避免后续读取产物时继续把 PC 设计图作为目标。

**Non-Goals:**

- 不修改前端 Vue 应用、后端服务、API、数据库或平台运行时代码。
- 不新增超出允许范围的文档目录或架构产物。
- 不重新定义会议室预约业务规则，仅调整这些规则在移动端 H5 上的 UI 表达。
- 不引入新的 UI 框架、构建工具或外部依赖。

## Decisions

### 1. 以移动端 H5 主原型替代 PC/响应式主原型

- 决策：`UI原型/设计稿/index.html` 应作为移动端 H5 主原型，采用手机视口、底部导航、单列卡片、触控优先按钮和底部抽屉交互。
- 理由：用户明确不要 PC 设计图，后续实现需要一个清晰的 H5 目标，而不是继续在 PC 与移动之间摇摆。
- 替代方案：保留 PC 主画布并新增移动页面。该方案会造成交付重点不清晰，且不符合“不要 PC 设计图”的反馈。

### 2. 保留目录结构，调整文件语义

- 决策：继续使用现有 `设计稿/`、`组件库/`、`交互原型/`、`移动端适配/` 目录，但将内容统一改为移动端 H5。
- 理由：允许修改范围限定在现有 UI 原型与 UI 规范文件内，保留目录可降低引用路径变化带来的风险。
- 替代方案：新增 `H5设计稿/` 或删除旧目录。该方案会改变交付结构并可能影响平台结果面板读取。

### 3. PC 设计图不作为核心交付，必要时改为废弃说明或移动替代图

- 决策：`desktop.svg` 不再承载 PC 端视觉方案；实施时可将其替换为“PC 设计图已取消，详见移动 H5 原型”的占位说明，或从 UI 规范映射中移除核心引用。
- 理由：直接保留旧 PC SVG 会与用户反馈冲突；删除文件可能造成既有 artifact 链接失效，占位说明更稳妥。
- 替代方案：物理删除 `desktop.svg`。该方案虽然语义更彻底，但可能导致已有阶段产物链接 404。

### 4. 组件库以移动端业务组件为核心

- 决策：组件库应按移动 H5 业务场景组织，包括移动顶部栏、底部 Tab、快捷入口卡片、会议卡片、时间段 Chip、状态标签、表单项、人员选择、底部抽屉、确认面板、Toast/结果页等。
- 理由：会议室预定在 H5 上的关键体验依赖触控选择、卡片列表和轻量反馈，PC 表格/看板组件不再是主组件。
- 替代方案：仅调整样式不调整组件清单。该方案无法指导后续移动端实现。

### 5. 移动预约采用“日期 + 会议室卡片 + 连续时间段选择”

- 决策：移动 H5 会议预约页不使用完整横向看板，而采用日期切换、会议室卡片、30 分钟时间段按钮组和底部预约表单。
- 理由：该模式适合手机阅读和触控，且能保留原有最多 2 小时、连续时间段、冲突提示等业务规则。
- 替代方案：在手机上横向滚动 PC 看板。该方案信息密度高、可读性差，不符合 H5 优先。

## Risks / Trade-offs

- [已有 artifact 链接仍包含 `desktop.svg`] → 通过占位/废弃说明或规范映射调整，避免链接失效同时明确不再作为设计依据。
- [移动端 H5 信息密度低于 PC 看板] → 使用筛选、状态摘要、会议室卡片折叠和时间段 Chip 分组降低认知负担。
- [组件库与主原型不一致] → 任务中要求同步更新 `组件清单.md`、`component-library.html` 和 `component-library.svg`。
- [响应式文档名称仍存在] → 将其内容改为 H5 适配与安全区说明，弱化 PC/平板断点内容。
- [后续实现误读旧 PC 文案] → UI 设计规范中明确“第一版以移动端 H5 为准，PC 设计图取消”。
