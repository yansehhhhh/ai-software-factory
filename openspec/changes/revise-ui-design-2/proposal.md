## Why

当前 UI 阶段产物仍以 PC/响应式 Web 为主，包含桌面端设计图与横向占用看板表达；用户明确反馈“不需要 PC 设计图”，需将本阶段 UI 产物聚焦为移动端 H5。现在修订可避免后续实现继续沿用 PC 优先设计，降低前后交付偏差。

## What Changes

- **BREAKING** 移除 PC 设计图作为本阶段交付重点，不再要求 `desktop.svg` 作为核心设计产物。
- 将 `UI原型/设计稿/index.html` 调整为移动端 H5 主原型，覆盖首页、会议预约、我的预约、扫码签到、转让确认、通知中心与会议室管理等核心移动路径。
- 将组件库 HTML/SVG/组件清单调整为移动端 H5 组件体系，突出移动卡片、底部抽屉、触控按钮、时间段选择、状态标签等组件。
- 同步修订交互说明、响应式/移动端适配文档与 UI 设计规范，明确移动端 H5 为第一版 UI 设计基准。
- 保留现有允许范围内的 UI 原型目录结构，避免引入新的并行交付目录。

## Capabilities

### New Capabilities
- `hx-meeting-mobile-h5-ui`: 定义 HX-Meeting UI 阶段产物从 PC/响应式 Web 调整为移动端 H5 的交付要求、原型范围、组件体系和移动交互规则。

### Modified Capabilities
- `hx-meeting-defect-regression`: 调整 UI 阶段回归关注点，确保后续验证不再以 PC 设计图为核心，而以移动端 H5 原型、组件库和交互一致性为准。

## Impact

- 影响文档与原型范围：`docs/# 会议室预定系统（HX-Meeting） ### 核心业务/UI原型/`。
- 影响 UI 设计规范：`docs/# 会议室预定系统（HX-Meeting） ### 核心业务/产品设计/03-UI设计规范/UI-Design-Spec.md`。
- 不涉及平台运行时代码、后端接口、数据库结构或依赖升级。
- 后续实现应以移动端 H5 交互和组件规范为优先输入。
