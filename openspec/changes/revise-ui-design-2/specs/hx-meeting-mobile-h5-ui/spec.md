## ADDED Requirements

### Requirement: UI 阶段必须以移动端 H5 为设计基准
HX-Meeting UI 阶段产物 SHALL 以移动端 H5 作为第一版设计基准，主原型、组件库、交互说明和 UI 设计规范 MUST 明确移动优先，不得继续将 PC 端设计图或桌面端看板作为核心交付依据。

#### Scenario: 查看 UI 设计规范
- **WHEN** 评审人员打开 `产品设计/03-UI设计规范/UI-Design-Spec.md`
- **THEN** 文档 MUST 明确第一版 UI 基准为移动端 H5，并说明 PC 设计图已取消或不作为核心交付

#### Scenario: 查看主 HTML 原型
- **WHEN** 评审人员打开 `UI原型/设计稿/index.html`
- **THEN** 页面 MUST 呈现移动端 H5 视口、单列布局和触控优先交互，而不是桌面端大屏布局

### Requirement: 移动端 H5 主原型必须覆盖核心业务路径
主 HTML 原型 SHALL 覆盖 HX-Meeting 核心移动业务路径，包括首页待办、会议预约、我的预约、扫码签到、转让确认、通知中心和会议室管理入口；每个路径 MUST 使用可识别的移动端页面或状态模块表达。

#### Scenario: 评审核心页面覆盖
- **WHEN** 评审人员在主 HTML 原型中浏览页面导航或原型分区
- **THEN** 原型 MUST 展示首页、预约、我的预约、签到、通知、转让确认和会议室管理相关的移动端页面或状态

#### Scenario: 评审移动端页面结构
- **WHEN** 用户查看任一核心业务页面
- **THEN** 页面 MUST 使用适合手机的顶部栏、卡片列表、底部导航、底部抽屉或全屏结果页等移动 H5 结构

### Requirement: 移动预约必须使用卡片与连续时间段选择
会议预约 UI SHALL 使用“日期选择 + 会议室卡片 + 30 分钟时间段按钮/Chip + 预约表单”的移动端交互模式。系统表达 MUST 保留连续选择、最多 2 小时、冲突提示和选中区间联动表单的业务规则。

#### Scenario: 选择可预约时间段
- **WHEN** 用户在移动端预约页选择日期并查看某个会议室卡片
- **THEN** 页面 MUST 展示按 30 分钟粒度组织的可预约时间段按钮或 Chip，并允许用户选择连续时间段

#### Scenario: 选择超过规则限制
- **WHEN** 用户尝试选择非连续时间段或超过 2 小时的预约区间
- **THEN** 原型 MUST 展示明确的规则提示，并阻止形成非法预约区间

### Requirement: 组件库必须改为移动端 H5 组件体系
UI 组件库 SHALL 围绕移动端 H5 业务组件组织，MUST 包含移动顶部栏、底部导航、快捷入口卡片、会议卡片、状态标签、时间段选择、移动表单、底部抽屉、确认面板、Toast/结果反馈等组件说明或示例。

#### Scenario: 查看组件清单
- **WHEN** 评审人员打开 `UI原型/组件库/组件清单.md`
- **THEN** 清单 MUST 以移动端 H5 组件为主，并包含关键业务组件的用途、状态和交互说明

#### Scenario: 查看组件库可视化产物
- **WHEN** 评审人员打开 `UI原型/组件库/component-library.html` 或 `component-library.svg`
- **THEN** 产物 MUST 呈现移动端 H5 组件样式，而不是桌面表格、PC 看板或 PC 导航为主的组件体系

### Requirement: PC 设计图必须被取消或降级为非核心说明
UI 阶段产物 SHALL 取消 PC 设计图的核心交付地位。若保留 `desktop.svg` 文件以避免既有链接失效，该文件内容 MUST 明确标注 PC 设计图已取消，并指向移动端 H5 主原型或移动设计稿作为评审依据。

#### Scenario: 打开 desktop.svg
- **WHEN** 评审人员打开 `UI原型/设计稿/desktop.svg`
- **THEN** 文件 MUST 不再展示可被误认为当前目标的 PC 端设计方案，并 MUST 明确提示以移动端 H5 原型为准

#### Scenario: 查看原型文件映射
- **WHEN** 评审人员查看 UI 设计规范中的原型文件映射
- **THEN** 映射 MUST 不再将桌面端总览图描述为核心设计依据，并 MUST 强调移动端 H5 相关产物

### Requirement: 移动端适配文档必须聚焦 H5 运行环境
移动端适配产物 SHALL 聚焦 H5 在手机浏览器、企业微信/微信 WebView 或类似移动 WebView 中的布局、安全区、触控、可访问性和状态反馈要求，不得以 PC 断点适配作为主要内容。

#### Scenario: 查看移动端适配文档
- **WHEN** 评审人员打开 `UI原型/移动端适配/响应式断点参考.md`
- **THEN** 文档 MUST 重点说明手机宽度、H5 安全区、底部导航、触控热区、键盘弹起和纵向卡片布局规则

#### Scenario: 查看响应式预览
- **WHEN** 评审人员打开 `UI原型/移动端适配/responsive-preview.html`
- **THEN** 预览 MUST 以移动端 H5 适配效果为主，不得以桌面端响应式布局作为主要展示内容
