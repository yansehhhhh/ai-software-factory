## 1. 产品概述

### 1.1 产品背景

用户希望搭建一个简单的个人博客，用于展示个人文章内容。产品不需要登录系统，不需要数据库，也不需要复杂后台管理能力，重点是以静态页面形式清晰、温暖、文艺地呈现博客首页、文章列表与文章详情。

该博客适合个人写作者、生活记录者、摄影爱好者、文艺类创作者或希望拥有轻量个人表达空间的用户。

### 1.2 产品目标

构建一个静态个人博客网站，满足以下目标：

- 提供一个有温度、有文艺感的个人博客展示入口
- 支持展示首页、文章列表、文章详情三类核心页面
- 内置几篇示例文章，便于后续通过修改代码手动更新内容
- 不依赖登录、不依赖数据库、不依赖后端服务
- 页面结构简单、视觉舒适、适合快速部署和二次修改

### 1.3 产品定位

一个轻量、静态、温暖文艺风格的个人博客网站。

## 4. 使用场景

### 4.1 访问首页

访客进入博客首页，看到博客名称、简介、精选文章或最近文章，并感受到温暖文艺的整体氛围。

### 4.2 查看文章列表

访客点击进入文章列表页面，浏览所有内置文章卡片，查看文章标题、摘要、日期、标签等信息。

### 4.3 阅读文章详情

访客点击某篇文章后进入详情页，阅读完整文章内容。

### 4.4 博主维护内容

博客拥有者后续通过修改代码中的文章数据或静态内容，新增、删除或调整文章。

## 7. 页面需求

## 7.1 首页

### 页面目标

作为博客的第一入口，传达个人博客的气质与主题，引导访客继续浏览文章。

### 页面内容

| 内容 | 说明 |
|---|---|
| 博客名称 | 例如：`慢慢写给时间的信` |
| 主视觉文案 | 温暖、文艺、有个人表达感 |
| 简短介绍 | 介绍博客主题或作者写作方向 |
| 精选文章 | 展示 2-3 篇示例文章 |
| 导航入口 | 可进入文章列表页面 |

### 推荐文案示例

```text
在平凡的日子里，记录一些缓慢发光的瞬间。
```

### 关键交互

- 点击“查看全部文章”进入文章列表
- 点击首页文章卡片进入对应文章详情

## 8. 内容需求

### 8.1 示例文章数量

建议内置 3 篇示例文章。

### 8.2 内容维护方式

文章内容直接内置在前端代码中，例如：

```ts
const posts = [
  {
    id: 'spring-letter',
    title: '春天来信',
    date: '2026-04-01',
    summary: '记录一个春日午后的细碎心情。',
    tags: ['生活', '随笔'],
    content: '这里是文章正文...'
  }
]
```

后续更新方式：

- 修改文章数组新增文章
- 修改已有文章标题、摘要、正文
- 删除不需要的文章对象

### 8.3 内容风格

文章内容应符合温暖文艺风格：

- 语言轻柔
- 表达个人化
- 不强调强功能性
- 适合生活随笔、读书笔记、旅行记录、个人观察

## 17. MVP 版本定义

### MVP 必须交付

一个可直接访问的静态个人博客，包含：

1. 首页
2. 文章列表页
3. 文章详情页
4. 至少 3 篇内置示例文章
5. 温暖文艺风格 UI
6. 移动端适配
7. 无登录、无数据库、无后端依赖

### MVP 不做

- 不做后台
- 不做数据库
- 不做登录
- 不做评论
- 不做文章在线编辑
- 不做搜索筛选

## 19. 风险与约束

| 风险 / 约束 | 说明 | 应对方式 |
|---|---|---|
| 无数据库 | 无法在线管理文章 | 使用静态文章数组维护 |
| 无后台 | 非技术用户更新成本较高 | 保持文章数据结构简单 |
| 静态内容 | 不适合大量文章 | 当前定位为简单个人博客 |
| 无搜索 | 文章多时查找不便 | MVP 阶段可接受 |
| 手动更新 | 需要改代码发布 | 后续可升级 Markdown 或 CMS |
