# PRD

## 产品需求文档
### 1.1 背景

用户希望快速搭建一个简单的个人博客网站，用于展示个人信息与博客文章内容。

该博客不需要登录、不需要后台管理系统、不需要数据库，所有内容以静态方式展示。适合用于个人主页、作品展示、技术文章沉淀或轻量级个人品牌展示。

---

### 1.2 产品目标

构建一个简洁、易用、静态展示的个人博客网站，用户访问后可以：

1. 在首页快速了解博主与精选内容。
2. 浏览博客文章列表。
3. 查看单篇文章详情。
4. 无需登录即可访问全部内容。
5. 网站无需数据库即可运行和部署。

---

### 1.3 需求范围

#### 1.3.1 本期包含

本期实现以下页面：

1. 首页
2. 文章列表页
3. 文章详情页

#### 1.3.2 本期不包含

以下能力不在本期范围内：

1. 用户登录 / 注册
2. 评论功能
3. 点赞 / 收藏
4. 搜索功能
5. 文章分类管理后台
6. 数据库
7. 动态发布文章
8. 服务端接口依赖

---

### 1.4 目标用户

| 用户类型 | 描述 |
|---|---|
| 博主本人 | 希望通过博客展示个人简介、文章和观点 |
| 普通访客 | 希望浏览博主发布的文章内容 |
| 招聘方 / 合作方 | 希望快速了解博主的能力、经历和内容产出 |

---

### 1.5 核心使用场景

#### 场景一：首页快速了解博主

访客进入博客首页后，可以看到博客名称、个人简介、精选文章入口，并快速进入文章列表或文章详情。

#### 场景二：浏览所有文章

访客进入文章列表页，可以查看所有静态文章，包括标题、摘要、发布日期等信息。

#### 场景三：阅读文章详情

访客点击某篇文章后，进入文章详情页，完整阅读文章内容。
