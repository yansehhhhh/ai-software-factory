# PM 产品产物包

## 1. 产品需求文档 PRD

### 产品背景

用户在日常工作、学习和生活中经常需要记录临时任务、跟踪事项完成状态，并快速清理已完成内容。当前需求是生成一个简单待办事项应用，作为 AI 软件工厂自动化主链路的第一版 MVP 交付目标，为后续 UI 设计、代码生成、测试验证提供明确输入。

本产品第一版聚焦“个人本地待办管理”，提供最基础的任务创建、查看、完成、删除能力，不引入复杂协作、登录、云同步、提醒通知等能力。

### 目标用户

- 个人用户：需要快速记录待办事项的普通用户。
- 学生：用于记录学习任务、作业、考试准备事项。
- 办公用户：用于记录日常工作事项、会议跟进项。
- Demo 体验用户：用于验证 AI 软件工厂生成应用能力的评审者或测试人员。

### 用户问题

- 用户需要快速记录一个待办事项，避免遗漏。
- 用户需要清楚看到当前还有哪些任务未完成。
- 用户需要标记任务完成状态。
- 用户需要删除不再需要的任务。
- 用户需要在空数据、异常输入等状态下获得清晰反馈。

### 产品目标

第一版 MVP 目标：

1. 支持用户新增待办事项。
2. 支持展示待办事项列表。
3. 支持将待办事项标记为完成或未完成。
4. 支持删除待办事项。
5. 支持基础输入校验，避免创建空任务。
6. 支持基础状态展示，包括空列表状态、错误提示、任务数量摘要。
7. 支持单用户本地使用，不要求后端账号体系。

### 功能范围

#### MVP 功能范围

| 模块 | 功能 | 说明 |
|---|---|---|
| 待办输入 | 新增待办 | 用户输入任务标题后创建任务 |
| 待办列表 | 查看全部待办 | 按创建时间展示任务列表 |
| 待办状态 | 标记完成/未完成 | 用户可切换任务完成状态 |
| 待办删除 | 删除任务 | 用户可删除单条任务 |
| 数据状态 | 空状态展示 | 无任务时展示引导文案 |
| 数据校验 | 输入校验 | 空内容、纯空格内容不可提交 |
| 统计摘要 | 任务数量展示 | 展示总任务数、未完成数、已完成数 |
| 本地持久化 | 本地保存 | 刷新页面后保留待办数据，建议使用 localStorage |

#### 页面范围

第一版建议仅包含一个主页面：

- 页面名称：待办事项首页
- 页面路径建议：`/` 或 `/todos`
- 页面组成：
  - 应用标题区
  - 新增待办输入区
  - 任务统计区
  - 待办列表区
  - 空状态区
  - 错误提示区

#### 状态范围

待办事项状态：

- `active`：未完成
- `completed`：已完成

页面状态：

- 初始加载状态
- 有任务状态
- 空列表状态
- 输入错误状态
- 本地存储读取失败状态，可降级为空列表并提示

### 非目标

第一版不包含以下能力：

- 用户注册、登录、退出。
- 多用户数据隔离。
- 服务端数据库存储。
- 跨设备同步。
- 任务截止时间、优先级、标签、分组。
- 搜索、筛选、排序配置。
- 批量操作。
- 日历视图。
- 推送提醒。
- 任务分享或协作。
- AI 自动拆解任务。
- 管理后台。
- 复杂权限体系。

### 核心用户流程

#### 流程一：新增待办事项

1. 用户打开待办事项应用。
2. 页面展示标题、输入框和新增按钮。
3. 用户在输入框输入任务内容。
4. 用户点击“添加”按钮或按回车提交。
5. 系统校验输入内容。
6. 若内容有效，系统创建待办事项并展示在列表顶部或底部。
7. 输入框清空。
8. 任务统计信息更新。

#### 流程二：标记任务完成

1. 用户在待办列表中看到未完成任务。
2. 用户点击任务前的复选框。
3. 系统将任务状态更新为已完成。
4. 任务标题展示完成样式，例如删除线或灰色文本。
5. 统计信息更新。

#### 流程三：恢复任务为未完成

1. 用户在列表中看到已完成任务。
2. 用户点击已勾选的复选框。
3. 系统将任务状态更新为未完成。
4. 任务标题恢复普通样式。
5. 统计信息更新。

#### 流程四：删除待办事项

1. 用户在任务列表中找到需要删除的任务。
2. 用户点击删除按钮。
3. 系统删除该任务。
4. 列表和统计信息更新。
5. 若删除后无任务，展示空状态。

#### 流程五：空列表展示

1. 用户首次打开应用或删除所有任务。
2. 系统检测当前没有待办事项。
3. 页面展示空状态文案，例如“暂无待办，添加你的第一个任务吧”。

### 验收摘要

MVP 验收需满足：

- 用户可以创建非空待办事项。
- 用户不能创建空待办事项或纯空格待办事项。
- 用户可以查看已创建的待办列表。
- 用户可以切换任意待办事项的完成状态。
- 用户可以删除任意待办事项。
- 页面可以正确展示任务统计信息。
- 无任务时展示空状态。
- 页面刷新后，已有待办事项仍然保留。
- 基础交互在桌面端和移动端可用。
- 所有核心流程具备可自动化测试场景。

## 2. 用户故事 User Stories

### 用户故事

#### US-001 新增待办事项

作为普通用户，我希望输入任务内容并添加待办事项，以便快速记录需要完成的事情。

#### US-002 阻止空任务提交

作为普通用户，我希望系统阻止我添加空白任务，以便待办列表保持有效和整洁。

#### US-003 查看待办列表

作为普通用户，我希望看到已添加的所有待办事项，以便了解当前需要处理的任务。

#### US-004 查看空状态

作为首次访问用户，我希望在没有任务时看到清晰提示，以便知道可以从哪里开始添加任务。

#### US-005 标记任务完成

作为普通用户，我希望将任务标记为已完成，以便区分已完成和未完成事项。

#### US-006 恢复任务为未完成

作为普通用户，我希望将已完成任务恢复为未完成，以便修正误操作或重新跟进任务。

#### US-007 删除待办事项

作为普通用户，我希望删除不再需要的待办事项，以便保持列表简洁。

#### US-008 查看任务统计

作为普通用户，我希望看到任务总数、已完成数和未完成数，以便快速了解任务完成进度。

#### US-009 刷新后保留任务

作为回访用户，我希望刷新页面后仍能看到之前创建的待办事项，以便继续管理未完成任务。

#### US-010 移动端使用

作为普通用户，我希望在手机屏幕上也能正常添加、完成和删除任务，以便随时记录待办事项。

## 3. WWAS 条目

### WWAS-001 新增待办事项

- Why：用户需要快速记录待完成事项，避免遗忘。
- What：提供输入框和添加按钮，用户输入任务标题后可创建待办事项。
- Acceptance：
  - 输入有效文本后点击添加，列表新增一条未完成任务。
  - 新增成功后输入框清空。
  - 新增成功后统计数据更新。
- Scope：
  - 支持文本标题。
  - 支持点击按钮提交。
  - 建议支持回车提交。
- Out of Scope：
  - 不支持截止日期。
  - 不支持优先级。
  - 不支持附件。
  - 不支持多人协作。

### WWAS-002 输入校验

- Why：防止空白或无意义任务进入列表，保持数据可用性。
- What：对新增待办输入内容进行基础校验。
- Acceptance：
  - 空字符串不可提交。
  - 纯空格不可提交。
  - 前后空格应自动裁剪。
  - 校验失败时展示明确提示。
- Scope：
  - 必填校验。
  - 空格裁剪。
  - 最大长度建议限制为 100 个字符。
- Out of Scope：
  - 不做敏感词检测。
  - 不做复杂语义校验。
  - 不做重复任务强阻断。

### WWAS-003 展示待办列表

- Why：用户需要查看当前所有待办事项并了解任务状态。
- What：展示待办事项列表，每条任务包含标题、状态控件和删除按钮。
- Acceptance：
  - 有数据时展示所有任务。
  - 每条任务可识别未完成或已完成状态。
  - 已完成任务应有明显视觉区分。
- Scope：
  - 单列表展示全部任务。
  - 默认按创建时间展示，建议新任务显示在列表顶部。
- Out of Scope：
  - 不支持筛选。
  - 不支持搜索。
  - 不支持拖拽排序。
  - 不支持分页。

### WWAS-004 标记完成状态

- Why：用户需要跟踪哪些任务已经完成。
- What：提供复选框或等效控件，支持切换任务完成状态。
- Acceptance：
  - 未完成任务点击后变为已完成。
  - 已完成任务再次点击后变为未完成。
  - 状态切换后统计数据同步更新。
  - 刷新后状态保持。
- Scope：
  - 支持单条任务状态切换。
- Out of Scope：
  - 不支持批量完成。
  - 不支持完成时间线展示。
  - 不支持审核状态。

### WWAS-005 删除待办事项

- Why：用户需要移除不再需要或误创建的任务。
- What：每条任务提供删除入口，点击后从列表中移除。
- Acceptance：
  - 点击删除后任务立即从列表消失。
  - 删除后统计数据更新。
  - 删除最后一条任务后展示空状态。
- Scope：
  - 支持单条删除。
- Out of Scope：
  - 不支持批量删除。
  - 不支持回收站。
  - 不支持撤销删除。
  - 不要求二次确认弹窗。

### WWAS-006 空状态展示

- Why：首次用户或无任务用户需要知道当前状态以及下一步操作。
- What：当列表为空时展示空状态文案。
- Acceptance：
  - 无任务时展示“暂无待办”类提示。
  - 空状态下仍可新增任务。
  - 新增任务后空状态消失。
- Scope：
  - 空状态文案。
  - 可选简单图标或插画。
- Out of Scope：
  - 不需要复杂引导流程。
  - 不需要新手教程。

### WWAS-007 任务统计

- Why：用户需要快速了解任务总体进度。
- What：展示总任务数、未完成数和已完成数。
- Acceptance：
  - 新增任务后总数和未完成数增加。
  - 完成任务后已完成数增加、未完成数减少。
  - 删除任务后对应统计更新。
- Scope：
  - 三类基础统计：总数、未完成、已完成。
- Out of Scope：
  - 不支持图表。
  - 不支持历史趋势。
  - 不支持完成率分析。

### WWAS-008 本地持久化

- Why：用户刷新页面后不应丢失已创建任务。
- What：使用浏览器本地存储保存待办事项。
- Acceptance：
  - 创建任务后刷新页面，任务仍存在。
  - 任务完成状态刷新后保持。
  - 删除任务后刷新页面，已删除任务不恢复。
- Scope：
  - localStorage 或等价浏览器本地存储。
  - 单设备单浏览器保存。
- Out of Scope：
  - 不支持云端同步。
  - 不支持跨浏览器同步。
  - 不支持账号数据迁移。

- 输入：100 个字符的任务标题。

- 输入：超过 100 个字符的任务标题。

  1. 连续快速输入并提交多个任务。

### 异常与错误状态

#### E-001 空输入提交

- 输入：空字符串。
- 预期结果：
  - 不创建任务。
  - 展示错误提示。
  - 统计不变。

#### E-002 纯空格提交

- 输入：`     `
- 预期结果：
  - 不创建任务。
  - 展示错误提示。
  - 统计不变。

#### E-003 localStorage 不可用

- 场景：浏览器禁用 localStorage 或写入失败。
- 预期结果：
  - 应用仍可在当前页面内使用。
  - 展示非阻断提示，例如“本地保存不可用，刷新后数据可能丢失”。
  - 不应导致页面崩溃。

#### E-004 localStorage 数据格式损坏

- 场景：本地存储中的待办数据不是合法 JSON 或结构不符合预期。
- 预期结果：
  - 应用应降级为空列表或忽略损坏数据。
  - 不应白屏。
  - 可提示“历史数据读取失败”。

#### E-005 删除不存在的任务

- 场景：任务已被删除后重复触发删除操作。
- 预期结果：
  - 应用不崩溃。
  - 列表保持正确。
  - 统计不出现负数。
