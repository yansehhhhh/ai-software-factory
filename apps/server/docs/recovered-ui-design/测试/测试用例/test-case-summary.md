# recovered-ui-design测试用例摘要

## 测试范围

覆盖用户身份、核心业务流程、异常约束、管理端操作、前后端集成和 E2E 验收。

## 必测分组

| 分组 | 覆盖内容 | 优先级 |
| --- | --- | --- |
| 用户身份与权限 | 登录态、普通用户、管理员权限 | P0 |
| 核心业务流程 | 查询、创建、详情、取消、状态流转 | P0 |
| 异常约束 | 参数校验、冲突、无权限、业务限制 | P0 |
| 管理端 | 审批、维护、服务确认、统计看板 | P1 |
| 集成验收 | 前端真实请求后端，禁止 mock API | P0 |

## 通过准则

P0 用例全部通过，关键接口不得出现 404/500，测试报告需记录实际验证结果。
