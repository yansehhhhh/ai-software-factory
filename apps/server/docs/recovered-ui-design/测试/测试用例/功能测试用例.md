# recovered-ui-design功能测试用例

## TC-001 当前用户信息

- 优先级：P0
- 步骤：使用有效 token 打开首页。
- 预期：用户信息接口返回 200，首页正常展示用户身份。

## TC-002 核心列表查询

- 优先级：P0
- 步骤：进入核心业务列表或首页。
- 预期：列表接口返回 200，页面展示真实后端数据。

## TC-003 创建核心业务记录

- 优先级：P0
- 步骤：填写合法表单并提交。
- 预期：创建接口返回成功，页面展示创建结果。

## TC-004 权限校验

- 优先级：P0
- 步骤：用普通用户访问管理员接口。
- 预期：返回 403，前端展示无权限状态。

## TC-005 异常输入校验

- 优先级：P0
- 步骤：提交缺失必填项或非法参数。
- 预期：后端返回明确错误，前端不伪造成功结果。

## TC-006 端到端验收

- 优先级：P0
- 步骤：启动 generated 后端和前端，执行 Playwright 测试。
- 预期：核心 happy path 通过，报告记录失败项与复现步骤。
