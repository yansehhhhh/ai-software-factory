# PM 产品产物包

## 1. 产品需求文档 PRD

### 产品背景

公司内部需要一个统一、轻量、可审计的投票页面，用于组织内部调研、意见征集、活动选择、制度反馈等场景。当前需求明确要求员工通过工号和密码登录，管理员可创建包含多个问题的投票问卷，并支持单选、多选和“其他”自定义答案。投票结果采用半匿名展示：普通用户只能在管理员公开后查看统计结果，管理员可查看统计结果和实名投票明细，并支持 Excel 导出。

### 目标用户

| 用户角色 | 说明 | 核心诉求 |
|---|---|---|
| 普通员工 | 使用工号和密码登录，参与投票问卷 | 快速完成投票、确认提交状态、在公开后查看统计结果 |
| 管理员 | 创建和管理投票问卷、维护员工账号、查看和导出结果 | 低成本配置问卷、控制投票周期、查看实名明细、导出 Excel |
| 系统维护人员 | 负责部署、初始化配置和故障排查 | 保证账号、权限、数据安全和系统可用性 |

### 用户问题

1. 公司内部投票需要确保每名员工只能提交一次，避免重复投票。
2. 投票问卷可能包含多个问题，且每个问题的题型和选项不同。
3. 部分问题需要允许员工选择“其他”并填写自定义答案。
4. 普通员工不应直接看到实名投票明细。
5. 管理员需要在后台创建问卷、维护员工账号、设置管理员角色。
6. 投票需要支持开始时间和截止时间，避免提前或超期提交。
7. 管理员需要导出统计结果和实名明细，便于归档和后续分析。
8. 员工投票和后台管理均需要适配 PC 与手机响应式场景。

### 产品目标

#### 第一版 MVP 目标

1. 支持工号 + 密码登录。
2. 支持员工账号维护和管理员角色设置。
3. 支持管理员创建、编辑、发布、关闭投票问卷。
4. 支持问卷设置开始时间和截止时间。
5. 支持多个问题，每个问题可配置为单选或多选。
6. 支持每个问题配置选项，并可启用“其他”自定义答案。
7. 支持员工在有效时间内提交投票，每个问卷每名员工仅可提交一次，提交后不可修改。
8. 支持管理员查看统计结果和实名明细。
9. 支持管理员手动公开统计结果，普通用户公开后可查看统计。
10. 支持管理员导出统计结果和实名明细 Excel。
11. 支持 PC 和手机端响应式展示。

#### 成功指标建议

| 指标 | 目标 |
|---|---|
| 员工完成一次投票耗时 | 常规问卷不超过 3 分钟 |
| 重复提交拦截率 | 100% 拦截同一员工对同一问卷的重复提交 |
| 管理员创建问卷耗时 | 10 个问题以内问卷不超过 10 分钟 |
| 投票结果导出成功率 | 正常数据下导出成功率 100% |
| 移动端核心流程可用性 | 登录、查看问卷、提交投票、查看统计均可完成 |

### 非目标

第一版不包含以下能力：

1. 不支持微信、企业微信、钉钉、LDAP、SSO 等第三方登录。
2. 不支持匿名投票；管理员始终可查看实名明细。
3. 不支持员工提交后修改答案。
4. 不支持问卷模板库、题库、复杂逻辑跳转。
5. 不支持评分题、填空题、矩阵题、排序题等题型；仅支持单选、多选和“其他”自定义补充。
6. 不支持实时大屏、复杂图表分析和交叉分析。
7. 不支持多租户、多公司隔离。
8. 不支持复杂审批流；管理员可直接创建和发布。
9. 不支持短信、邮件、站内信通知。
10. 不支持外部用户访问。

### 核心用户流程

#### 员工投票流程

1. 员工打开投票系统。
2. 输入工号和密码登录。
3. 系统展示问卷列表。
4. 员工选择一个进行中的问卷。
5. 系统展示问卷说明、开始时间、截止时间和题目列表。
6. 员工按题目选择单选或多选答案。
7. 若题目启用了“其他”选项，员工可选择“其他”并填写自定义答案。
8. 员工点击提交。
9. 系统校验：是否登录、问卷是否在有效时间内、是否已经提交、必填题是否完成、多选数量是否合法、“其他”是否填写。
10. 校验通过后保存提交记录。
11. 系统展示提交成功页，并提示“每个问卷仅可提交一次，提交后不可修改”。
12. 若管理员已公开结果，员工可查看统计；否则展示“结果暂未公开”。

#### 管理员创建问卷流程

1. 管理员登录系统。
2. 进入后台管理。
3. 新建问卷，填写标题、说明、开始时间、截止时间。
4. 新增题目，选择题型：单选或多选。
5. 配置题目选项，设置是否允许“其他”自定义答案。
6. 保存草稿。
7. 预览问卷。
8. 确认无误后发布问卷。
9. 问卷进入未开始或进行中状态。
10. 到达截止时间后问卷自动结束，管理员也可手动关闭。

#### 管理员查看和导出结果流程

1. 管理员进入后台问卷管理列表。
2. 选择目标问卷，进入结果页。
3. 查看每个题目的统计结果，包括选项票数和占比。
4. 查看实名明细，包括员工信息、提交时间和选择内容。
5. 点击导出统计结果 Excel。
6. 点击导出实名明细 Excel。
7. 根据需要手动公开统计结果。
8. 公开后普通员工可在前台查看该问卷统计。

#### 员工账号维护流程

1. 管理员进入员工账号管理。
2. 新增员工，填写工号、姓名、部门、初始密码和角色。
3. 编辑员工信息或重置密码。
4. 设置员工是否具备管理员角色。
5. 停用离职或不再需要访问的员工账号。

### 状态定义

#### 问卷状态

| 状态 | 触发条件 | 员工可见 | 员工可提交 | 管理员操作 |
|---|---|---|---|---|
| 草稿 | 管理员保存但未发布 | 否 | 否 | 编辑、发布、删除 |
| 未开始 | 已发布且当前时间早于开始时间 | 是 | 否 | 编辑有限信息、关闭 |
| 进行中 | 当前时间在开始和截止时间之间 | 是 | 是 | 查看结果、关闭、公开/取消公开 |
| 已截止 | 当前时间晚于截止时间 | 是 | 否 | 查看结果、导出、公开/取消公开 |
| 已关闭 | 管理员手动关闭 | 是 | 否 | 查看结果、导出、公开/取消公开 |

#### 员工提交状态

| 状态 | 说明 |
|---|---|
| 未提交 | 员工尚未提交该问卷 |
| 已提交 | 员工已提交该问卷，不可修改 |
| 不可提交 | 问卷未开始、已截止、已关闭或员工账号无效 |

#### 结果公开状态

| 状态 | 说明 |
|---|---|
| 未公开 | 普通员工不可查看统计结果；管理员可查看 |
| 已公开 | 普通员工可查看统计结果，但不可查看实名明细 |

#### 数据安全要求

1. 密码必须加密存储，不允许明文保存。
2. 普通员工接口不得返回实名明细数据。
3. 管理员导出的实名明细应包含操作权限校验。
4. 停用员工不得登录和提交投票。
5. 重复提交需要在服务端强校验，不能只依赖前端按钮禁用。
6. 投票提交记录需要保留提交时间和员工身份，用于管理员审计。

### 验收摘要

1. 普通员工可通过有效工号和密码登录。
2. 无效工号、错误密码、停用账号均不可登录，并展示明确错误提示。
3. 管理员可新增和编辑员工账号，并设置普通员工或管理员角色。
4. 管理员可创建包含多个问题的问卷。
5. 每个问题可配置单选或多选。
6. 每个问题可配置多个选项，并可启用“其他”选项。
7. 员工在进行中的问卷中可提交答案。
8. 每名员工对同一问卷只能提交一次。
9. 提交后再次进入问卷，系统提示已提交且不可修改。
10. 未开始、已截止、已关闭的问卷不可提交。
11. 普通员工在结果未公开时不能查看统计。
12. 管理员公开统计后，普通员工可查看统计结果。
13. 普通员工任何情况下都不能查看实名明细。
14. 管理员可查看统计结果和实名明细。
15. 管理员可导出统计结果 Excel 和实名明细 Excel。
16. PC 和手机端均可完成员工投票主流程。
17. PC 和手机端均可完成后台关键管理流程。

#### US-001 工号密码登录

- Given 员工账号存在且状态为启用，When 员工输入正确工号和密码，Then 系统登录成功并进入问卷列表页。
- Given 工号不存在，When 用户提交登录，Then 系统提示“工号或密码错误”。
- Given 密码错误，When 用户提交登录，Then 系统提示“工号或密码错误”。
- Given 员工账号被停用，When 用户提交登录，Then 系统提示“账号已停用，请联系管理员”。

#### US-006 创建投票问卷

- Given 管理员已登录，When 管理员填写问卷标题、说明、开始时间、截止时间并保存，Then 系统创建问卷草稿。
- Given 截止时间早于或等于开始时间，When 管理员保存问卷，Then 系统阻止保存并提示“截止时间必须晚于开始时间”。
- Given 问卷缺少标题，When 管理员保存问卷，Then 系统提示标题必填。

#### US-009 配置单选/多选题

- Given 管理员正在编辑问卷，When 管理员新增题目并选择单选或多选，Then 系统保存题型配置。
- Given 题目少于 2 个选项，When 管理员保存题目，Then 系统提示“每个题目至少需要 2 个选项”。

#### US-018 其他自定义答案

- Given 题目启用了“其他”选项，When 员工选择“其他”但未填写内容并提交，Then 系统提示“请填写其他内容”。
- Given 题目启用了“其他”选项，When 员工选择“其他”并填写内容，Then 系统保存自定义答案。
- Given 题目未启用“其他”选项，When 员工查看题目，Then 不展示“其他”输入入口。

#### US-020 防重复提交

- Given 员工已提交某问卷，When 员工再次打开该问卷，Then 系统显示已提交状态且不允许再次提交。
- Given 员工已提交某问卷，When 员工绕过前端再次调用提交接口，Then 服务端拒绝并返回重复提交错误。

#### US-021 查看统计结果

- Given 管理员已登录，When 管理员打开问卷结果页，Then 系统展示每个题目下各选项票数和占比。
- Given 多选题存在多项选择，When 系统统计结果，Then 每个选项按被选择次数统计。
- Given 存在“其他”自定义答案，When 管理员查看结果，Then 系统展示“其他”汇总数量并可在实名明细中看到具体内容。

#### US-024 普通员工查看公开统计

- Given 问卷结果已公开，When 普通员工打开统计结果页，Then 系统展示选项票数和占比。
- Given 问卷结果未公开，When 普通员工打开统计结果页，Then 系统不展示统计数据并提示“结果暂未公开”。
- Given 普通员工查看统计结果，When 页面返回数据，Then 不包含员工姓名、工号等实名明细。

#### US-026/US-027 Excel 导出

- Given 管理员已登录且问卷存在提交数据，When 管理员点击导出统计结果，Then 系统下载统计结果 Excel。
- Given 管理员已登录且问卷存在提交数据，When 管理员点击导出实名明细，Then 系统下载实名明细 Excel。
- Given 普通员工调用导出接口，When 请求发送，Then 系统返回无权限错误。

## 3. WWAS 条目

### WWAS-001 工号密码登录

- Why：公司内部投票需要确认参与者是内部员工，并能基于员工身份控制每人仅提交一次。
- What：交付工号 + 密码登录能力，支持普通员工和管理员角色识别。
- Acceptance：
  - 有效启用账号可登录。
  - 工号不存在或密码错误不可登录。
  - 停用账号不可登录。
  - 登录后系统能识别普通员工和管理员权限。
- Scope：登录页、登录接口、会话管理、权限识别。
- Out of Scope：SSO、LDAP、企业微信、短信验证码、邮箱登录。

### WWAS-002 员工账号与角色维护

- Why：系统需要维护员工可登录账号，并授权部分员工成为管理员。
- What：交付员工账号管理功能，包括新增、编辑、停用、重置密码、角色设置。
- Acceptance：
  - 管理员可新增员工账号。
  - 管理员可编辑姓名、部门、角色和状态。
  - 管理员可重置员工密码。
  - 普通员工不可进入账号管理页或调用账号管理接口。
- Scope：员工列表、新增/编辑表单、角色字段、账号状态。
- Out of Scope：批量导入、组织架构同步、密码自助找回。

### WWAS-003 问卷创建与发布

- Why：管理员需要灵活创建不同主题的内部投票问卷。
- What：交付问卷创建、编辑、保存草稿、发布、关闭能力。
- Acceptance：
  - 管理员可填写标题、说明、开始时间、截止时间。
  - 截止时间必须晚于开始时间。
  - 草稿不对员工可见。
  - 发布后员工可在列表中看到问卷。
  - 已关闭问卷不可提交。
- Scope：问卷管理列表、问卷编辑页、发布/关闭操作。
- Out of Scope：审批流、问卷模板、自动通知。

### WWAS-004 多问题与题型配置

- Why：一次投票可能需要收集多个问题的意见，不同问题需要单选或多选。
- What：交付多问题配置能力，每个问题支持单选或多选，并配置多个选项。
- Acceptance：
  - 一个问卷可包含多个问题。
  - 每个问题可选择单选或多选。
  - 每个问题至少包含 2 个选项。
  - 员工端按题型正确展示单选控件或多选控件。
- Scope：题目列表、题型字段、选项配置、员工端渲染。
- Out of Scope：填空题、评分题、矩阵题、逻辑跳转。

### WWAS-005 “其他”自定义答案

- Why：预设选项不能覆盖全部员工意见，需要允许员工提交补充答案。
- What：交付可配置的“其他”选项，员工选择后可填写自定义文本。
- Acceptance：
  - 管理员可为题目开启或关闭“其他”选项。
  - 开启后员工端展示“其他”选项和输入框。
  - 选择“其他”后自定义文本必填。
  - 管理员可在实名明细中查看具体自定义答案。
- Scope：题目配置、员工投票表单、提交校验、结果明细展示。
- Out of Scope：对自定义答案做自动分类、敏感词审核、AI 汇总。

### WWAS-006 一人一次且提交后不可修改

- Why：投票公平性要求同一员工对同一问卷只能提交一次，且提交后不能变更结果。
- What：交付服务端防重复提交和提交后只读状态。
- Acceptance：
  - 员工首次提交成功后生成提交记录。
  - 员工再次访问显示已提交，不展示可提交按钮。
  - 绕过前端直接重复提交时服务端拒绝。
  - 管理员可在实名明细中看到该员工提交记录。
- Scope：提交接口、唯一约束、提交状态展示。
- Out of Scope：撤回、修改、管理员代改答案。

### WWAS-007 投票时间控制

- Why：投票需要在指定时间范围内开放，避免提前或过期提交。
- What：交付开始时间、截止时间和问卷状态判断。
- Acceptance：
  - 未开始问卷不可提交。
  - 进行中问卷可提交。
  - 已截止问卷不可提交。
  - 管理员手动关闭后不可提交。
  - 服务端以系统时间为准进行最终校验。
- Scope：时间字段、状态计算、前后端提交限制。
- Out of Scope：按部门分批开放、延期审批、时区多地区配置。

### WWAS-008 结果统计与半匿名展示

- Why：员工需要了解整体投票情况，但实名明细只应对管理员可见。
- What：交付统计结果、实名明细和权限隔离。
- Acceptance：
  - 管理员可查看统计结果和实名明细。
  - 普通员工仅在结果公开后查看统计结果。
  - 普通员工不能查看实名明细。
  - 普通员工统计结果接口不返回姓名、工号、提交明细。
- Scope：统计页、实名明细页/表格、权限校验、公开状态。
- Out of Scope：完全匿名投票、复杂交叉分析、部门维度钻取。

### WWAS-009 Excel 导出

- Why：投票结果需要归档、汇报和离线分析。
- What：交付统计结果 Excel 导出和实名明细 Excel 导出。
- Acceptance：
  - 管理员可下载统计结果 Excel。
  - 管理员可下载实名明细 Excel。
  - 普通员工不可导出。
  - 导出文件包含问卷标题、导出时间和必要字段。
- Scope：导出按钮、导出接口、Excel 文件生成。
- Out of Scope：定时导出、邮件发送、复杂图表报表。

### WWAS-010 PC 与手机响应式适配

- Why：员工和管理员可能在不同设备上访问系统，核心流程需要可用。
- What：交付 PC 和手机端响应式页面。
- Acceptance：
  - 手机端可完成登录、查看问卷、投票提交、查看公开统计。
  - PC 端可完成所有员工和后台管理功能。
  - 后台关键操作在手机端可用，不出现遮挡或无法点击。
- Scope：登录页、问卷列表、投票页、统计页、后台关键页面响应式布局。
- Out of Scope：原生 App、小程序、离线访问。

### Demo 使用说明

1. 使用管理员账号 `A0001` 登录后台。
2. 在员工账号管理中确认存在普通员工 `E10001` 和 `E10002`。
3. 创建“2026 年团建活动投票”问卷。
4. 添加两个题目：一个单选题、一个多选题，并均启用“其他”。
5. 发布问卷，并确保当前时间处于开始时间和截止时间之间。
6. 使用员工账号 `E10001` 登录员工端并提交投票。
7. 使用管理员账号查看统计结果和实名明细。
8. 管理员公开统计结果。
9. 员工再次登录查看公开统计。
10. 管理员导出统计结果和实名明细 Excel。

### 测试使用说明

1. 登录测试使用有效账号、错误密码、停用账号三类数据。
2. 投票测试使用未提交员工、已提交员工、管理员员工三类身份。
3. 时间测试构造未开始、进行中、已截止、已关闭四类问卷。
4. 权限测试分别使用未登录、普通员工、管理员三类身份访问同一资源。
5. 统计测试至少包含：无提交、单人提交、多人提交、多选题、“其他”答案。
6. 导出测试需要校验文件可下载、文件名合理、字段完整、普通员工无权限。

## 6. UI 设计规范

### 设计目标

1. 内部系统风格：清晰、稳重、高效，避免过度视觉装饰。
2. 员工投票路径尽可能短，突出问卷状态、题目和提交按钮。
3. 管理后台强调信息密度、可编辑性和结果可读性。
4. PC 与手机响应式适配，确保核心操作在移动端可完成。
5. 权限差异清晰，避免普通员工误以为可查看实名数据。

### 视觉风格

| 项目 | 建议 |
|---|---|
| 主色 | 企业蓝 `#2563EB`，用于主按钮、选中态、链接 |
| 成功色 | `#16A34A`，用于提交成功、已公开状态 |
| 警告色 | `#F59E0B`，用于未开始、即将截止、未公开提示 |
| 错误色 | `#DC2626`，用于校验错误、失败状态 |
| 中性色 | `#111827`、`#374151`、`#6B7280`、`#E5E7EB`、`#F9FAFB` |
| 背景 | 页面背景 `#F3F4F6`，卡片背景 `#FFFFFF` |
| 圆角 | 8px-12px，表单控件和卡片保持一致 |
| 阴影 | 轻量阴影，仅用于卡片和弹窗 |

### 字体与排版

### 布局规范

#### PC 端

1. 登录页：居中登录卡片，宽度约 360-420px。
2. 员工端：顶部导航 + 内容居中容器，最大宽度 960-1120px。
3. 投票页：问卷信息卡片 + 题目卡片纵向排列 + 底部提交区。
4. 后台：左侧导航 + 顶部栏 + 主内容区域。
5. 表格页：顶部筛选/操作区，中部表格，底部分页。
6. 编辑页：基本信息表单在上，题目配置列表在下，右上或底部提供保存/发布按钮。

#### 手机端

1. 登录页：表单宽度占屏幕 100%，左右边距 16px。
2. 问卷列表：卡片式列表，状态标签和时间清晰可见。
3. 投票页：单列布局，题目卡片间距 12-16px。
4. 提交按钮：建议固定在底部或页面底部明显位置。
5. 后台管理：优先卡片化展示；表格可转换为列表卡片。
6. 复杂操作如问卷编辑，可使用分步或折叠面板降低拥挤感。

### 组件规范

#### 按钮

| 类型 | 使用场景 | 样式 |
|---|---|---|
| 主按钮 | 登录、提交投票、保存、发布、导出 | 蓝底白字 |
| 次按钮 | 取消、返回、预览 | 白底灰边 |
| 危险按钮 | 关闭问卷、停用账号 | 红色或红色描边 |
| 文本按钮 | 查看详情、编辑 | 蓝色文字 |

按钮状态需包含：默认、悬停、按下、禁用、加载中。

#### 表单

1. 必填字段使用红色星号标记。
2. 校验错误展示在字段下方。
3. 日期时间选择器需展示格式提示。
4. 密码字段默认隐藏，支持显示/隐藏切换。
5. 题目选项支持新增、删除、排序。
6. 删除题目或选项需二次确认，避免误删。

#### 状态标签

| 状态 | 标签文案 | 建议颜色 |
|---|---|---|
| 草稿 | 草稿 | 灰色 |
| 未开始 | 未开始 | 橙色 |
| 进行中 | 进行中 | 绿色 |
| 已截止 | 已截止 | 灰色 |
| 已关闭 | 已关闭 | 红色 |
| 已提交 | 已提交 | 蓝色 |
| 结果已公开 | 已公开 | 绿色 |
| 结果未公开 | 未公开 | 橙色 |

#### 投票题目控件

1. 单选题使用 Radio 列表。
2. 多选题使用 Checkbox 列表。
3. “其他”选项被选择后展示输入框。
4. 输入框 placeholder 示例：“请输入其他答案”。
5. 必填题未完成时，题目卡片顶部或底部展示错误提示。

#### 统计展示

1. 每个问题使用独立统计卡片。
2. 选项展示：选项名称、票数、百分比、进度条。
3. 多选题需提示“多选题百分比按提交人数计算，合计可能超过 100%”。
4. 无数据时展示空状态：“暂无提交数据”。
5. 普通员工统计页不得出现工号、姓名、提交时间。

### 交互规范

1. 提交投票前弹出确认提示：“提交后不可修改，确认提交吗？”
2. 提交过程中按钮进入 loading，防止重复点击。
3. 提交成功后跳转成功页或显示成功状态。
4. 管理员发布问卷前需校验问卷完整性。
5. 管理员关闭问卷前弹出确认提示。
6. 管理员公开结果前提示：“公开后普通员工可查看统计结果”。
7. 导出按钮点击后显示下载中状态，失败时展示错误提示。

### 可访问性要求

1. 表单控件需有明确 label。
2. Radio 和 Checkbox 可通过键盘切换。
3. 错误提示颜色之外需有文本说明。
4. 颜色对比度满足常规可读性要求。
5. 移动端点击区域建议不小于 44px 高度。

### 关键页面 UI 要点

#### 登录页

- 标题：公司内部投票系统。
- 字段：工号、密码。
- 操作：登录。
- 错误：工号或密码错误、账号已停用。

#### 员工问卷列表页

- 顶部展示用户姓名和退出登录入口。
- 列表卡片展示问卷标题、状态、开始时间、截止时间、提交状态、结果公开状态。
- 主操作按钮根据状态变化：去投票、查看提交、查看统计、未开始、已截止。

#### 员工投票页

- 顶部问卷说明和截止时间。
- 中部题目卡片。
- 底部提交按钮。
- 已提交时展示只读状态，不允许修改。

#### 管理员问卷编辑页

## 10. 术语表

| 术语 | 定义 |
|---|---|
| 工号 | 公司内部员工唯一编号，用于登录和身份识别 |
| 普通员工 | 具备投票和查看公开统计权限的用户 |
| 管理员 | 具备问卷管理、员工账号管理、结果查看和导出权限的用户 |
| 投票问卷 | 一组用于收集员工选择的题目集合 |
| 题目 | 问卷中的单个问题，第一版支持单选和多选 |
| 单选题 | 员工只能选择一个选项的题目 |
| 多选题 | 员工可以选择多个选项的题目 |
| 选项 | 题目下可供员工选择的预设答案 |
| 其他选项 | 一种特殊选项，允许员工填写自定义文本答案 |
| 自定义答案 | 员工选择“其他”后填写的文本内容 |
| 半匿名 | 普通员工只能查看统计，管理员可查看实名明细的结果展示方式 |
| 实名明细 | 包含员工工号、姓名、部门、提交时间和具体答案的投票记录 |
| 统计结果 | 按题目和选项聚合后的票数、占比等数据 |
| 结果公开 | 管理员手动允许普通员工查看统计结果的操作 |
| 草稿 | 管理员保存但未发布的问卷状态，员工不可见 |
| 未开始 | 已发布但尚未到开始时间的问卷状态 |
| 进行中 | 当前时间处于开始时间和截止时间之间，员工可提交的状态 |
| 已截止 | 当前时间已超过截止时间，员工不可再提交的状态 |
| 已关闭 | 管理员手动关闭问卷，员工不可再提交的状态 |
| 提交记录 | 某员工对某问卷的一次投票提交，包含提交时间和答案 |
| 一人一次 | 同一员工对同一问卷只能提交一次 |
| Excel 导出 | 管理员将统计结果或实名明细下载为 Excel 文件 |
| 响应式适配 | 页面能在 PC 和手机屏幕下保持可读、可操作 |

## 11. 待确认问题

1. 密码安全策略是否需要明确，例如初始密码规则、最小长度、复杂度、错误次数锁定、定期修改。
2. 员工是否需要首次登录强制修改初始密码。
3. 管理员新增员工时，密码是手动设置、系统生成，还是统一初始密码。
4. 是否需要支持员工批量导入；当前 MVP 仅定义后台手动新增/编辑。
5. 问卷发布后是否允许编辑题目和选项；建议第一版若已有提交则禁止修改题目和选项，仅允许关闭或公开结果。
6. 多选题是否需要配置最少选择数和最多选择数；当前 MVP 默认至少选择 1 项，不限制最多选择数。
7. 截止时间的边界规则是否采用“提交时间必须早于截止时间”；本文档建议等于截止时间时不可提交。
8. 普通员工在未提交问卷前是否允许查看已公开统计；建议仅公开后可看，不强制要求必须已提交，需确认。
9. 管理员是否可以删除问卷；若允许，已有提交数据的问卷是否禁止删除或仅软删除。
10. 实名明细 Excel 需要包含哪些固定字段，是否包含部门、角色、客户端类型、IP 地址。
11. 统计结果 Excel 是否需要包含“其他”自定义文本明细，还是只在实名明细中包含。
12. 是否需要记录管理员导出日志和公开/取消公开操作日志。
13. 手机端后台管理是否需要支持完整问卷编辑，还是仅支持查看结果、公开结果和关闭问卷等关键操作。
14. 系统是否需要支持多个管理员同时编辑同一问卷时的冲突处理。
15. 是否需要对“其他”自定义答案进行敏感词过滤或长度限制；本文档建议限制 200 字。
