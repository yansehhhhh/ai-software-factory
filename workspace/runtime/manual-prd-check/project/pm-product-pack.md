# PM 产品产物包

## 1. 产品需求文档 PRD

### 1.1 产品背景
用户希望搭建一个**温暖、文艺风格**的静态个人博客，用于展示个人内容与文章创作成果。第一版不涉及登录、评论、数据库或后台管理，仅聚焦公开内容展示与基础浏览体验。

该产品适合用于：
- 个人写作爱好者的线上内容展示
- 独立创作者的作品沉淀
- 作为简洁可部署的静态站点示例

### 1.2 目标用户
1. **博客访客**：浏览站点首页、文章列表和文章详情内容的普通访问者。
2. **博客主人**：以站点维护者身份，关注站点风格表达、文章展示效果与内容编排，但不在本期范围内提供后台编辑能力。

### 1.3 用户问题
当前用户在构建个人博客时可能面临以下问题：
- 想要一个风格统一、情绪化表达明确的博客站点，但不希望系统过重。
- 只需要内容展示，不需要登录、评论、管理后台等复杂功能。
- 希望首页能传达个人气质，文章列表便于浏览，详情页适合沉浸式阅读。
- 希望产品可以快速落地为静态页面，便于部署与演示。

### 1.4 产品目标
本产品第一版目标如下：
- 提供一个**温暖文艺风格**的静态个人博客界面。
- 完成三个核心页面：**首页、文章列表页、文章详情页**。
- 支持用户顺畅完成“进入首页 → 浏览文章列表 → 打开文章详情”的主流程。
- 以静态内容方式实现，无需登录、无需数据库、无需动态交互依赖。
- 为后续 UI 设计、前端开发、测试用例编写提供明确产品输入。

### 1.5 功能范围
本次 MVP 范围包含：

#### A. 首页
- 展示博客名称 / 个人标语 / 简短介绍。
- 展示温暖文艺风格的首屏视觉氛围。
- 提供进入文章列表的明确入口。
- 可展示精选文章或最近文章摘要（静态内容）。

#### B. 文章列表页
- 展示文章卡片列表。
- 每篇文章至少展示：标题、摘要、发布日期、标签、封面图（可选静态图）。
- 支持点击进入文章详情页。
- 支持基础的静态排序展示（默认按时间倒序）。

#### C. 文章详情页
- 展示文章标题、发布日期、标签、正文内容。
- 提供返回文章列表或返回首页的入口。
- 阅读区排版应突出舒适、安静、沉浸感。

#### D. 全局基础能力
- 基础站点导航（如首页 / 文章）。
- 响应式适配桌面端与移动端基础浏览。
- 使用静态数据驱动页面渲染。

### 1.6 非目标
以下内容不属于本次第一版范围：
- 登录、注册、用户中心
- 评论、点赞、收藏、分享统计
- 数据库、CMS、后台管理系统
- 文章搜索、筛选、分页的复杂交互
- 多作者协作
- 在线编辑、草稿保存
- 国际化、多语言
- SEO 深度优化与埋点分析系统

### 1.7 风格与体验要求
#### 风格关键词
- 温暖
- 文艺
- 安静
- 柔和
- 有呼吸感
- 适度留白

#### 视觉方向建议
- 主色调偏米白、奶油、浅棕、灰绿、豆沙、暖灰等低饱和色。
- 排版强调阅读舒适性，正文建议较高行高与适中段间距。
- 卡片与区块层级柔和，不使用强烈科技感视觉。
- 图片、插画、引言文案可增强情绪表达，但不喧宾夺主。

#### 交互体验原则
- 导航清晰，层级少，尽量降低操作成本。
- 页面切换直观，不引入复杂状态。
- 文章列表信息密度适中，避免过于拥挤。
- 文章详情以“阅读”为核心，不被多余元素干扰。

### 1.8 信息架构
- 首页
  - 博客标题区
  - 简介区
  - 精选文章 / 最近文章区
  - 导航入口
- 文章列表页
  - 页面标题区
  - 文章卡片列表区
- 文章详情页
  - 标题信息区
  - 正文区
  - 返回导航区

### 1.9 核心用户流程
#### 流程 1：首次访问博客
1. 用户进入首页。
2. 看到博客名称、风格化介绍与主视觉。
3. 用户点击“查看文章”进入文章列表页。

#### 流程 2：浏览文章
1. 用户在文章列表页查看多篇文章卡片。
2. 根据标题、摘要、标签或时间判断感兴趣内容。
3. 点击某篇文章进入文章详情页。

#### 流程 3：阅读文章并返回
1. 用户在文章详情页阅读完整正文。
2. 阅读结束后通过页面入口返回文章列表或首页。

### 1.10 页面需求拆解
#### 首页需求
- 必须有博客名称与一句核心标语。
- 必须有博客简介或作者自我介绍短文。
- 必须有进入文章列表的按钮或链接。
- 可展示 2~4 篇精选文章摘要。
- 页面首屏需体现风格一致性。

#### 文章列表页需求
- 文章列表应使用统一卡片样式。
- 每篇文章卡片必须包含标题、摘要、发布日期。
- 标签为可展示字段，但本期不要求可点击筛选。
- 列表为空状态需有占位说明。

#### 文章详情页需求
- 必须展示文章完整内容。
- 正文应支持基础富文本结构：标题、段落、引用、列表、图片（如有）。
- 页面需提供明确返回路径。
- 当文章不存在时，需展示友好提示页或提示文案。

### 1.11 内容策略
由于本产品为静态博客，第一版建议采用预置文章数据方式：
- 使用固定文章列表作为展示内容。
- 文章数量建议 5~8 篇，便于完整展示首页与列表页状态。
- 内容风格与页面视觉保持一致，可偏散文、随笔、旅行、阅读感悟、生活记录等。

### 1.12 非功能要求
- **性能**：页面应为轻量静态站点，首屏资源不过度堆积。
- **兼容性**：支持现代主流浏览器的基础访问。
- **可维护性**：文章数据结构清晰，便于后续增加文章内容。
- **可用性**：桌面端和移动端均可顺畅浏览。
- **可访问性**：基础文本对比度、图片 alt、可点击元素语义需合理。

### 1.13 MVP 边界
第一版只验证以下核心价值：
- 博客风格是否成立
- 内容浏览流程是否顺畅
- 静态站点结构是否完整可演示

因此 MVP 只交付：
- 首页
- 文章列表页
- 文章详情页
- 静态文章数据
- 基础导航与空/错状态

### 1.14 验收摘要
若满足以下条件，可视为本期产品需求完成：
- 用户可以从首页进入文章列表页。
- 用户可以从文章列表进入任一文章详情页。
- 页面整体视觉符合“温暖文艺”方向。
- 全站无需登录、无需数据库即可完成浏览演示。
- 对空列表、文章不存在等基础状态有明确处理。

## 2. 用户故事 User Stories

### 2.1 用户角色
- **访客**：希望轻松浏览博客内容。
- **博客主人（内容展示视角）**：希望通过页面传达个人风格与内容气质。

### 2.2 Epic 列表
1. **Epic A：首页风格展示**
2. **Epic B：文章列表浏览**
3. **Epic C：文章详情阅读**
4. **Epic D：静态内容与基础状态处理**

### 2.3 用户故事

#### Epic A：首页风格展示
**US-A1** 作为访客，我希望在首页看到博客名称和一句有风格的介绍，以便快速理解这个博客的气质与主题。  
**US-A2** 作为访客，我希望在首页看到清晰的文章入口，以便快速开始浏览内容。  
**US-A3** 作为博客主人，我希望首页风格温暖文艺，以便传达个人审美与表达方式。  
**US-A4** 作为访客，我希望首页可看到精选文章摘要，以便快速发现值得阅读的内容。  

#### Epic B：文章列表浏览
**US-B1** 作为访客，我希望在文章列表页看到多篇文章的标题、摘要和日期，以便快速决定要读哪一篇。  
**US-B2** 作为访客，我希望文章列表排版清晰舒适，以便长时间浏览时不感到压迫。  
**US-B3** 作为访客，我希望点击任意文章卡片即可进入详情页，以便阅读完整内容。  
**US-B4** 作为访客，我希望即使当前没有文章，也能看到友好的空状态提示，以便知道并非页面出错。  

#### Epic C：文章详情阅读
**US-C1** 作为访客，我希望在文章详情页看到完整正文，以便沉浸式阅读。  
**US-C2** 作为访客，我希望在详情页看到标题、日期和标签，以便理解文章背景信息。  
**US-C3** 作为访客，我希望在阅读结束后可以返回文章列表或首页，以便继续浏览其他内容。  
**US-C4** 作为访客，我希望当访问不存在的文章时收到友好提示，以便知道内容不可用。  

#### Epic D：静态内容与基础状态处理
**US-D1** 作为开发者，我希望页面由静态数据驱动，以便在不接入数据库的情况下快速实现。
**US-D2** 作为博客主人，我希望第一版不包含登录与后台，以便聚焦内容展示本身。
**US-D3** 作为访客，我希望在手机上也能正常浏览博客，以便随时查看内容。

### 2.4 验收条件

#### US-A1 验收条件
- Given 用户进入首页
- When 首页加载完成
- Then 用户应看到博客名称与一句核心标语或介绍文案

#### US-A2 验收条件
- Given 用户位于首页
- When 用户查看首屏或主要内容区
- Then 用户应能看到进入文章列表页的明确入口

#### US-A4 验收条件
- Given 首页配置了精选文章
- When 用户浏览首页内容区
- Then 用户应能看到精选文章标题或摘要信息

#### US-B1 验收条件
- Given 系统存在多篇静态文章数据
- When 用户进入文章列表页
- Then 页面应展示每篇文章的标题、摘要、发布日期

#### US-B3 验收条件
- Given 用户位于文章列表页
- When 用户点击某篇文章卡片或标题
- Then 系统应跳转至对应文章详情页

#### US-B4 验收条件
- Given 当前文章数据为空
- When 用户进入文章列表页
- Then 页面应展示友好的空状态提示，而非空白页面

#### US-C1 验收条件
- Given 用户已进入某篇文章详情页
- When 页面加载成功
- Then 用户应能看到文章完整正文内容

#### US-C2 验收条件
- Given 用户进入文章详情页
- When 页面渲染完成
- Then 页面应展示文章标题、发布日期和标签信息

#### US-C3 验收条件
- Given 用户位于文章详情页
- When 用户点击返回入口
- Then 用户应返回文章列表页或首页

#### US-C4 验收条件
- Given 用户访问了不存在的文章路径
- When 系统无法匹配到对应文章
- Then 页面应展示友好错误提示与可返回入口

#### US-D1 验收条件
- Given 本期为静态博客项目
- When 开发实现页面
- Then 页面内容应可由本地静态数据文件或内置数据结构驱动

#### US-D3 验收条件
- Given 用户使用移动端设备访问
- When 用户浏览首页、列表页、详情页
- Then 页面应保持可读、可点击、无明显布局错乱

### 2.5 优先级建议
- **P0（必须有）**：US-A1、US-A2、US-B1、US-B3、US-C1、US-C2、US-C3、US-D1
- **P1（应该有）**：US-A3、US-A4、US-B2、US-B4、US-C4、US-D3
- **P2（可后续增强）**：文章封面强化、精选推荐策略、更多风格化内容模块

## 3. WWAS 条目

### WWAS-1：首页建立博客风格识别
- **Why**：用户首次访问时需要快速感知博客主题与个人气质。
- **What**：交付一个包含博客名称、标语、简介和主入口的首页。
- **Acceptance**：
  - 首页可正常访问。
  - 页面包含博客标题、介绍文案、进入文章列表的入口。
  - 视觉调性符合温暖文艺风格。
- **Scope**：首屏展示、简介区、精选文章区、导航入口。
- **Out of Scope**：动态轮播、个性化推荐、访客行为分析。

### WWAS-2：提供文章列表浏览能力
- **Why**：访客需要快速浏览所有文章并选择感兴趣的内容。
- **What**：交付一个展示多篇文章摘要信息的文章列表页。
- **Acceptance**：
  - 列表页展示文章标题、摘要、日期、标签。
  - 用户可点击进入详情页。
  - 空列表时展示友好占位。
- **Scope**：静态列表展示、卡片布局、跳转链接、空状态。
- **Out of Scope**：分页、搜索、筛选、排序切换。

### WWAS-3：提供沉浸式文章详情阅读体验
- **Why**：博客的核心价值在于内容阅读，详情页体验决定最终感受。
- **What**：交付一个具备完整文章内容展示能力的详情页。
- **Acceptance**：
  - 详情页展示标题、日期、标签、正文。
  - 支持返回列表或首页。
  - 不存在文章时展示友好错误提示。
- **Scope**：文章正文排版、元信息展示、返回导航、404/缺失处理。
- **Out of Scope**：评论区、阅读进度同步、目录锚点、相关文章推荐算法。

### WWAS-4：采用静态数据实现最小可交付版本
- **Why**：本期目标是快速构建可展示的个人博客 MVP，而非搭建完整内容系统。
- **What**：使用静态文章数据完成页面渲染。
- **Acceptance**：
  - 无需登录、数据库、后端接口即可运行。
  - 页面内容来自预置数据结构。
  - 可新增多篇样例文章支撑演示与测试。
- **Scope**：本地数据文件、页面静态渲染、演示数据管理。
- **Out of Scope**：后台管理、在线新增文章、数据库建模。

### WWAS-5：保证多端基础可用性
- **Why**：博客内容可能在手机和桌面端被访问，需确保基本浏览体验一致。
- **What**：交付支持基础响应式的页面布局。
- **Acceptance**：
  - 首页、列表页、详情页在移动端可正常阅读与点击。
  - 无明显溢出、遮挡、不可操作问题。
- **Scope**：基础响应式布局、字号间距调整、按钮点击区域合理化。
- **Out of Scope**：平板端精细适配、深度动画适配、设备能力检测。

## 4. 测试场景 Test Scenarios

### 4.1 测试目标
验证该静态个人博客是否满足以下目标：
- 核心页面完整可访问
- 主要浏览路径闭环可用
- 数据渲染正确
- 异常与空状态具备明确反馈
- 无登录和数据库前提下可正常展示
- 页面风格与阅读体验符合预期

### 4.2 Happy Path
1. **首页正常访问**
   - 打开站点默认进入首页
   - 可见博客标题、介绍文案、进入文章列表入口

2. **从首页进入文章列表**
   - 点击“查看文章”后成功进入文章列表页

3. **文章列表正常展示**
   - 页面展示多篇文章卡片
   - 每篇卡片展示标题、摘要、日期、标签

4. **从列表进入详情页**
   - 点击任一文章后进入对应详情页
   - URL 与内容匹配正确

5. **文章详情正常阅读**
   - 显示文章标题、日期、标签、正文
   - 正文排版可读

6. **从详情页返回**
   - 用户可通过返回入口返回列表页或首页

### 4.3 边界场景
1. **仅 1 篇文章时**
   - 列表页仍正常展示
   - 首页精选区仍不破版

2. **文章标题较长**
   - 卡片布局不应错乱
   - 详情页标题换行正常

3. **文章摘要较短或为空**
   - 列表页应有合理占位或兼容样式

4. **文章标签数量较多**
   - 标签展示不应造成明显溢出

5. **正文内容较长**
   - 详情页长文排版正常，可持续滚动阅读

6. **正文包含引用、列表、图片时**
   - 富文本结构样式正常

7. **移动端窄屏访问**
   - 导航、按钮、文本排版保持可用

### 4.4 异常与错误状态
1. **文章列表为空**
   - 列表页展示“暂无文章”类友好提示

2. **访问不存在的文章详情**
   - 页面展示“文章不存在/已下线”提示
   - 提供返回入口

3. **文章缺少封面图**
   - 卡片仍可正常展示，不因图片缺失报错

4. **文章缺少标签或摘要**
   - 页面采用兼容展示，不出现技术性报错信息

5. **静态数据字段异常**
   - 页面应尽量兜底，不出现明显空白崩溃状态

### 4.5 权限与访问场景
由于本项目无登录、无权限系统，本部分重点验证公开访问一致性：
1. 未登录用户可直接访问首页、文章列表页、文章详情页。
2. 不应出现登录拦截或鉴权提示。
3. 任意公开文章链接应可直接访问。
4. 不存在“仅作者可见”等权限分支逻辑。

### 4.6 数据校验场景
1. 文章 `slug` 唯一，避免详情页路由冲突。
2. 文章发布日期格式一致。
3. 标题不能为空。
4. 正文不能为空。
5. 标签字段可为空数组，但数据类型需一致。
6. 摘要建议有长度上限，避免卡片展示过长。

### 4.7 场景矩阵

| 场景编号 | 场景名称 | 前置条件 | 操作 | 预期结果 |
|---|---|---|---|---|
| TS-01 | 访问首页 | 站点已启动 | 打开首页 | 成功看到博客标题、介绍、入口 |
| TS-02 | 首页跳转列表 | 位于首页 | 点击查看文章 | 进入文章列表页 |

| TS-04 | 列表进入详情 | 位于列表页 | 点击某篇文章 | 进入对应详情页 |
| TS-05 | 详情展示正文 | 文章存在 | 打开详情页 | 展示完整标题与正文 |
| TS-06 | 详情返回列表 | 位于详情页 | 点击返回 | 返回列表页或首页 |

### 5.6 Demo 使用说明
- 首页可使用 `featured = true` 的文章作为精选文章展示来源。
- 列表页可使用全部正常样例记录作为主数据源。
- 详情页使用 `slug` 进行页面匹配。
- 边界样例用于 UI 演示和样式抗压测试。
- 无效样例不应进入正式展示流，但应纳入测试数据。

### 5.7 测试使用说明
- **正常测试**：使用 N1、N2、N3 验证完整流程。
- **样式边界测试**：使用 B1、B2、B3 验证长标题、空摘要、多标签。
- **数据校验测试**：使用 I1、I2、I3 验证必填、唯一性与正文完整性。
- **空状态测试**：将正常样例集替换为空数组，验证列表页空状态。
- **异常路径测试**：访问不存在的 `slug` 验证详情缺失提示。
