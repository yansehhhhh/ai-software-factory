# 简单个人博客 UI 设计规范

> 适用产品：静态个人博客  
> 页面范围：首页、文章列表页、文章详情页  
> 设计目标：简洁、清爽、内容优先、无需登录、无需数据库、可静态部署  
> 产物版本：V1.0  
> 日期：2026-04-27

---

## 1. 设计定位

### 1.1 产品气质

本个人博客以“安静阅读、轻量浏览、个人表达”为核心气质。界面应减少复杂视觉装饰，把注意力集中在标题、摘要、文章正文和导航路径上。

关键词：

- 内容优先
- 极简清爽
- 温暖克制
- 阅读友好
- 静态轻量

### 1.2 设计原则

1. **先读内容，再看装饰**：页面视觉重点始终服务于文章内容。
2. **清晰路径**：用户在首页、文章列表、文章详情之间能明确知道当前位置和下一步操作。
3. **低认知负担**：不使用复杂交互，不引入登录、评论、后台等干扰元素。
4. **移动端优先**：手机端应保持舒适阅读和易点击导航。
5. **可扩展**：后续可自然扩展“关于我”“标签”“搜索”等模块。

## 2. 页面结构规范

### 2.1 全局布局

| 区域 | 说明 |
|---|---|
| 顶部导航 | 固定或常规顶部区域，包含博客名称、首页、文章列表 |
| 主内容区 | 承载当前页面核心内容，宽度受控 |
| 页脚 | 展示版权、博客说明或静态部署提示 |

### 2.2 内容宽度

| 场景 | 建议宽度 |
|---|---|
| 桌面端页面容器 | 最大 1120px |
| 文章详情正文 | 最大 760px |
| 移动端左右边距 | 20px |
| 平板端左右边距 | 32px |
| 桌面端左右边距 | 40px |

### 2.3 响应式断点

| 断点 | 设备 | 规则 |
|---|---|---|
| `< 640px` | 手机 | 单列布局，导航压缩间距，卡片纵向排列 |
| `640px - 1023px` | 平板 | 内容保持单列，卡片可两列视情况使用 |
| `≥ 1024px` | 桌面 | 首页精选文章可双列或三列，列表保持舒适阅读宽度 |

## 3. 色彩规范

### 3.1 色彩策略

采用浅色阅读主题，使用米白背景、深灰文字和温暖琥珀色作为品牌强调色。整体避免高饱和色大面积使用。

### 3.2 设计令牌

| Token | 色值 | 用途 |
|---|---|---|
| `--color-bg` | `#FAF7F0` | 页面主背景 |
| `--color-surface` | `#FFFFFF` | 卡片、文章容器 |
| `--color-surface-soft` | `#F5EFE3` | 柔和分区背景 |
| `--color-text` | `#1F2933` | 主文本 |
| `--color-text-muted` | `#697586` | 摘要、日期、辅助说明 |
| `--color-border` | `#E7DDCE` | 分割线、卡片边框 |
| `--color-primary` | `#B7791F` | 主按钮、重点链接 |
| `--color-primary-dark` | `#8A5A14` | 主按钮悬停态 |
| `--color-accent` | `#2F6F73` | 次级强调，如标签 |
| `--color-focus` | `#2563EB` | 键盘焦点描边 |

### 3.3 使用规则

1. 正文文本与背景对比度需满足 WCAG AA，普通文本至少 4.5:1。
2. 主色主要用于按钮、链接、激活导航，不大面积铺满页面。
3. 日期、标签、摘要使用辅助色，但不得过浅。
4. 不用颜色作为唯一状态提示，链接需要下划线或明确 hover/focus 样式。

## 4. 字体与排版规范

### 4.1 字体族

```css
font-family: Inter, "Noto Sans SC", "PingFang SC", "Microsoft YaHei", system-ui, sans-serif;
```

说明：

- 英文与数字优先使用 Inter。
- 中文使用系统无衬线字体，保证跨平台稳定。
- 不依赖必须联网加载的字体，确保静态 HTML 可直接打开。

### 4.2 字号体系

| Token | 尺寸 | 行高 | 用途 |
|---|---:|---:|---|
| `text-xs` | 12px | 1.5 | 标签、辅助说明 |
| `text-sm` | 14px | 1.6 | 日期、导航、小按钮 |
| `text-base` | 16px | 1.7 | 正文、摘要 |
| `text-lg` | 18px | 1.7 | 文章列表标题、强调段落 |
| `text-xl` | 22px | 1.35 | 区块标题 |
| `text-2xl` | 28px | 1.25 | 页面标题 |
| `text-3xl` | 40px | 1.15 | 首页 Hero 标题 |

### 4.3 正文排版

文章详情页正文需满足：

- 正文字号不小于 17px。
- 行高建议 `1.85`。
- 段落间距建议 `1.1em - 1.4em`。
- 桌面端每行约 60-75 个中文/英文混合字符以内。
- 标题层级从 `h1` 到 `h3` 顺序清晰，不跳级。

## 5. 间距规范

采用 4px / 8px 基础节奏。

| Token | 值 | 用途 |
|---|---:|---|
| `space-1` | 4px | 极小间距 |
| `space-2` | 8px | 标签内距、图标间距 |
| `space-3` | 12px | 小组件间距 |
| `space-4` | 16px | 卡片内部紧凑间距 |
| `space-5` | 20px | 移动端页面边距 |
| `space-6` | 24px | 卡片内距、段落组间距 |
| `space-8` | 32px | 区块间距 |
| `space-12` | 48px | 大区块间距 |
| `space-16` | 64px | 首页 Hero 垂直间距 |

## 6. 组件规范

### 6.1 顶部导航

#### 结构

- 左侧：博客名称 / Logo 文本
- 右侧：首页、文章列表

#### 状态

| 状态 | 样式 |
|---|---|
| 默认 | 深灰文本 |
| 悬停 | 主色文字，轻微背景色 |
| 当前页 | 主色文字 + 浅色胶囊背景 |
| 键盘聚焦 | 2px 蓝色外描边 |

#### 交互要求

- 可点击区域高度不小于 44px。
- 移动端导航仍保持文本可读，不使用无标签图标导航。

### 6.2 按钮

#### 主要按钮

用于首页“查看文章”等核心 CTA。

- 背景：`--color-primary`
- 文本：白色
- 圆角：999px
- 高度：44px 以上
- 悬停：`--color-primary-dark`
- 聚焦：蓝色 focus ring

#### 次要按钮 / 文本链接

用于“阅读全文”“返回文章列表”。

- 文本主色或深灰
- 可搭配箭头符号，但不使用 emoji 作为图标
- hover 时下划线或背景变化

### 6.3 文章卡片

#### 内容

- 标题
- 摘要
- 发布日期
- 标签（可选）
- 阅读入口

#### 样式

- 背景：白色
- 边框：1px `--color-border`
- 圆角：20px
- 内距：24px
- 阴影：轻微阴影，不制造强烈浮层感

#### 状态

| 状态 | 表现 |
|---|---|
| 默认 | 边框 + 轻微阴影 |
| 悬停 | 上移 2px，阴影增强，边框偏主色 |
| 聚焦 | 明确 focus ring |

### 6.4 标签

- 小号文字
- 圆角胶囊
- 背景使用 `--color-surface-soft`
- 文本使用 `--color-accent`
- 不依赖标签颜色表达唯一语义

### 6.5 页脚

- 内容居中或左对齐均可，但与主容器对齐。
- 文本使用辅助色。
- 顶部可加细分割线。

## 7. 页面设计规范

### 7.1 首页

首页应表达博客定位，并提供清晰文章入口。

#### 必备模块

1. 顶部导航
2. Hero 区域
3. 精选文章
4. 最新文章
5. 页脚

#### Hero 区域

- 标题：一句清晰欢迎语，例如“记录技术、生活与思考”。
- 副标题：简短说明博客内容方向。
- 主 CTA：查看文章。
- 辅助信息：可展示文章数量、静态博客说明等。

### 7.2 文章列表页

文章列表页应便于扫描。

#### 必备模块

1. 页面标题
2. 页面说明
3. 文章卡片列表
4. 空状态预留

#### 列表规则

- 桌面端可以使用纵向列表，保证摘要可读。
- 不建议瀑布流，避免阅读路径混乱。
- 每张卡片点击目标清晰，标题和“阅读全文”均可进入详情。

### 7.3 文章详情页

文章详情页必须以正文阅读为核心。

#### 必备模块

1. 返回文章列表入口
2. 文章标题
3. 发布时间与标签
4. 正文内容
5. 推荐阅读或上一篇/下一篇（可选）

#### 正文规则

- 正文容器宽度小于列表页容器。
- 段落之间保持明显间距。
- 引用、列表、小标题应有明确视觉层级。

## 8. 交互与动效规范

### 8.1 基础交互

| 元素 | 交互 |
|---|---|
| 导航链接 | hover 背景变化，focus ring |
| 文章卡片 | hover 上移 2px，focus ring |
| 按钮 | hover 颜色加深，active 轻微缩放 |
| 返回链接 | hover 下划线或左箭头位移 |

### 8.2 动效参数

| Token | 值 |
|---|---|
| 快速反馈 | 120ms |
| 标准过渡 | 180ms |
| 页面内微动效 | 220ms |
| 缓动 | `cubic-bezier(.2,.8,.2,1)` |

### 8.3 减少动态

必须支持：

```css
@media (prefers-reduced-motion: reduce) {
  * {
    animation: none !important;
    transition: none !important;
    scroll-behavior: auto !important;
  }
}
```

## 9. 可访问性规范

1. 页面必须包含 `lang="zh-CN"`。
2. 需要 `viewport` meta，禁止禁用缩放。
3. 使用语义化结构：`header`、`nav`、`main`、`article`、`footer`。
4. 提供“跳到主内容”的 skip link。
5. 所有交互元素支持键盘 Tab 访问。
6. Focus 样式不能移除。
7. 链接文本要可理解，避免只有“点击这里”。
8. 文章标题使用唯一 `h1`。
9. 装饰性图形使用 `aria-hidden="true"`。
10. 文本对比度满足 WCAG AA。

---

## 10. 静态原型实现约束

1. 不创建 Vue、React 或其他前端工程。
2. 原型使用单个 `prototype/index.html` 即可直接浏览器打开。
3. CSS 与少量 JS 可内嵌在 HTML 中。
4. 文章数据以静态数组或静态 HTML 片段维护。
5. 页面切换可使用锚点、hash 路由或同页视图切换。
6. 不依赖数据库、登录态或后端 API。

---

## 11. 质量检查清单

交付前应确认：

- [ ] 首页、文章列表、文章详情均可访问。
- [ ] 全站无登录入口和数据库依赖。
- [ ] 文章列表可点击进入详情。
- [ ] 详情页可返回列表。
- [ ] 移动端无横向滚动。
- [ ] 正文阅读宽度舒适。
- [ ] 交互元素触达区域不小于 44px。
- [ ] 键盘 Tab 可访问所有导航和文章入口。
- [ ] Focus ring 清晰可见。
- [ ] 支持 `prefers-reduced-motion`。
