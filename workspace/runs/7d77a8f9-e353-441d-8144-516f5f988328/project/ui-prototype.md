# 静态 HTML UI 原型说明

> 原型文件：`workspace/runs/7d77a8f9-e353-441d-8144-516f5f988328/project/prototype/index.html`  
> 产品：简单个人博客  
> 技术形式：单文件静态 HTML + 内嵌 CSS + 内嵌 JavaScript  
> 页面范围：首页、文章列表、文章详情

---

## 1. 原型目标

本原型用于展示“简单个人博客”的核心界面与交互体验，满足以下要求：

- 不创建 Vue / React 工程。
- 不依赖数据库。
- 不需要登录。
- 可直接用浏览器打开 `prototype/index.html` 查看。
- 通过静态文章数据模拟首页、列表、详情完整浏览链路。

## 3. 原型路由结构

原型采用 hash 路由，无需服务器即可运行。

| Hash 路径 | 页面 | 说明 |
|---|---|---|
| `#/` | 首页 | 展示博客介绍、精选文章、最新文章 |
| `#/posts` | 文章列表页 | 展示所有静态文章卡片 |
| `#/post/hello-blog` | 文章详情页 | 展示“我的第一篇博客”详情 |
| `#/post/why-write` | 文章详情页 | 展示“为什么要写博客”详情 |
| `#/post/keep-simple` | 文章详情页 | 展示“保持简单”详情 |

## 4. 页面说明

### 4.1 首页

#### 页面目标

让访客快速理解博客定位，并引导进入文章列表或第一篇文章详情。

#### 页面模块

1. 顶部导航
   - 博客名称
   - 首页入口
   - 文章列表入口
2. Hero 区域
   - 博客定位：“记录技术、生活与思考”
   - 简短说明
   - 主按钮：查看全部文章
   - 次按钮：阅读第一篇
3. 博客特点卡片
   - 无需登录
   - 无需数据库
   - 适合阅读
4. 精选文章
   - 展示 3 篇文章卡片
5. 最新文章
   - 以列表形式展示最新内容
6. 页脚
   - 版权信息
   - 静态展示说明

#### 主要交互

| 操作 | 结果 |
|---|---|
| 点击“查看全部文章” | 跳转 `#/posts` |
| 点击“阅读第一篇” | 跳转 `#/post/hello-blog` |
| 点击文章卡片标题 | 跳转对应文章详情 |
| 点击顶部“文章列表” | 跳转文章列表页 |

---

### 4.2 文章列表页

#### 页面目标

让访客快速浏览全部文章，并选择感兴趣内容阅读。

#### 页面模块

1. 页面标题：文章列表
2. 页面说明
3. 文章卡片列表
   - 发布时间
   - 标题
   - 摘要
   - 标签
   - 阅读全文入口

#### 主要交互

| 操作 | 结果 |
|---|---|
| 点击文章标题 | 进入对应详情页 |
| 点击“阅读全文” | 进入对应详情页 |
| 点击顶部“首页” | 返回首页 |

#### 空状态

原型中预留空状态组件逻辑：当文章数组为空时显示“暂无文章，稍后再来看看。”。

---

### 4.3 文章详情页

#### 页面目标

提供专注、舒适的单篇文章阅读体验。

#### 页面模块

1. 返回文章列表入口
2. 文章标题
3. 发布时间
4. 标签
5. 文章正文
6. 底部操作
   - 返回文章列表
   - 回到首页

#### 主要交互

| 操作 | 结果 |
|---|---|
| 点击“返回文章列表” | 跳转 `#/posts` |
| 点击“回到首页” | 跳转 `#/` |
| 访问不存在文章 ID | 展示“文章不存在”状态 |

## 5. 静态数据说明

文章数据维护在 `prototype/index.html` 内部的 JavaScript 数组中。

### 5.1 数据结构

```js
{
  id: 'hello-blog',
  title: '我的第一篇博客',
  date: '2026-04-27',
  tags: ['随笔', '个人博客'],
  summary: '这是我的第一篇个人博客文章，用来记录博客的开始。',
  content: '<p>文章正文 HTML</p>'
}
```

### 5.2 当前示例文章

| ID | 标题 | 日期 | 标签 |
|---|---|---|---|
| `hello-blog` | 我的第一篇博客 | 2026-04-27 | 随笔、个人博客 |
| `why-write` | 为什么要写博客 | 2026-04-28 | 写作、成长 |
| `keep-simple` | 保持简单 | 2026-04-29 | 生活、思考 |

## 6. 交互规则

### 6.1 导航交互

- 顶部导航在所有页面保持一致。
- 当前页面导航项使用胶囊背景和主色文字高亮。
- 博客名称始终链接到首页。

### 6.2 页面切换

- 使用 hash 路由监听 `hashchange`。
- 切换页面后滚动到顶部。
- 切换页面后将焦点移动到 `main`，提升键盘和屏幕阅读器体验。

### 6.3 卡片交互

- 文章卡片 hover 时轻微上移 2px。
- 卡片阴影和边框增强，但不改变内容布局。
- 标题和“阅读全文”均可点击。

### 6.4 按钮交互

- 主按钮 hover 时颜色加深。
- active 时轻微缩放。
- 所有按钮最小高度 44px。

## 7. 可访问性说明

原型已包含以下可访问性设计：

1. `html lang="zh-CN"`。
2. `meta viewport` 支持移动端布局，不禁用缩放。
3. 使用 `header`、`nav`、`main`、`article`、`footer` 语义标签。
4. 提供“跳到主内容”链接。
5. 使用 `:focus-visible` 提供清晰键盘焦点。
6. 交互元素触达高度不小于 44px。
7. 导航包含 `aria-label`。
8. 装饰性元素使用 `aria-hidden="true"`。
9. 支持 `prefers-reduced-motion`。
10. 页面标题层级清晰。

## 8. 响应式说明

### 8.1 桌面端

- 首页 Hero 使用双栏布局。
- 精选文章使用三列卡片。
- 文章详情正文控制在 760px 宽度内。

### 8.2 平板端

- Hero 切换为单列或更紧凑布局。
- 文章卡片保持可读宽度。

### 8.3 手机端

- 顶部导航上下排列。
- 文章卡片单列展示。
- 内容容器左右边距缩小到 16px。
- 无横向滚动。

---

## 9. 视觉风格说明

### 9.1 色彩

- 背景：温暖米白色。
- 主文本：深灰蓝色。
- 主色：琥珀棕色，用于 CTA 和链接。
- 辅助色：低饱和青绿色，用于标签。

### 9.2 排版

- 系统无衬线字体，保证无需网络加载即可显示。
- 首页标题使用大字号和紧凑字距，形成清晰视觉焦点。
- 正文使用较大行高，保障阅读舒适度。

### 9.3 组件

- 卡片使用白色半透明表面、柔和边框和轻阴影。
- 标签采用圆角胶囊样式。
- 按钮采用圆角 pill 风格。
