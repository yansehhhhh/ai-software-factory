## 1. 项目背景

### 1.1 项目概述
本系统是一个面向公开投票活动的投票管理平台，支持比赛评选、民意调查等场景，为组织者提供便捷的投票创建和管理能力，为参与者提供安全、公正的投票体验。

### 1.2 业务目标
- 提供一个安全、公正的公开投票平台
- 防止刷票行为，确保投票结果的真实性
- 简化投票活动组织流程
- 提高投票效率和参与度

### 1.3 目标用户
- **管理员**: 创建和管理投票活动的组织者
- **投票用户**: 参与投票活动的普通用户

## 2. 需求确认

### 2.1 需求讨论记录

| 问题 | 用户选择 | 说明 |
|------|---------|------|
| 主要使用场景 | 公开投票活动（如比赛评选、民意调查） | 面向公众的投票场景 |
| 投票方式 | 单选投票（每个用户只能选一个选项） | 简化投票逻辑，降低复杂度 |
| 身份验证方式 | 手机号验证码登录 | 防刷票，确保一人一票 |
| 结果展示方式 | 投票结束后才公开结果 | 避免投票过程中的从众效应 |
| 创建权限 | 仅管理员后台创建 | 控制投票活动的发布权限 |

### 2.2 已确认需求总结
- 面向公开投票活动场景
- 仅支持单选投票方式
- 使用手机号验证码进行身份验证
- 投票结果在活动结束后公开
- 投票活动创建权限仅限管理员

## 3. 核心功能需求

### 3.1 用户管理模块

#### 3.1.1 用户注册登录
- **功能描述**: 用户通过手机号注册和登录系统
- **优先级**: P0
- **需求详情**:
  - 支持手机号注册
  - 发送短信验证码（6位数字）
  - 验证码有效期：5分钟
  - 同一手机号在投票活动中只能投票一次

#### 3.1.2 管理员权限
- **功能描述**: 管理员登录后台管理系统
- **优先级**: P0
- **需求详情**:
  - 管理员账号由系统预设
  - 支持管理员登录/退出
  - 管理员可创建、编辑、删除投票活动

### 3.2 投票活动管理模块

#### 3.2.1 创建投票活动
- **功能描述**: 管理员创建新的投票活动
- **优先级**: P0
- **需求详情**:
  - 设置活动标题和描述
  - 上传活动封面图
  - 添加投票选项（支持文字、图片）
  - 设置投票时间范围（开始时间、结束时间）
  - 设置投票规则说明
  - 活动状态：草稿、未开始、进行中、已结束

#### 3.2.2 编辑投票活动
- **功能描述**: 管理员编辑投票活动信息
- **优先级**: P0
- **需求详情**:
  - 仅草稿状态的活动可编辑
  - 进行中的活动不可修改投票选项
  - 可延长活动结束时间

#### 3.2.3 删除投票活动
- **功能描述**: 管理员删除投票活动
- **优先级**: P1
- **需求详情**:
  - 仅草稿状态的活动可删除
  - 删除需二次确认
  - 删除后不可恢复

#### 3.2.4 活动列表管理
- **功能描述**: 管理员查看所有投票活动
- **优先级**: P0
- **需求详情**:
  - 展示活动列表（标题、状态、时间、参与人数）
  - 按状态筛选活动
  - 按时间排序
  - 搜索活动

### 3.3 投票执行模块

#### 3.3.1 用户投票
- **功能描述**: 用户参与投票活动
- **优先级**: P0
- **需求详情**:
  - 用户需登录后才可投票
  - 每个用户在单个活动中只能投一票
  - 选择选项后点击确认投票
  - 投票成功后显示提示信息
  - 投票后不可更改

#### 3.3.2 投票记录
- **功能描述**: 记录用户投票行为
- **优先级**: P0
- **需求详情**:
  - 记录投票用户ID
  - 记录投票时间
  - 记录投票选项
  - 投票数据不可篡改

### 3.4 结果展示模块

#### 3.4.1 投票结果统计
- **功能描述**: 系统统计投票结果
- **优先级**: P0
- **需求详情**:
  - 实时统计各选项票数
  - 活动结束后计算排名
  - 支持导出投票数据（管理员）

#### 3.4.2 结果公开展示
- **功能描述**: 向用户展示投票结果
- **优先级**: P0
- **需求详情**:
  - 活动进行中不显示票数
  - 活动结束后显示各选项票数和占比
  - 显示排名（第一名、第二名等）
  - 可视化展示（柱状图或饼图）

## 4. 非功能需求

### 4.1 性能需求
- 页面加载时间 < 2秒
- 支持并发投票用户数 ≥ 1000人
- 投票响应时间 < 500ms
- 系统可用性 ≥ 99.9%

### 4.2 安全需求
- 手机号验证码防刷机制
- 同一手机号在同一活动中只能投票一次
- 防止SQL注入、XSS攻击
- 数据传输加密（HTTPS）
- 敏感信息加密存储

### 4.3 数据需求
- 投票数据实时备份
- 数据保留周期：活动结束后保留1年
- 支持数据导出（CSV格式）

### 4.4 兼容性需求
- 支持主流浏览器（Chrome、Firefox、Safari、Edge）
- 移动端自适应（响应式设计）
- 支持微信内置浏览器

## 6. 约束条件

### 6.1 技术约束
- 后端使用 Spring Boot 框架
- 前端使用 Vue 3 框架
- 需要短信验证码服务
- 需要图片存储服务

### 6.2 业务约束
- 首期仅支持单选投票
- 不支持匿名投票
- 投票结果仅在活动结束后公开
