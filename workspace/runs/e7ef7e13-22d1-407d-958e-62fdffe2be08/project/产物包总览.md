## 核心业务流程

### 用户投票流程
```
浏览活动列表 → 点击活动 → 登录（手机号+验证码） →
选择投票选项 → 确认投票 → 投票成功 →
活动结束后查看结果
```

### 管理员创建活动流程
```
登录后台 → 创建活动 → 填写基本信息 →
设置时间范围 → 添加投票选项 →
保存为草稿 → 发布活动
```

## 关键设计要点

### 安全设计
- 手机号验证码防刷机制
- 一人一票限制
- 验证码有效期控制（5分钟）
- IP频率限制（每分钟最多10次请求）

### 防刷票机制
- 同一手机号只能投票一次
- 验证码时效性验证
- IP频率限制
- 数据加密传输

### 结果展示策略
- 活动进行中：不显示票数，避免从众效应
- 活动结束后：公开完整结果，包括票数、占比、排名、可视化图表
