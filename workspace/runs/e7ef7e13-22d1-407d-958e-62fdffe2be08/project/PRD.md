## 1. 产品概述

### 1.1 产品定位
公开投票管理系统是一个面向公众投票活动的在线平台，旨在为比赛评选、民意调查等场景提供安全、公正、便捷的投票服务。系统通过手机号验证码确保投票真实性，采用投票结束后公开结果的机制避免投票过程中的从众效应。

### 1.2 核心价值
- **公正性**: 一人一票机制，防止刷票行为
- **透明性**: 投票结果活动结束后公开，过程公正透明
- **便捷性**: 用户无需注册复杂账号，手机号即可参与
- **安全性**: 身份验证机制确保投票真实有效

### 1.3 产品目标
- MVP 版本上线后支持至少 10 个并发投票活动
- 单个活动支持最多 1000 人参与投票
- 用户投票成功率 ≥ 98%
- 投票争议投诉率 < 1%

## 2. 目标用户分析

### 2.1 管理员用户画像

**基本信息**
- 角色: 活动组织者、管理员
- 年龄: 25-45岁
- 技术水平: 中等（能熟练使用后台管理系统）

**核心需求**
- 快速创建投票活动
- 管理多个投票活动
- 查看投票参与情况
- 导出投票结果数据

**痛点**
- 传统投票方式统计困难
- 线下投票参与度低
- 容易出现刷票作弊
- 结果统计耗时耗力

### 2.2 投票用户画像

**基本信息**
- 角色: 活动参与者、投票者
- 年龄: 18-60岁
- 技术水平: 低-中（能使用手机浏览网页）

**核心需求**
- 快速找到要参与的投票活动
- 简单便捷的投票流程
- 查看投票结果
- 投票过程公平透明

**痛点**
- 投票流程复杂繁琐
- 不确定是否投票成功
- 无法看到投票结果
- 担心投票被泄露

## 4. 功能详细设计

### 4.1 用户管理功能

#### 4.1.1 用户注册登录流程

**流程描述**
1. 用户输入手机号
2. 点击"获取验证码"
3. 系统发送短信验证码（6位数字）
4. 用户输入验证码
5. 验证成功后登录系统

**业务规则**
- 手机号格式校验：11位数字，以1开头
- 验证码有效期：5分钟
- 同一手机号验证码发送频率限制：1分钟内最多发送1次
- 验证码错误次数限制：最多3次，超过需重新获取
- 登录态保持时间：7天

**异常处理**
- 手机号已存在：提示"该手机号已登录，可直接投票"
- 验证码错误：提示"验证码错误，请重新输入"
- 验证码过期：提示"验证码已过期，请重新获取"
- 发送失败：提示"验证码发送失败，请稍后重试"

#### 4.1.2 管理员登录

**登录方式**
- 管理员账号密码登录
- 账号由系统预设，不可自行注册
- 支持记住密码功能

**权限管理**
- 管理员可创建、编辑、删除活动
- 管理员可查看所有投票数据
- 管理员可导出投票结果

### 4.2 活动管理功能

#### 4.2.1 创建投票活动

**必填字段**
- 活动标题（≤50字符）
- 活动描述（≤500字符）
- 活动封面图（≤2MB，支持jpg/png）
- 开始时间（YYYY-MM-DD HH:mm）
- 结束时间（YYYY-MM-DD HH:mm）
- 投票选项（至少2个，最多20个）

**投票选项字段**
- 选项名称（≤30字符）
- 选项描述（≤100字符）
- 选项图片（≤2MB，可选）

**活动状态**
- 草稿：创建中，未发布
- 未开始：已发布，但未到开始时间
- 进行中：活动进行中，可投票
- 已结束：活动已结束，可查看结果

**业务规则**
- 结束时间必须大于开始时间
- 活动时长最少1小时，最多30天
- 草稿状态可编辑，其他状态不可编辑基本信息
- 进行中状态不可修改投票选项
- 草稿状态可删除，其他状态不可删除

#### 4.2.2 编辑投票活动

**可编辑内容**
- 草稿状态：所有内容均可编辑
- 未开始状态：仅可修改开始结束时间
- 进行中状态：仅可延长结束时间
- 已结束状态：不可编辑

#### 4.2.3 活动列表管理

**列表展示字段**
- 活动标题
- 活动状态（草稿/未开始/进行中/已结束）
- 开始时间
- 结束时间
- 参与人数
- 创建时间

**筛选和排序**
- 按状态筛选：全部、草稿、未开始、进行中、已结束
- 按时间排序：创建时间降序
- 搜索：按活动标题模糊搜索

### 4.3 投票执行功能

#### 4.3.1 用户投票流程

**流程描述**
1. 用户浏览活动列表
2. 点击进入活动详情页
3. 查看活动介绍和投票选项
4. 选择一个投票选项
5. 点击"确认投票"
6. 系统验证用户身份
7. 记录投票结果
8. 显示投票成功提示

**业务规则**
- 用户必须登录才能投票
- 同一用户在同一个活动中只能投票一次
- 投票后不可更改
- 活动进行中才能投票
- 活动未开始或已结束不可投票

**异常处理**
- 未登录：提示"请先登录后再投票"，跳转登录页
- 已投票：提示"您已参与过投票，不能重复投票"
- 活动未开始：提示"活动未开始，请等待开始时间"
- 活动已结束：提示"活动已结束，感谢您的关注"

#### 4.3.2 防刷票机制

**验证机制**
- 手机号唯一性验证：同一手机号只能投票一次
- 验证码时效性：验证码5分钟内有效
- IP频率限制：同一IP每分钟最多发起10次投票请求

### 4.4 结果展示功能

#### 4.4.1 投票结果统计

**统计维度**
- 各选项票数
- 各选项占比（百分比）
- 参与总人数
- 投票时间分布

**数据可视化**
- 柱状图：展示各选项票数对比
- 饼图：展示各选项占比
- 时间趋势图：展示投票时间分布（可选）

#### 4.4.2 结果公开展示

**展示时机**
- 活动进行中：不显示票数和排名
- 活动结束后：显示完整投票结果

**展示内容**
- 各选项名称、票数、占比
- 排名（第一名、第二名等）
- 参与总人数
- 可视化图表

## 9. UI 设计规范

### 9.1 设计原则
- 简洁明了：减少用户认知负担
- 重点突出：投票操作清晰可见
- 一致性：保持整体风格统一
- 易用性：操作流程简单直观

### 9.2 色彩规范

**主色调**
- 主色：#3B82F6（蓝色）
- 辅助色：#10B981（绿色，成功）
- 警告色：#F59E0B（橙色）
- 错误色：#EF4444（红色）

**背景色**
- 主背景：#FFFFFF
- 次背景：#F3F4F6
- 深色背景：#1F2937

**文字色**
- 主文字：#1F2937
- 次文字：#6B7280
- 提示文字：#9CA3AF

### 9.3 字体规范

**字体**
- 主字体：PingFang SC, Microsoft YaHei, sans-serif
- 数字字体：Helvetica Neue

**字号**
- 大标题：24px
- 标题：18px
- 正文：14px
- 小字：12px

### 9.4 间距规范
- 页面边距：16px
- 卡片间距：12px
- 内容间距：8px
- 文字间距：4px

### 9.5 响应式设计

**断点**
- 移动端：< 768px
- 平板：768px - 1024px
- PC端：> 1024px

**布局**
- 移动端：单列布局
- 平板：双列布局
- PC端：三列布局或单列居中
