## 2. 页面结构

原型仅包含一个页面：待办事项首页。

### 2.1 页面区域

从上到下依次为：

1. 跳转链接：键盘用户可跳到主要内容。
2. 页面头部：展示产品类型、页面标题和说明文案。
3. 统计区：展示全部任务、未完成任务、已完成任务。
4. 新增待办表单：输入待办内容并添加任务。
5. 待办列表：展示所有任务。
6. 空状态：无任务时展示。
7. Toast 提示：删除或异常状态的轻反馈。

## 3. 页面内容

### 3.1 Header

- 产品标签：个人效率工具
- 页面标题：今天要完成什么？
- 页面说明：记录任务，逐项完成，保持专注。这个静态原型支持新增、完成、删除和本地保存。

### 3.2 统计区

展示 3 个统计项：

| 统计项 | 说明 |
|---|---|
| 全部任务 | 当前列表中的任务总数 |
| 未完成 | `completed=false` 的任务数量 |
| 已完成 | `completed=true` 的任务数量 |

### 3.3 新增待办区

组成：

- 可见标签：新增待办
- 输入框 placeholder：例如：购买牛奶
- 主按钮：添加任务
- 帮助文案：按 Enter 或点击按钮添加，最多 100 个字符。
- 错误文案区域：用于展示输入校验错误。

### 3.4 待办列表

每条任务包含：

- 复选框：切换完成状态。
- 任务标题：展示待办内容。
- 状态文案：未完成 / 已完成。
- 删除按钮：删除当前任务。

### 3.5 空状态

当任务数量为 0 时显示：

- 标题：暂无待办
- 描述：添加你的第一个任务，开始保持专注。
- 线性 SVG 图形：作为轻量视觉提示，不使用 emoji。

## 4. 交互说明

### 4.1 新增任务

触发方式：

- 输入内容后点击“添加任务”。
- 输入内容后按 Enter。

处理规则：

1. 对输入内容执行 `trim()`。
2. 若为空，阻止提交并显示“请输入待办内容。”。
3. 若超过 100 个字符，阻止提交并显示“任务内容不能超过 100 个字符。”。
4. 校验通过后，将任务插入列表顶部。
5. 输入框清空。
6. 焦点回到输入框。
7. 统计区同步更新。
8. 数据写入 `localStorage`。

### 4.2 完成任务

触发方式：点击任务左侧复选框。

效果：

- 未完成变为已完成。
- 已完成变为未完成。
- 已完成状态显示：
  - 复选框选中。
  - 标题删除线。
  - 标题颜色降低强调。
  - 状态文案变为“已完成”。
- 统计区同步更新。
- 状态写入 `localStorage`。

### 4.3 删除任务

触发方式：点击任务右侧“删除”按钮。

效果：

- 当前任务从列表移除。
- 统计区同步更新。
- 若删除后无任务，展示空状态。
- Toast 显示“任务已删除。”，约 3 秒后消失。
- 数据写入 `localStorage`。

### 4.4 本地持久化

原型使用浏览器 `localStorage` 保存任务。

- key：`simple-todo-prototype-items`
- 首次打开时会写入 3 条演示任务。
- 后续刷新页面保留用户操作后的任务状态。
- 如果本地数据损坏，原型会降级为空列表并显示提示。

如需重置原型，可在浏览器控制台执行：

```js
localStorage.removeItem('simple-todo-prototype-items')
location.reload()
```

## 5. 状态说明

### 5.1 默认状态

首次打开时展示演示任务：

1. 添加第一个待办事项：已完成
2. 点击复选框完成任务：未完成
3. 删除不需要的任务：未完成

### 5.2 空状态

当任务列表为空时展示空状态卡片。

进入方式：

- 删除所有任务。
- 清空 localStorage 后修改逻辑为空列表。
- localStorage 损坏且降级为空。

### 5.3 错误状态

输入错误展示在输入框下方。

错误类型：

| 错误 | 文案 |
|---|---|
| 空输入 | 请输入待办内容。 |
| 纯空格 | 请输入待办内容。 |
| 超过 100 字符 | 任务内容不能超过 100 个字符。 |

### 5.4 完成状态

完成状态不只依赖颜色，还通过复选框选中、删除线和状态文案共同表达。

### 5.5 Toast 状态

用于非阻断反馈：

- 删除成功。
- 本地保存不可用。
- 历史数据读取失败。

## 6. 响应式说明

### 6.1 桌面端

- 页面居中，最大宽度约 768px。
- 主卡片有圆角和阴影。
- 表单输入框和按钮横向排列。

### 6.2 移动端

在 `520px` 以下：

- 表单输入框和添加按钮改为纵向排列。
- 添加按钮宽度为 100%。
- 待办项删除按钮换到标题下方，避免横向拥挤。
- 保持所有点击区域不小于 44px。

### 6.3 小屏验收

建议检查：

- 375px 宽度无横向滚动。
- 任务长标题可换行。
- 删除按钮可点击。
- 输入框字号不小于 16px。

## 7. 可访问性说明

原型包含以下可访问性设计：

- `lang="zh-CN"`。
- `meta viewport` 未禁用缩放。
- 页面有跳转到主要内容的 skip link。
- 页面有唯一 `h1`。
- 输入框有可见 `label`。
- 输入框通过 `aria-describedby` 关联帮助和错误信息。
- 错误信息使用 `role="alert"`。
- Toast 使用 `aria-live="polite"`。
- 复选框有描述性 `aria-label`。
- 删除按钮有描述性 `aria-label`。
- 键盘焦点环清晰可见。
- 支持 `prefers-reduced-motion: reduce`。
