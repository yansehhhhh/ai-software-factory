# 简单待办事项应用 UI 设计规范

## 1. 设计目标

简单待办事项应用面向个人用户，用于快速记录、查看、完成和删除任务。第一版 UI 设计应强调：

- **轻量高效**：用户打开页面后能立即输入任务。
- **清晰可读**：待办状态、统计信息和操作入口一眼可见。
- **低干扰**：避免复杂视觉装饰，突出任务内容本身。
- **移动优先**：在 375px 手机宽度下仍可舒适使用。
- **可访问性优先**：保证键盘操作、屏幕阅读器语义、颜色对比和触控尺寸。

## 2. 产品定位与设计风格

### 产品类型

- 类型：个人效率工具 / Task Manager
- 使用场景：工作、学习、生活中的即时任务记录
- 使用频率：高频、短时、多次访问

### 推荐风格

采用 **Warm Minimal Productivity / 温和极简效率风**：

- 大面积浅色背景，降低认知负担。
- 使用柔和蓝色作为主色，传达可靠、专注和秩序感。
- 使用绿色表达完成状态，但不能只依赖颜色，需配合勾选状态和删除线。
- 使用圆角卡片和轻阴影提升亲和力。
- 不使用表情符号作为结构性图标。

## 3. 信息架构

应用第一版仅包含一个页面：待办首页。

页面从上到下的信息层级：

1. 应用标题与说明
2. 今日任务概览统计
3. 新增待办输入区
4. 待办列表
5. 空状态 / 错误提示 / 操作反馈

## 4. 页面布局规范

### 桌面端布局

- 页面背景：浅灰蓝渐变或浅色背景。
- 主容器最大宽度：`720px`。
- 主容器居中显示。
- 顶部留白：`48px`。
- 页面左右安全边距：`24px`。
- 卡片内边距：`24px`。

### 移动端布局

- 设计基准宽度：`375px`。
- 页面左右边距：`16px`。
- 顶部留白：`24px`。
- 表单输入与按钮在窄屏下可纵向排列。
- 所有可点击目标高度不小于 `44px`。
- 禁止横向滚动。

### 断点建议

| 断点 | 用途 |
|---|---|
| `0-479px` | 小屏手机 |
| `480-767px` | 大屏手机 |
| `768-1023px` | 平板 |
| `1024px+` | 桌面端 |

## 5. 色彩系统

### 语义色板

| Token | 色值 | 用途 |
|---|---:|---|
| `--color-bg` | `#F6F8FC` | 页面背景 |
| `--color-surface` | `#FFFFFF` | 主卡片、列表项 |
| `--color-surface-muted` | `#EEF4FF` | 统计卡片浅背景 |
| `--color-primary` | `#2563EB` | 主按钮、焦点、强调信息 |
| `--color-primary-hover` | `#1D4ED8` | 主按钮悬停 |
| `--color-primary-soft` | `#DBEAFE` | 主色浅背景 |
| `--color-success` | `#16A34A` | 完成状态 |
| `--color-success-soft` | `#DCFCE7` | 完成状态浅背景 |
| `--color-danger` | `#DC2626` | 删除、错误 |
| `--color-danger-soft` | `#FEE2E2` | 错误提示背景 |
| `--color-text` | `#0F172A` | 主文本 |
| `--color-text-muted` | `#64748B` | 次级文本 |
| `--color-border` | `#E2E8F0` | 边框、分割线 |
| `--color-focus` | `#93C5FD` | 键盘焦点环 |

### 色彩使用规则

- 主操作只使用 `primary`，每个页面仅保留一个主要 CTA：添加任务。
- 完成任务必须同时使用：复选框选中、文本删除线、文本变浅，不能只靠绿色表达。
- 删除操作使用危险色，但默认保持低强调，仅在 hover/focus 时增强。
- 正文与背景对比度需达到 WCAG AA：普通文本至少 4.5:1。

## 9. 组件规范

### 9.1 主按钮

用途：添加待办事项。

规范：

- 高度：`48px`。
- 最小宽度：`96px`。
- 圆角：`12px`。
- 背景：`primary`。
- 文字：白色，`16px / 650`。
- hover：使用 `primary-hover`。
- focus：显示 3px 焦点环。
- disabled：透明度降低，不可点击。

### 9.2 文本输入框

用途：输入待办标题。

规范：

- 高度：`48px`。
- 字号：`16px`，避免移动端 iOS 自动缩放。
- 边框：`1px solid border`。
- 圆角：`12px`。
- focus：边框变为主色，显示焦点环。
- 错误状态：边框危险色，错误文案显示在输入区下方。
- 必须有可访问标签，不能仅依赖 placeholder。

### 9.3 任务列表项

每条任务包含：

- 完成复选框
- 任务标题
- 创建或状态辅助信息
- 删除按钮

规范：

- 最小高度：`56px`。
- 内边距：`14px 16px`。
- 列表项之间间距：`12px`。
- 已完成状态：
  - 复选框选中。
  - 标题增加删除线。
  - 标题颜色使用次级文本。
  - 列表项背景可轻微变淡。

### 9.4 删除按钮

规范：

- 可点击区域不小于 `44px × 44px`。
- 使用文字“删除”或 SVG 图标 + `aria-label="删除任务"`。
- 默认低强调，hover/focus 使用危险色浅背景。
- 不在 MVP 中强制弹确认框，删除后可显示轻量提示。

### 9.5 统计卡片

展示：总数、未完成、已完成。

规范：

- 使用三列布局，移动端保持三列但缩小间距；极窄情况下可纵向堆叠。
- 数字使用 `24px / 700`。
- 标签使用 `12px / 500`。
- 不能只通过颜色区分统计类型，应有文字标签。

### 9.6 空状态

无任务时展示。

内容建议：

- 标题：暂无待办
- 描述：添加你的第一个任务，开始保持专注。
- 行动：聚焦输入框或提示用户在上方输入。

规范：

- 使用浅色虚线框或轻背景。
- 不使用 emoji 作为图标。
- 可使用简单 SVG 线性图形。

## 10. 交互规范

### 新增任务

- 用户输入有效内容后点击“添加任务”或按 Enter 创建。
- 创建成功后：
  - 新任务加入列表。
  - 输入框清空。
  - 错误提示清除。
  - 统计更新。

### 输入校验

- 空字符串不可提交。
- 纯空格不可提交。
- 提交时自动裁剪前后空格。
- 超过 100 字符时提示：“任务内容不能超过 100 个字符”。
- 错误文案展示在输入框下方，并使用 `role="alert"`。

### 完成 / 取消完成

- 点击复选框切换状态。
- 状态切换动画控制在 `150-200ms`。
- 已完成任务不从列表中隐藏。

### 删除任务

- 点击删除按钮后移除任务。
- 显示 3 秒轻提示：“任务已删除”。
- MVP 不要求撤销。

## 11. 动效规范

- 微交互持续时间：`150-250ms`。
- 缓动：`cubic-bezier(0.2, 0, 0, 1)`。
- 仅使用 `opacity` 和 `transform` 实现动效，避免触发布局抖动。
- 尊重 `prefers-reduced-motion: reduce`，关闭非必要动画。

## 12. 可访问性规范

- 页面必须有唯一 `h1`。
- 输入框必须有 `label`。
- 图标按钮必须提供 `aria-label`。
- 错误信息使用 `role="alert"` 或 `aria-live="polite"`。
- Toast 使用 `aria-live="polite"`，不可强行抢焦点。
- 所有交互元素支持键盘 Tab 聚焦。
- 焦点样式不可移除。
- 触控目标不小于 `44px × 44px`。
- 颜色不是唯一状态表达方式。

## 13. 响应式规范

- 主容器宽度使用 `width: min(100%, 720px)`。
- 页面内边距在移动端为 `16px`，桌面端为 `24px`。
- 表单在 `520px` 以下变为纵向布局。
- 列表项标题允许换行，避免溢出。
- 禁止固定宽度导致横向滚动。

## 14. 空、错误与加载状态

### 空状态

- 无任务时必须展示。
- 文案应说明当前状态和下一步行动。

### 错误状态

- 输入错误在字段下方展示。
- 本地存储不可用时展示非阻断提示。

### 加载状态

- 本 MVP 为本地应用，一般无需全屏加载。
- 若未来接入接口，请使用骨架屏或按钮 loading 状态，不使用长时间全屏 spinner。

## 15. 设计验收清单

- [ ] 页面在 375px 宽度下无横向滚动。
- [ ] 所有按钮和复选框点击区域不小于 44px。
- [ ] 输入框字号不小于 16px。
- [ ] 页面有明确标题、输入区、统计区、列表区。
- [ ] 空状态、错误状态、成功反馈完整。
- [ ] 任务完成状态不只依赖颜色。
- [ ] 键盘可完成新增、切换、删除操作。
- [ ] 焦点环清晰可见。
- [ ] 正文对比度达到 WCAG AA。
- [ ] 未使用 emoji 作为结构图标。
