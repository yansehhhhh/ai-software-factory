## 1. 产品概述

### 1.1 产品定义

一个基于 **VitePress** 构建的个人博客静态网站，面向单作者场景。作者以本地 Markdown 文件编写文章，通过 Git 推送触发自动构建部署到 GitHub Pages。读者通过浏览器访问，浏览文章列表、阅读文章详情、按标签筛选感兴趣的内容。

### 1.2 产品目标

| 目标 | 衡量指标 |
|------|----------|
| 作者发布效率 | 从写完 Markdown 到线上可见 ≤ 3 分钟（含 CI 构建） |
| 阅读体验 | 首屏加载时间 ≤ 2s（3G 网络），Lighthouse Performance ≥ 90 |
| 内容可发现性 | 支持标签筛选、文章列表浏览 |
| 多端适配 | 桌面端、平板、手机均可正常阅读 |

### 1.3 产品范围

```
┌──────────────────────────────────────────┐
│              博客网站 v1.0                │
├──────────────────────────────────────────┤
│  首页        文章列表      文章详情       │
│  (文章卡片)  (按标签筛选)  (Markdown渲染) │
│                                          │
│  标签页      关于页        全局导航       │
│  (标签聚合)  (作者信息)    (顶部+底部)     │
└──────────────────────────────────────────┘

不包含：评论、搜索、用户系统、后台管理、RSS
```

## 2. 用户角色与场景

### 2.1 角色画像

#### 角色 A：博客作者 (Author)

| 属性 | 描述 |
|------|------|
| 身份 | 独立开发者 / 技术写作者 |
| 技能 | 熟悉 Git、Markdown、命令行操作 |
| 目标 | 高效发布技术文章，无需维护服务器 |
| 痛点 | 不想使用 WordPress 等需运维的后台系统 |
| 使用频率 | 每周 1-3 篇文章 |

#### 角色 B：读者 (Reader)

| 属性 | 描述 |
|------|------|
| 身份 | 开发者、技术爱好者、偶然访客 |
| 设备 | 桌面端为主，部分移动端阅读 |
| 目标 | 快速找到感兴趣的技术文章并阅读 |
| 痛点 | 内容混乱的博客难以筛选，加载慢的页面难以忍受 |
| 使用频率 | 不定，通过搜索引擎/社交链接进入 |

### 2.2 核心场景

#### 场景 1：作者发布新文章

```
1. 作者在本地编辑器编写 post-2026-04-27-new-feature.md
2. 填写 frontmatter（title, date, tags, summary）
3. git add . && git commit -m "post: new feature article"
4. git push origin main
5. GitHub Actions 自动构建 VitePress 站点
6. 约 60 秒后，文章在 blog.example.com 上线
7. 作者在浏览器验证文章渲染效果
```

#### 场景 2：读者浏览博客

```
1. 读者通过搜索引擎点击进入文章详情页
2. 阅读文章内容，注意到底部标签
3. 点击感兴趣的标签（如 "Vue"），进入标签筛选页
4. 浏览该标签下的其他文章
5. 点击首页导航，查看最新文章列表
6. 有兴趣则收藏站点，后续定期访问
```

#### 场景 3：移动端阅读

```
1. 读者在手机上打开文章链接
2. 页面自适应移动屏幕宽度
3. 顶部导航折叠为汉堡菜单
4. 文章内容字体大小适合移动阅读
5. 流畅滚动，无横向溢出
```

## 5. 数据模型

### 5.1 文章 Frontmatter

```yaml
---
title: 文章标题（必填）
date: 2026-04-27（必填，YYYY-MM-DD）
tags:
  - Vue
  - TypeScript
  - 前端（可选，建议 1-5 个）
summary: 文章摘要，用于列表展示（可选，不填则截取正文前 150 字）
author: 作者名（可选，默认使用站点配置的作者）
draft: false（可选，true 时构建跳过）
---
```

### 5.2 站点配置

```typescript
// .vitepress/config.ts 关键配置项
interface SiteConfig {
  title: string;           // 博客名称
  description: string;     // 站点描述
  author: {
    name: string;          // 作者名称
    avatar: string;        // 头像路径
    bio: string;           // 一句话简介
    links: {               // 社交链接
      github?: string;
      twitter?: string;
      email?: string;
    };
  };
  nav: NavItem[];          // 导航配置
  postsPerPage: number;    // 每页文章数（默认 10）
}
```

### 5.3 文章对象（运行时）

```typescript
interface Post {
  slug: string;            // URL 标识，如 "hello-world"
  title: string;           // 文章标题
  date: string;            // 发布日期 YYYY-MM-DD
  tags: string[];          // 标签数组
  summary: string;         // 摘要
  content: string;         // Markdown 原始内容
  readingTime: number;     // 阅读时长（分钟）
  prev: Post | null;       // 上一篇文章
  next: Post | null;       // 下一篇文章
}
```

## 6. 交互规范

| 元素 | 悬停态 | 点击态 |

| 断点 | 宽度 | 布局变化 |

| 场景 | 展示 |
|------|------|

| 标签筛选无结果 | "该标签下暂无文章" |
| 搜索无结果 | 不适用（v1 无搜索） |

{

```

### 7.3 GitHub Actions 工作流（示意）

```yaml
name: Deploy Blog
on:
  push:
    branches: [main]
jobs:
  build-and-deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
      - run: npm ci
      - run: npm run docs:build
      - uses: peaceiris/actions-gh-pages@v3
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: .vitepress/dist
```
