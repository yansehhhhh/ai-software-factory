## 1. 项目背景

### 1.1 项目目标

构建一个个人博客网站。博客主（单作者）通过本地编写 Markdown 文件管理内容，经 Git 推送后自动更新站点，部署于 GitHub Pages。

### 1.2 核心价值

- **低摩擦写作**：作者用熟悉的 Markdown 编辑器 + Git 工作流管理内容
- **零运维成本**：静态站点托管于 GitHub Pages，无服务器维护负担
- **快速访问**：纯静态页面，加载速度快，SEO 友好

## 2. 需求概览

### 2.1 已确认需求总结

| # | 需求项 | 确认结论 | 来源 |
|---|--------|----------|------|
| R1 | 博客类型 | 个人博客，单作者，以文章展示为主 | Q1 回答 |
| R2 | 内容管理方式 | 本地编写 Markdown 文件，通过 Git 推送到仓库 | Q2 回答 |
| R3 | 技术栈 | 推荐 VitePress 静态站点（契合 GitHub Pages + Markdown + Git 工作流） | Q3 回答 |
| R4 | 前端功能范围 | 文章列表 + 文章详情 + 标签/分类 | Q4 回答 |
| R5 | 部署目标 | GitHub Pages | Q5 回答 |

### 2.2 功能边界

**包含（In Scope）**：
- 文章列表页（分页或无限滚动）
- 文章详情页（Markdown 渲染）
- 标签/分类体系（侧边栏或导航筛选）
- 博客首页（含个人简介区域 + 文章列表）
- 响应式布局（适配桌面端和移动端）

**不包含（Out of Scope）**：
- 评论功能
- 全文搜索
- 用户注册/登录
- 多作者支持
- 后台管理界面
- RSS 订阅（可后续迭代）

## 3. 用户角色

### 3.1 角色定义

| 角色 | 描述 |
|------|------|
| 博客作者（Author） | 唯一内容生产者，通过 Git 仓库管理 Markdown 文章 |
| 读者（Reader） | 普通访客，浏览文章列表和详情，按标签/分类筛选 |

### 3.2 作者工作流

```
编写 Markdown → git add → git commit → git push → GitHub Actions 构建 → GitHub Pages 部署
```

作者无需登录任何后台，一切操作在本地完成。

## 6. 技术方案推荐

### 6.1 推荐方案：VitePress

| 维度 | 说明 |
|------|------|
| 静态站点生成 | 构建时生成纯静态 HTML，部署 GitHub Pages 零成本 |
| Markdown 原生 | 直接读取 `.md` 文件，支持 frontmatter 元数据 |
| Vue 3 驱动 | 可嵌入 Vue 组件增强交互，与本仓库技术栈一致 |
| 主题定制 | 提供默认博客主题，支持自定义布局 |
| 构建速度 | Vite 驱动，秒级热更新，全站构建快 |
| 社区生态 | Vue 官方维护，文档完善 |

### 6.2 备选方案

| 方案 | 优势 | 劣势 |
|------|------|------|
| Astro | 多框架组件、极致性能 | 额外学习成本 |
| VuePress | 功能类似 VitePress | 构建较慢，已逐步被 VitePress 取代 |
| Hexo | Node.js 生态、主题丰富 | 非 Vue 技术栈 |

### 6.3 项目结构（推荐）

```
blog/
├── .vitepress/
│   └── config.ts          # 站点配置、导航、侧边栏
├── posts/                  # Markdown 文章目录
│   ├── 2026-04-27-hello-world.md
│   └── 2026-04-28-second-post.md
├── public/                 # 静态资源（头像、favicon）
│   └── avatar.png
├── index.md               # 首页
├── tags.md                # 标签聚合页
├── about.md               # 关于页
└── package.json
```
