## 1. 产品概述

### 1.1 产品定位

一个面向多用户的博客发布与阅读平台。用户注册后可撰写纯文字文章、按分类组织内容、参与文章评论互动。管理员负责平台内容与用户的整体管理。

### 1.2 目标用户

| 用户类型 | 描述 | 核心诉求 |
|----------|------|----------|
| 内容创作者 | 注册用户，主要发布文章 | 便捷发布、内容管理、获得反馈 |
| 读者 | 浏览文章的用户 | 发现内容、阅读体验、参与讨论 |
| 平台管理员 | 平台运营者 | 内容治理、用户管理、平台健康 |

### 1.3 核心价值

- **创作者**：零门槛发布，即时公开，按分类组织内容
- **读者**：按分类浏览，阅读流畅，参与评论互动
- **管理员**：全局管控，保障平台内容质量

## 2. 功能需求

### 2.1 用户系统

#### 2.1.1 用户注册

| 属性 | 说明 |
|------|------|
| 功能描述 | 新用户创建账号，成为平台注册用户 |
| 输入字段 | 用户名、邮箱、密码 |
| 业务规则 | 用户名全局唯一；密码长度 ≥ 8 位 |
| 角色 | 注册后默认角色为"普通用户" |
| 待确认 | 是否需要邮箱验证（Q-07） |

#### 2.1.2 用户登录

| 属性 | 说明 |
|------|------|
| 功能描述 | 已注册用户通过凭证登录平台 |
| 输入字段 | 用户名/邮箱 + 密码 |
| 业务规则 | 连续失败 5 次锁定账号 15 分钟 |
| 会话策略 | 登录态保持 7 天（可配置） |

#### 2.1.3 角色与权限矩阵

| 操作 | 普通用户 | 管理员 |
|------|:---:|:---:|
| 浏览文章 | ✓ | ✓ |
| 发布文章 | ✓ | ✓ |
| 编辑自己的文章 | ✓ | ✓ |
| 删除自己的文章 | ✓ | ✓ |
| 发表评论 | ✓ | ✓ |
| 删除自己的评论 | ✓ | ✓ |
| 编辑/删除任意文章 | ✗ | ✓ |
| 删除任意评论 | ✗ | ✓ |
| 管理用户（封禁/删除） | ✗ | ✓ |
| 管理分类 | ✗ | ✓ |

### 2.2 文章管理

#### 2.2.1 文章发布

| 属性 | 说明 |
|------|------|
| 功能描述 | 用户撰写并发布纯文字文章 |
| 输入字段 | 标题、正文、分类 |
| 内容格式 | 纯文字（不含图片、富文本、Markdown） |
| 业务规则 | 标题必填（≤100字）；正文必填（≤50000字）；分类必选 |
| 发布流程 | 提交后立即公开，无需审核 |

#### 2.2.2 文章编辑

| 属性 | 说明 |
|------|------|
| 功能描述 | 作者编辑自己已发布的文章 |
| 业务规则 | 仅作者本人和管理员可编辑；保留修改历史（待确认） |

#### 2.2.3 文章删除

| 属性 | 说明 |
|------|------|
| 功能描述 | 删除已发布的文章 |
| 业务规则 | 仅作者本人和管理员可删除；删除时同时删除关联评论；支持软删除（待确认） |

#### 2.2.4 文章列表与浏览

| 属性 | 说明 |
|------|------|
| 功能描述 | 用户浏览文章列表和查看文章详情 |
| 列表字段 | 标题、摘要（正文前200字）、作者、分类、发布时间、评论数 |
| 排序 | 默认按发布时间倒序 |
| 筛选 | 支持按分类筛选 |
| 待确认 | 分页策略（Q-06）；搜索功能（Q-04） |

### 2.3 评论功能

#### 2.3.1 评论发布

| 属性 | 说明 |
|------|------|
| 功能描述 | 用户对文章发表评论 |
| 输入字段 | 评论内容（纯文字） |
| 业务规则 | 评论内容 ≤ 2000 字 |
| 待确认 | 是否要求登录后评论（Q-01） |

#### 2.3.2 评论区展示

| 属性 | 说明 |
|------|------|
| 功能描述 | 文章详情页展示评论区 |
| 展示方式 | 按时间正序排列；显示评论者、时间、内容 |
| 数量控制 | 默认展示最新 20 条，支持加载更多 |

### 2.4 分类管理

| 属性 | 说明 |
|------|------|
| 功能描述 | 文章按分类组织，支持分类浏览 |
| 分类模型 | 预设固定分类 vs 管理员动态管理 |
| 待确认 | 由管理员创建和管理，还是预设固定（Q-02） |

### 2.5 管理后台

#### 2.5.1 文章管理

- 查看所有文章列表
- 编辑、删除任意文章

#### 2.5.2 评论管理

- 查看所有评论
- 删除不当评论

#### 2.5.3 用户管理（待确认 Q-03）

- 查看用户列表
- 封禁/解封用户
- 删除用户

#### 2.5.4 分类管理（待确认 Q-02）

- 创建、编辑、删除分类

## 5. 接口设计概要

### 5.1 认证相关

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 注册 | POST | `/api/auth/register` | 用户注册 |
| 登录 | POST | `/api/auth/login` | 用户登录，返回 token |
| 登出 | POST | `/api/auth/logout` | 用户登出 |

### 5.2 文章相关

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 文章列表 | GET | `/api/articles` | 分页，支持分类筛选 |
| 文章详情 | GET | `/api/articles/:id` | 包含评论列表 |
| 发布文章 | POST | `/api/articles` | 需登录 |
| 编辑文章 | PUT | `/api/articles/:id` | 需作者/管理员 |
| 删除文章 | DELETE | `/api/articles/:id` | 需作者/管理员 |

### 5.3 评论相关

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 评论列表 | GET | `/api/articles/:id/comments` | 分页 |
| 发表评论 | POST | `/api/articles/:id/comments` | 需登录 |
| 删除评论 | DELETE | `/api/comments/:id` | 需评论者/管理员 |

### 5.4 分类相关

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 分类列表 | GET | `/api/categories` | 公开 |
| 创建分类 | POST | `/api/categories` | 管理员 |
| 编辑分类 | PUT | `/api/categories/:id` | 管理员 |
| 删除分类 | DELETE | `/api/categories/:id` | 管理员 |

### 5.5 用户相关（管理后台）

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 用户列表 | GET | `/api/admin/users` | 管理员 |
| 封禁/解封用户 | PUT | `/api/admin/users/:id/status` | 管理员 |
| 删除用户 | DELETE | `/api/admin/users/:id` | 管理员 |

## 6. 数据模型概要

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Long (PK) | 主键 |

| 字段 | 类型 | 说明 |

| content | Text | 正文（纯文字） |
| user_id | Long (FK → User) | 作者 |
| category_id | Long (FK → Category) | 分类 |
| status | Enum: PUBLISHED, DELETED | 状态 |
| created_at | DateTime | 发布时间 |
| updated_at | DateTime | 编辑时间 |

### 6.3 Comment（评论）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Long (PK) | 主键 |
| content | Text | 评论内容 |
| user_id | Long (FK → User) | 评论者 |
| article_id | Long (FK → Article) | 所属文章 |
| created_at | DateTime | 评论时间 |

### 6.4 Category（分类）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Long (PK) | 主键 |
| name | String (unique) | 分类名称 |
| description | String | 分类描述 |
| created_at | DateTime | 创建时间 |

## 7. 非功能需求

### 7.1 性能

| 指标 | 目标值 |
|------|--------|
| 首屏加载时间 | ≤ 3 秒 |
| API 响应时间 (P95) | ≤ 500ms |
| 并发用户支持 | 100 并发 |

### 7.2 安全

- 密码使用 bcrypt/argon2 哈希存储
- 接口鉴权使用 JWT
- 防 XSS：输入转义，输出编码
- 防 CSRF：SameSite Cookie + CSRF Token
- 防 SQL 注入：使用参数化查询 / ORM

### 7.3 可用性

- 浏览器支持：Chrome、Firefox、Safari、Edge 最新两个大版本
- 响应式设计：待确认（Q-10）

### 7.4 可维护性

- 前后端分离架构
- RESTful API
- 代码仓库结构一致
