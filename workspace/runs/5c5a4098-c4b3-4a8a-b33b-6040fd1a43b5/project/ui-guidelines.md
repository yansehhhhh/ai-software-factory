# 个人博客 UI 设计规范

> 项目：慢慢写生活  
> 类型：静态个人博客  
> 页面：首页、文章列表、文章详情  
> 风格：温暖文艺风，偏生活记录  
> 版本：V1.0

---

## 1. 设计目标

本博客面向个人生活记录、随笔、阅读感悟和旅行片段展示。UI 设计应优先服务阅读体验，通过柔和色彩、充足留白、圆角卡片和温暖文案营造“安静、亲近、可停留”的感受。

核心设计目标：

- 让用户进入首页后快速理解博客主题与气质。
- 让精选文章在首页具有明确吸引力。
- 让文章列表便于浏览和选择。
- 让文章详情页形成舒适、低干扰的长文阅读环境。
- 全站无需登录、无需数据库、无需复杂操作。

---

## 2. 视觉关键词

| 关键词 | 设计表达 |
|---|---|
| 温暖 | 米白、杏色、焦糖棕、陶土色等低饱和暖色 |
| 文艺 | 细腻标题、柔和分隔线、诗意短文案 |
| 生活化 | 使用“日常”“午后”“雨天”“书桌”等内容意象 |
| 安静 | 少量动画、低对比背景、避免强烈视觉刺激 |
| 阅读友好 | 合理行高、窄阅读栏、清晰层级 |

---

## 3. 色彩规范

### 3.1 设计原则

- 不使用强烈纯白作为大面积背景，采用暖米白降低刺眼感。
- 正文文字使用深棕灰，保证阅读对比度。
- 主色用于按钮、链接、导航激活态，不大面积铺满。
- 辅助色用于标签、装饰色块、卡片背景。

### 3.2 色彩 Token

| Token | 色值 | 用途 |
|---|---|---|
| `--color-bg` | `#FBF5EC` | 页面主背景 |
| `--color-surface` | `#FFFDF8` | 卡片、正文容器背景 |
| `--color-surface-soft` | `#F7EADB` | 柔和区块背景 |
| `--color-primary` | `#A55F3F` | 主按钮、链接、重点元素 |
| `--color-primary-dark` | `#7E432D` | 主色 hover/active |
| `--color-secondary` | `#D9A875` | 装饰、渐变、标签背景 |
| `--color-accent` | `#7E8F68` | 辅助强调、自然感点缀 |
| `--color-text` | `#3F312A` | 正文主文本 |
| `--color-muted` | `#7D6E64` | 次级说明文本 |
| `--color-border` | `#E9D8C7` | 边框、分隔线 |
| `--color-focus` | `#6E7F53` | 键盘焦点描边 |

### 3.3 对比度要求

- 正文文本与背景对比度应满足 WCAG AA：至少 4.5:1。
- 链接和按钮不能仅依靠颜色区分，应同时使用字重、下划线或形态变化。
- 标签文字需保持可读，不使用过浅的浅棕文字叠加浅色背景。

## 4. 字体规范

### 4.1 字体栈

```css
font-family: -apple-system, BlinkMacSystemFont, "PingFang SC", "Microsoft YaHei", "Helvetica Neue", Arial, sans-serif;
```

### 4.2 字号层级

| 层级 | 桌面端 | 移动端 | 用途 |
|---|---:|---:|---|
| Display | 56px | 38px | 首页主标题 |
| H1 | 42px | 32px | 页面标题、文章标题 |
| H2 | 30px | 26px | 区块标题 |
| H3 | 22px | 20px | 卡片标题 |
| Body | 17px | 16px | 正文内容 |
| Small | 14px | 14px | 日期、标签、辅助信息 |
| Caption | 12px | 12px | 少量补充说明 |

### 4.3 行高

| 文本类型 | 行高 |
|---|---:|
| 标题 | 1.15 - 1.25 |
| 卡片摘要 | 1.65 |
| 正文段落 | 1.85 |
| 辅助文字 | 1.5 |

### 4.4 阅读宽度

- 文章详情正文最大宽度：`760px`。
- 桌面端正文每行建议 60-75 个中文字符以内。
- 移动端正文不小于 16px，避免 iOS 自动放大。

## 5. 间距与布局

### 5.1 基础间距系统

采用 4/8 倍数间距：

| Token | 值 | 用途 |
|---|---:|---|
| `space-1` | 4px | 极小间隔 |
| `space-2` | 8px | 图文微间距 |
| `space-3` | 12px | 标签内边距 |
| `space-4` | 16px | 组件内边距 |
| `space-6` | 24px | 卡片间距 |
| `space-8` | 32px | 区块内间距 |
| `space-12` | 48px | 区块间距 |
| `space-16` | 64px | 首屏大间距 |

### 5.2 页面容器

- 主容器最大宽度：`1120px`。
- 文章详情容器最大宽度：`760px`。
- 桌面端左右页边距：`32px`。
- 移动端左右页边距：`20px`。

### 5.3 响应式断点

| 断点 | 宽度 | 设计策略 |
|---|---:|---|
| Small | `< 640px` | 单列布局，导航简化，卡片全宽 |
| Medium | `640px - 1023px` | 单列或双列卡片，增加留白 |
| Large | `>= 1024px` | 多列精选文章，宽屏首屏布局 |

## 6. 组件规范

### 6.1 顶部导航 Header

内容：博客名称、首页入口、文章入口。

规范：

- 高度建议 72px 左右。
- 背景使用半透明暖白，并加轻微模糊或边框。
- 当前页面导航项使用主色和浅色胶囊背景突出。
- 移动端保持同一行，导航项不超过 2 个。

交互：

- Hover：背景变为浅杏色。
- Focus：显示 2px 焦点描边。
- Active：文字主色加深。

### 6.2 按钮 Button

主要按钮用于“查看所有文章”“阅读全文”等。

规范：

- 高度不低于 44px。
- 圆角：999px 胶囊形。
- 主按钮背景：`--color-primary`。
- 主按钮文字：暖白色。
- 次按钮使用透明背景 + 边框。

状态：

| 状态 | 表现 |
|---|---|
| Default | 主色背景或描边 |
| Hover | 背景加深、轻微上浮 |
| Active | 轻微缩放至 0.98 |
| Focus | 显示焦点环 |
| Disabled | 透明度 0.5，禁止点击 |

### 6.3 文章卡片 Post Card

用于首页精选文章和文章列表。

必备内容：

- 文章标题。
- 日期。
- 分类标签。
- 摘要。
- 阅读入口。

视觉规范：

- 背景：`--color-surface`。
- 圆角：24px。
- 边框：1px `--color-border`。
- 阴影：柔和、低透明度。
- 顶部可使用暖色装饰条或色块。

交互：

- 卡片可点击时需有 `cursor: pointer`。
- Hover 时轻微上浮 `translateY(-4px)`。
- 不应依赖 hover 才显示重要信息，移动端也必须可见。

### 6.4 标签 Tag

用于文章分类，如“生活记录”“阅读随笔”。

规范：

- 圆角：999px。
- 背景：浅杏色或淡橄榄色。
- 字号：13-14px。
- 颜色需与背景保持足够对比。

### 6.5 文章正文 Article Body

规范：

- 正文容器最大宽度 760px。
- 段落间距 18-24px。
- 行高 1.85。
- 段落首行不强制缩进。
- 引用或重点句可使用左边框和浅色背景。

## 7. 页面规范

### 7.1 首页

首页结构：

1. 顶部导航。
2. Hero 区：博客标题、简介、主 CTA。
3. 精选文章区：展示 3 篇精选文章。
4. 底部说明。

设计重点：

- Hero 应体现“生活记录”的气质。
- 精选文章需要在首屏或首屏下方容易被看到。
- 不放置过多入口，避免复杂化。

### 7.2 文章列表页

文章列表结构：

1. 页面标题“所有文章”。
2. 页面说明。
3. 文章卡片列表。

设计重点：

- 按发布时间倒序展示。
- 卡片摘要最多 2-3 行。
- 移动端单列，桌面端可使用双列网格。

### 7.3 文章详情页

文章详情结构：

1. 返回入口。
2. 文章标题。
3. 日期和分类。
4. 正文内容。
5. 底部返回列表入口。

设计重点：

- 详情页应降低装饰密度，以阅读为核心。
- 标题和正文之间留足空间。
- 返回入口明显但不喧宾夺主。

## 8. 交互规范

### 8.1 导航

- 使用锚点或前端静态切换模拟页面切换。
- 所有主要页面必须可以从顶部导航访问。
- 文章详情页必须提供返回文章列表入口。

### 8.2 动效

- 微交互时长：150-240ms。
- 动画属性优先使用 `transform` 和 `opacity`。
- 不使用大幅度移动、旋转和持续循环动画。
- 必须支持 `prefers-reduced-motion`，用户开启减少动态效果时禁用过渡动画。

### 8.3 可访问性

- HTML 使用语义标签：`header`、`nav`、`main`、`article`、`section`、`footer`。
- 页面必须只有一个主 `h1`。
- 交互元素必须支持键盘 Tab 聚焦。
- 焦点状态清晰可见。
- 按钮和链接文案应明确，例如“阅读《春日午后的散步》”。

---

## 9. 空状态与异常状态

### 9.1 无文章状态

如文章数据为空，展示：

> 还没有文章。等风来，也等第一篇文字。

并保留返回首页入口。

### 9.2 文章不存在

访问不存在的文章时，展示：

> 没有找到这篇文章，也许它还在路上。

并提供“返回所有文章”按钮。
