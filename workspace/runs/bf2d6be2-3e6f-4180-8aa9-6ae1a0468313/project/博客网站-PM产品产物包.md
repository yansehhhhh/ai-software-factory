## 一、需求文档

### 1.1 项目概述

**项目名称**：个人博客网站（静态站点）

**一句话描述**：一个基于 Markdown 文件 + Git 管理的个人博客网站，作者通过编辑代码仓库中的 Markdown 文件发布文章，站点通过静态生成方式呈现文章列表与详情。

### 1.2 需求来源 & 确认过程

| 轮次 | 问题 | 用户选择 |
|------|------|----------|
| Q1 | 博客类型？ | 个人博客（单作者） |
| Q2 | 内容管理方式？ | Markdown 文件 + Git 管理，静态生成 |
| Q3 | 额外功能需求？ | 只需文章列表和详情，暂不需要其他 |

### 1.3 已确认需求清单

| # | 需求项 | 确认内容 |
|---|--------|----------|
| R1 | 博客类型 | 个人博客，仅一个作者撰写和发布文章 |
| R2 | 内容管理 | Markdown 文件存储在代码仓库中，通过 Git 提交发布 |
| R3 | 构建方式 | 静态站点生成（SSG），读取 Markdown 文件生成 HTML 页面 |
| R4 | 页面范围 | 仅包含文章列表页 + 文章详情页 |
| R5 | 功能范围 | 不做标签分类、不做 RSS、不做评论、不做搜索 |
| R6 | 发布流程 | 作者在代码仓库中新增/修改 Markdown 文件 → 提交 Git → 触发构建部署 |

### 1.4 需求边界

**本次做**：
- 文章列表页：按时间倒序展示所有文章，含标题、日期、摘要
- 文章详情页：渲染 Markdown 正文内容，含标题、日期、返回链接
- Markdown 文件约定：统一的 front matter（标题、日期） + 正文

**本次不做**：
- 标签/分类系统
- RSS/Atom feed
- 评论系统
- 全文搜索
- 后台管理界面 / 在线编辑器
- 用户注册/登录
- 多作者支持
- 草稿/发布状态管理
- 文章统计/阅读量
- 社交媒体分享

## 二、PRD（产品需求文档）

### 2.1 产品定位

面向个人技术/写作者的低维护成本博客。作者只需在本地编写 Markdown 文件并推送 Git，即可自动更新网站。适合习惯用 Markdown 写作、偏好版本控制管理内容的用户。

### 2.2 用户画像

| 属性 | 描述 |
|------|------|
| 角色 | 独立写作者（也是本站唯一作者和运维者） |
| 技能 | 熟悉 Markdown 和 Git 基本操作 |
| 诉求 | 希望有一个简洁的写作发布渠道，不想维护数据库和后台系统 |
| 频率 | 不定期写作发布，低频更新 |

### 2.3 功能规格

#### 2.3.1 文章列表页（首页）

| 属性 | 规格 |
|------|------|
| URL | `/` 或 `/index.html` |
| 数据源 | 读取 `content/posts/` 目录下所有 `.md` 文件 |
| 排序 | 按 front matter 中的 `date` 降序 |
| 每项内容 | 文章标题（链接到详情页）、发布日期、摘要（正文前 N 字或 front matter 中的 `excerpt`） |
| 空状态 | 无文章时显示"暂无文章"占位提示 |

#### 2.3.2 文章详情页

| 属性 | 规格 |
|------|------|
| URL | `/posts/{slug}.html` |
| 数据源 | 读取对应 slug 的 `.md` 文件 |
| 内容 | 标题、发布日期、完整的 Markdown 渲染正文 |
| 导航 | 返回文章列表的链接 |
| 404 | 文章不存在时显示"文章未找到"提示，并提供返回首页链接 |

#### 2.3.3 Markdown 文件约定

```yaml
---
title: 文章标题
date: 2026-04-27
---
文章正文（Markdown 格式）...
```

| 字段 | 必填 | 说明 |
|------|------|------|
| `title` | 是 | 文章标题，显示在列表页和详情页 |
| `date` | 是 | 发布日期，格式 `YYYY-MM-DD` |
| slug | 自动 | 由文件名派生，如 `hello-world.md` → slug = `hello-world` |

### 2.4 非功能需求

| 类别 | 要求 |
|------|------|
| 性能 | 纯静态 HTML，首屏加载 < 1s（Fast 3G 网络下） |
| 响应式 | 适配桌面端（≥1024px）、平板端（768-1023px）、手机端（<768px） |
| 可访问性 | 语义化 HTML，图片需 `alt` 属性，合理的标题层级 |
| SEO | 含 `<title>`、`<meta description>`、Open Graph 标签 |
| 构建 | 构建时间应随文章数量线性增长，100 篇文章构建 < 10s |
| 浏览器 | 支持 Chrome/Firefox/Safari/Edge 最近两个大版本 |

### 2.5 信息架构

```
个人博客站点
├── /（文章列表页 — 首页）
│   └── 文章卡片 × N
│       └── 点击跳转 → /posts/{slug}.html
└── /posts/{slug}.html（文章详情页）
    └── 返回首页链接
```

### 2.6 技术约束

| 约束 | 说明 |
|------|------|
| 无服务端运行时 | 纯静态 HTML/CSS/JS 文件，可部署到任意静态托管（GitHub Pages、Vercel、Netlify 等） |
| 构建工具待定 | 需选择 SSG 框架（如 VuePress、VitePress、Astro、11ty 等）或自研脚本 |
| 内容目录 | Markdown 源文件统一存放于 `content/posts/` 目录 |

## 三、原型说明

### 3.1 全局布局结构

```
┌──────────────────────────────────────────────┐
│  Header                                       │
│  ┌────────────────────────────────────────┐   │
│  │  博客名称                    [可选导航]  │   │
│  └────────────────────────────────────────┘   │
├──────────────────────────────────────────────┤
│                                               │
│  页面内容区（列表页 or 详情页）                 │
│                                               │
├──────────────────────────────────────────────┤
│  Footer                                       │
│  ┌────────────────────────────────────────┐   │
│  │  © 2026 作者名 · Powered by xxx        │   │
│  └────────────────────────────────────────┘   │
└──────────────────────────────────────────────┘
```

**Header 组件**：
- 左侧：博客名称（可点击，回到首页）
- 右侧：可选导航链接（如"关于"页面，待确认后实现）
- 移动端：博客名称居中，导航可收起到汉堡菜单

**Footer 组件**：
- 居中版权信息
- 技术标识（如"Built with XXX"）

### 3.2 页面一：文章列表页（首页）

#### 3.2.1 页面结构

```
┌─────────────────────────────────────────────────┐
│  Header                                          │
├─────────────────────────────────────────────────┤
│                                                  │

│  │  文章卡片 #1                               │   │
│  │  ┌─────────────────────────────────────┐  │   │

│  │  │  摘要文字摘要文字摘要文字摘要文字...     │  │   │
│  │  └─────────────────────────────────────┘  │   │

```

#### 3.2.2 核心交互

| 交互 | 触发 | 行为 |
|------|------|------|
| 点击文章标题 | 用户点击卡片标题/区域 | 导航到文章详情页 `/posts/{slug}.html` |
| 点击博客名称（Header） | 用户在非首页点击 | 回到首页 `/` |
| 页面滚动 | 用户滚动页面 | 正常滚动浏览所有文章卡片 |
| 响应式断点 | 浏览器窗口变化 | 卡片宽度自适应，移动端单列，桌面端可略宽 |

#### 3.2.3 状态流转

```
[页面加载] → 读取文章列表数据
                ├── 有文章 → 渲染文章卡片列表
                └── 无文章 → 显示"暂无文章"空状态

[用户点击文章] → 浏览器导航到详情页
```

#### 3.2.4 空状态设计

│      暂无文章                               │

```

```

## 四、HTML 原型结构说明

### 4.1 站点文件结构

```
dist/                          # 构建输出目录
├── index.html                 # 首页（文章列表）
├── posts/
│   ├── hello-world.html       # 文章详情页
│   └── ...
├── assets/
│   ├── css/
│   │   └── style.css          # 全局样式
│   └── images/                # 文章引用的图片（如有）

    ├── hello-world.md
    └── ...
```

## 六、用户故事

### Epic：读者浏览个人博客

#### US-01：访问博客首页查看文章列表
**作为** 读者  
**我想要** 打开博客首页时看到所有已发布文章的列表  
**以便于** 快速浏览和选择感兴趣的文章阅读  

**验收标准**：
- [ ] 首页 URL 为 `/`
- [ ] 文章按发布日期降序排列（最新的在最上面）
- [ ] 每篇文章显示：标题（可点击）、发布日期、摘要
- [ ] 无文章时显示"暂无文章"占位提示
- [ ] 页面含 Header（博客名称）和 Footer（版权信息）

#### US-02：阅读单篇文章
**作为** 读者  
**我想要** 从文章列表点击标题进入文章详情页  
**以便于** 阅读完整的文章内容  

**验收标准**：
- [ ] 详情页 URL 为 `/posts/{slug}.html`
- [ ] 页面显示文章标题、发布日期、完整正文（Markdown 渲染）
- [ ] 页面顶部和底部均有"返回文章列表"链接
- [ ] 页面含 Header 和 Footer

#### US-03：处理文章不存在的情况
**作为** 读者  
**我想要** 在访问不存在的文章时看到友好的提示  
**以便于** 知道文章不存在并返回首页  

**验收标准**：
- [ ] 不存在的 slug 显示 404 提示："文章未找到"
- [ ] 提供"返回首页"链接

#### US-04：在移动设备上正常浏览
**作为** 读者  
**我想要** 在手机和平板上舒适地阅读文章  
**以便于** 随时随地浏览博客  

**验收标准**：
- [ ] 在宽度 < 768px 的设备上布局正常（单列，无横向滚动）
- [ ] 标题和正文字号适合移动阅读
- [ ] 代码块可水平滚动

#### US-05：通过 Markdown 文件发布新文章

- [ ] 修改 front matter 中的 `date` 会影响列表排序
