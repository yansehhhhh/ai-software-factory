## 一、需求文档

### 1.1 项目背景

在企业和团队日常运营中，经常需要进行各类决策投票，如方案选择、意见征集、团队决策等。传统的线下投票方式存在效率低、统计困难、参与度受限等问题。本项目旨在构建一个面向内部决策场景的在线投票系统，提升决策效率和参与便利性。

### 1.2 业务目标

- **提升决策效率**：将传统线下投票转为线上，减少组织成本和时间成本
- **增强参与便利性**：支持随时随地参与投票，提高团队参与度
- **保证投票公正性**：支持匿名投票，确保投票过程公平透明
- **简化结果统计**：自动统计投票结果，实时展示数据，减少人工统计误差

### 1.3 目标用户

| 用户角色 | 描述 | 核心需求 |
|---------|------|---------|
| 投票发起人 | 创建和管理投票的管理员 | 快速创建投票、设置权限、查看统计结果 |
| 投票参与者 | 参与投票的团队成员 | 简单易用的投票界面、查看投票结果 |
| 系统管理员 | 系统运维人员 | 用户管理、权限配置、系统监控 |

### 1.4 核心业务流程

```
投票发起人 → 创建投票 → 设置投票规则 → 发布投票 → 参与者投票 → 实时统计 → 结果展示
```

### 1.5 功能范围

#### 1.5.1 核心功能（MVP）

**投票管理**
- 创建投票：支持单选和多选投票类型
- 编辑投票：修改投票标题、描述、选项
- 删除投票：移除不再需要的投票
- 发布投票：设置投票状态为进行中
- 结束投票：手动结束或自动到期结束

**投票参与**
- 查看投票列表：展示所有可参与的投票
- 投票操作：单选或多选提交
- 查看投票结果：查看实时统计数据

**投票设置**
- 匿名/实名设置：保护投票隐私
- 投票时间限制：设置开始和结束时间
- 参与权限：限制参与人员范围
- 结果可见性：设置结果公开时间

**用户管理**
- 用户注册/登录：基础身份认证
- 权限管理：区分管理员和普通用户

#### 1.5.2 扩展功能（二期）

- 投票模板：常用投票场景模板
- 投票提醒：消息通知功能
- 数据导出：导出投票数据和统计报表
- 移动端适配：响应式设计或原生 App

### 1.6 非功能性需求

- **性能**：支持 500 人同时投票，系统响应时间 < 2 秒
- **可用性**：系统可用性 ≥ 99.5%
- **安全性**：
  - 投票数据加密存储
  - 防止重复投票
  - 权限隔离，防止越权操作
- **易用性**：
  - 用户无需培训即可完成投票操作
  - 投票创建流程简洁，3 步内完成
- **兼容性**：支持主流浏览器（Chrome、Firefox、Safari、Edge）

## 二、产品需求文档（PRD）

### 2.1 产品概述

在线投票系统是一个面向企业内部决策场景的轻量级投票工具，支持单选和多选投票类型，提供投票创建、参与、统计一站式服务。

**作为** 投票发起人
**我想要** 创建一个新的投票
**以便于** 收集团队成员的意见

- [ ] 能够输入投票标题（必填，≤50 字符）

- [ ] 能够添加至少 2 个选项（每个选项 ≤100 字符）
- [ ] 能够删除选项
- [ ] 能够设置投票时间范围
- [ ] 能够设置匿名/实名
- [ ] 能够保存为草稿或直接发布

### 2.4 业务规则

#### BR-001：投票创建规则
- 单选投票至少 2 个选项，最多 20 个选项
- 多选投票至少 2 个选项，最多 20 个选项
- 每个选项文字长度 ≤ 100 字符
- 投票标题 ≤ 50 字符
- 投票描述 ≤ 500 字符

#### BR-002：投票参与规则
- 每个用户对每个投票只能投票一次
- 单选投票只能选择一个选项
- 多选投票可选择多个选项，可设置最大选择数量
- 投票提交后不可修改

#### BR-003：投票结果展示规则
- 实名投票：所有人可查看投票详情和投票人
- 匿名投票：仅发起人可查看统计结果，不显示投票人信息
- 投票结束前是否显示结果由发起人设置

#### BR-004：权限规则
- 投票发起人：可创建、编辑、删除、发布、结束自己创建的投票
- 投票参与者：可查看、参与有权限参与的投票
- 系统管理员：拥有所有权限，可管理所有投票和用户

### 2.5 数据模型（概要）

#### 投票（Vote）
```
- id: 投票ID
- title: 投票标题
- description: 投票描述
- type: 投票类型（单选/多选）
- is_anonymous: 是否匿名
- status: 状态（草稿/进行中/已结束）
- start_time: 开始时间
- end_time: 结束时间
- creator_id: 创建人ID
- max_choices: 多选最大选择数（多选时）
- show_result_before_end: 结束前是否显示结果
- created_at: 创建时间
- updated_at: 更新时间
```

#### 投票选项（VoteOption）
```
- id: 选项ID
- vote_id: 所属投票ID
- content: 选项内容
- sort_order: 排序
- created_at: 创建时间
```

#### 投票记录（VoteRecord）
```
- id: 记录ID
- vote_id: 投票ID
- user_id: 用户ID（匿名投票时加密存储）
- option_ids: 选择的选项ID列表
- submitted_at: 提交时间
```

#### 用户（User）
```
- id: 用户ID
- username: 用户名
- password: 密码（加密）
- email: 邮箱
- role: 角色（管理员/普通用户）
- created_at: 创建时间
```

### 3.2 核心页面

#### 3.2.1 登录页面

**页面元素**：
- 顶部：系统 Logo + 标题
- 中部：登录表单
  - 用户名/邮箱输入框
  - 密码输入框
  - "记住我"复选框
  - 登录按钮
- 底部：注册链接

**交互流程**：
1. 用户输入用户名和密码
2. 点击"登录"按钮
3. 验证成功 → 跳转至投票列表页
4. 验证失败 → 显示错误提示，用户可重新输入

**状态说明**：
- 默认状态：表单空白，登录按钮禁用
- 输入状态：用户名和密码都输入后，登录按钮启用
- 加载状态：点击登录后，按钮显示"登录中..."并禁用
- 成功状态：跳转至投票列表
- 失败状态：显示"用户名或密码错误"

---

#### 3.2.2 投票列表页面

**页面元素**：
- 顶部导航栏：
  - 左侧：系统 Logo
  - 中间：搜索框
  - 右侧：用户头像 + 下拉菜单（个人中心、退出）
- 筛选区：
  - 状态筛选标签：全部、进行中、未开始、已结束
  - 创建投票按钮（右上角）
- 列表区：
  - 卡片式展示每个投票
  - 每张卡片包含：投票标题、状态标签、参与人数、结束时间、操作按钮（查看/编辑/删除）

**交互流程**：
1. 默认展示所有投票，按创建时间倒序
2. 点击状态标签筛选对应状态的投票
3. 搜索框输入关键词，实时过滤投票
4. 点击"创建投票"跳转至创建页面
5. 点击卡片上的"查看"跳转至投票详情
6. 点击"编辑"跳转至编辑页面（仅发起人可见）
7. 点击"删除"弹出确认对话框

**状态流转**：
- 未开始 → 进行中：到达开始时间自动切换
- 进行中 → 已结束：到达结束时间自动切换或手动结束

#### 3.2.3 投票详情页面

**页面元素**：
- 投票信息区：
  - 投票标题
  - 投票描述
  - 状态标签
  - 时间信息（开始时间、结束时间、剩余时间）
- 投票操作区（未投票用户可见）：
  - 单选：单选按钮列表
  - 多选：复选框列表
  - 提交按钮

  - 各选项得票数和百分比
  - 柱状图或饼图可视化
  - 总投票人数
- 操作按钮区（发起人可见）：
  - 编辑按钮（未开始/进行中状态）
  - 结束投票按钮（进行中状态）
  - 删除按钮

**交互流程**：
1. 进入页面，加载投票详情
2. 未投票用户：
   - 选择选项
   - 点击"提交投票"
   - 确认提交 → 显示成功提示 → 更新页面显示结果
3. 已投票用户：
   - 显示"已投票"状态
   - 根据设置显示或隐藏结果
4. 发起人：
   - 可见编辑、结束、删除按钮
   - 点击"结束投票" → 确认 → 投票状态变为"已结束"

**关键组件**：
- 单选组件：Radio Button Group
- 多选组件：Checkbox Group
- 结果图表：柱状图（推荐）/ 饼图

#### 4.4.1 投票提交互动

```javascript
// 提交投票
async function submitVote() {
  // 验证是否选择选项
  if (selectedOptions.value.length === 0) {
    showToast('请选择投票选项');
    return;
  }

  // 弹出确认对话框
  const confirmed = await showConfirmDialog('确认提交投票吗？提交后不可修改。');
  if (!confirmed) return;

  try {
    // 提交投票
    await api.vote.submit({
      voteId: vote.value.id,
      optionIds: selectedOptions.value
    });

    // 显示成功提示
    showToast('投票成功！');

    // 更新页面状态
    hasVoted.value = true;

    // 刷新投票结果（如果允许显示）
    if (vote.value.showResultBeforeEnd) {
      fetchVoteResult();
    }
  } catch (error) {
    showToast('投票失败：' + error.message);
  }
}
```

#### 4.4.2 创建投票验证

```javascript
// 表单验证
function validateForm() {
  // 标题验证
  if (!voteForm.value.title.trim()) {
    showToast('请输入投票标题');
    return false;
  }
  if (voteForm.value.title.length > 50) {
    showToast('投票标题不能超过50个字符');
    return false;
  }

  // 选项验证
  const validOptions = voteForm.value.options.filter(opt => opt.content.trim());
  if (validOptions.length < 2) {
    showToast('至少需要2个选项');
    return false;
  }
  if (validOptions.some(opt => opt.content.length > 100)) {
    showToast('选项内容不能超过100个字符');
    return false;
  }

  // 时间验证
  if (voteForm.value.endTime <= voteForm.value.startTime) {
    showToast('结束时间必须晚于开始时间');
    return false;
  }

  return true;
}
```
