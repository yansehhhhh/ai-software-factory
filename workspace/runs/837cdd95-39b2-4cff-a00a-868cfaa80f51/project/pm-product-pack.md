## 1. 产品需求文档 PRD

### 产品背景

随着内容创作者数量增长，个人博客平台已不能满足多人协作、社区互动和内容管理的需求。用户需要一个支持多作者、评论互动、关注系统和内容分类检索的现代化博客平台，以促进创作者社群建设和内容传播。

### 目标用户

**主要用户群体：**
- 内容创作者（博主）：希望发布文章、管理个人内容、建立读者社群
- 内容读者：希望阅读优质文章、参与评论、关注喜欢的作者
- 平台管理员：负责内容审核、用户管理和平台运营

**用户画像：**
- 创作者：25-45岁，技术、生活、商业等领域博主，每周发布1-3篇文章
- 读者：18-35岁，希望通过关注系统跟踪感兴趣的作者和主题
- 管理员：平台运营人员，负责审核内容、处理举报和维护社区秩序

### 用户问题

**创作者痛点：**
- 缺乏便捷的多用户协作和内容管理工具
- 无法有效建立读者社群和粉丝互动机制
- 文章分类和标签管理不够灵活

**读者痛点：**
- 无法一站式关注多个喜欢的作者
- 缺乏高效的内容搜索和筛选方式
- 评论互动体验不流畅

**管理员痛点：**
- 内容审核和用户管理缺乏系统化工具
- 无法有效监控平台运营数据

### 产品目标

**第一版 MVP 目标：**
- 提供完整的文章发布、阅读和管理能力
- 实现评论、关注和标签分类等核心社交功能
- 支持全文搜索和内容筛选
- 提供简洁现代的用户体验

**成功指标：**
- 用户注册数：第一月达到 1000+ 注册用户
- 文章发布：第一月发布 300+ 篇文章
- 用户活跃度：DAU/MAU ≥ 30%
- 评论互动率：文章平均评论数 ≥ 5

### 功能范围

**第一版 MVP 包含：**

| 功能模块 | 核心能力 | 优先级 |
|---------|---------|--------|
| 用户管理 | 注册、登录、个人主页、个人设置 | P0 |
| 文章管理 | 发布、编辑、删除、草稿保存、富文本编辑器 | P0 |
| 文章展示 | 首页列表、详情页、作者主页、时间排序 | P0 |
| 评论系统 | 发布评论、回复评论、删除评论、评论排序 | P0 |
| 关注系统 | 关注作者、取消关注、关注列表、粉丝列表 | P0 |
| 标签分类 | 创建标签、文章标签关联、标签筛选、热门标签 | P0 |
| 全文搜索 | 关键词搜索、结果排序、搜索历史 | P0 |
| 内容审核 | 待审核列表、审核通过/拒绝、举报处理 | P1 |

**技术实现：**
- 前端：Nuxt.js（Vue 全栈框架）+ 简约现代 UI
- 后端：Python + FastAPI（推荐，高性能异步 API）
- 数据库：PostgreSQL（推荐，适合复杂查询和标签系统）

### 非目标

**第一版不包含：**
- 富媒体内容（视频、音频上传）
- 付费订阅和会员系统
- 邮件通知和推送系统
- 数据分析和可视化仪表盘
- 第三方登录（OAuth）
- 文章导入导出
- Markdown 编辑器（第一版使用富文本）
- 移动端 App
- 多语言支持

### 核心用户流程

**流程 1：创作者发布文章**
```
创作者登录 → 进入创作中心 → 点击"发布新文章" → 填写标题、内容、标签 → 选择发布状态（发布/草稿） → 提交审核 → 等待审核结果 → 发布成功/失败提示
```

**流程 2：读者阅读与互动**
```
读者访问首页 → 浏览文章列表 → 点击文章标题 → 进入详情页 → 阅读内容 → 发布评论 → 关注作者 → 查看作者其他文章
```

**流程 3：管理员内容审核**
```
管理员登录后台 → 查看待审核列表 → 点击文章 → 阅读内容 → 决定通过/拒绝 → 执行审核操作 → 通知作者
```

### 验收摘要

**功能验收：**
- 所有 P0 功能模块完整可用
- 核心用户流程顺畅无阻
- 无阻塞性 bug 和安全漏洞

**性能验收：**
- 页面首屏加载时间 ≤ 2 秒
- API 响应时间 ≤ 500ms（P95）
- 支持 1000 并发用户访问

**体验验收：**
- UI 符合简约现代设计规范
- 关键操作提供明确反馈
- 错误提示友好且可理解

```gherkin
Then 系统保存文章并提交审核
And 创作者收到"已提交审核"提示

Given 文章标题或正文不符合要求
When 创作者提交发布
Then 系统显示字段验证错误并阻止提交
```

#### US-012 发布评论

```gherkin

When 填写评论内容（5-500字符）并提交
Then 系统保存评论并立即显示在评论区
And 评论包含发布时间和用户信息

Given 评论内容为空或过短
When 用户提交评论
Then 系统显示"评论内容需5-500字符"提示
```

#### US-015 关注作者

```gherkin

When 点击"关注"按钮

When 再次点击关注按钮
Then 系统取消关注并更新按钮状态为"关注"
And 作者粉丝数减少 1
```

## 3. WWAS 条目

### WWAS-01: 用户注册与登录系统

**Why：**
用户需要账号体系才能区分创作者和读者身份，实现个性化内容管理和社交互动。账号系统是所有后续功能的基础。

**What：**
- 用户注册页面：邮箱、密码、昵称字段，字段验证逻辑
- 用户登录页面：邮箱和密码验证，Session/JWT 管理
- 个人主页：展示用户昵称、简介、头像、文章列表、粉丝数
- 个人设置页面：修改昵称、简介、头像上传

**Acceptance：**
- 注册字段验证：邮箱格式有效、密码 8-20 位、昵称 2-20 字符，邮箱不可重复
- 登录验证：正确邮箱密码可登录，错误时显示提示
- 个人主页正常展示用户信息和文章列表
- 个人设置修改后立即生效并展示更新

**Scope：**
- 用户注册、登录、登出流程
- 个人主页基础展示
- 个人设置基础功能

**Out of Scope：**
- 第三方登录（OAuth）
- 找回密码功能
- 账号注销功能
- 用户等级和积分系统

### WWAS-02: 文章发布与管理系统

**Why：**
创作者需要便捷的工具发布和管理文章，这是博客平台的核心功能。良好的文章管理体验能吸引更多创作者使用平台。

**What：**
- 文章编辑器：标题、正文（富文本）、标签选择、封面图上传、发布状态（发布/草稿）
- 文章管理列表：展示创作者所有文章，支持编辑、删除操作
- 文章状态管理：草稿、待审核、已发布、已拒绝
- 文章详情页：展示完整内容、作者信息、发布时间、标签、评论区

**Acceptance：**
- 文章标题 5-100 字符，正文 ≥100 字符，标签 1-5 个
- 富文本编辑器支持基本格式（标题、段落、粗体、斜体、链接）
- 草稿保存后可再次编辑和发布
- 已发布文章可编辑内容并重新提交审核
- 删除文章时需二次确认

**Scope：**
- 文章创建、编辑、删除流程
- 草稿保存和状态管理
- 富文本编辑器基础功能
- 文章详情页展示

**Out of Scope：**
- Markdown 编辑器
- 文章导入导出
- 文章版本历史
- 多语言支持

### WWAS-03: 评论互动系统

**Why：**
评论是读者与创作者互动的核心方式，能够促进社区活跃度和用户粘性。良好的评论体验能增强用户参与感。

**What：**
- 文章详情页评论区：评论输入框、评论列表、回复按钮
- 评论列表排序：按时间倒序展示
- 评论回复：支持回复他人评论，形成评论树
- 评论管理：用户可删除自己的评论

**Acceptance：**
- 评论内容 5-500 字符，支持基本文本格式
- 评论发布后立即显示在评论区
- 回复评论显示在父评论下方，形成层级结构
- 删除评论需二次确认，删除后不可恢复

**Scope：**
- 评论发布、回复、删除功能
- 评论列表展示和排序
- 评论与文章的关联关系

**Out of Scope：**
- 评论点赞功能
- 评论举报功能
- 评论审核机制
- @用户提醒功能

### WWAS-04: 关注与粉丝系统

**Why：**
关注系统能帮助读者跟踪喜欢的创作者，促进创作者与读者的社群连接，增强平台社交属性和用户粘性。

**What：**
- 关注按钮：文章详情页、作者主页的关注/取消关注按钮
- 关注列表页面：展示用户关注的所有作者及其最新文章
- 粉丝列表页面：展示创作者的所有粉丝
- 关注状态同步：关注后按钮状态立即更新，粉丝数实时变化

**Acceptance：**
- 点击关注按钮后状态立即切换为"已关注"，粉丝数增加
- 点击取消关注后状态切换为"关注"，粉丝数减少
- 关注列表展示所有已关注作者，可点击进入作者主页
- 粉丝列表展示所有粉丝用户

**Scope：**
- 关注和取消关注操作
- 关注列表和粉丝列表展示
- 关注状态实时同步

**Out of Scope：**
- 关注分组管理
- 关注提醒和通知
- 推荐关注功能
- 粉丝互动统计

### WWAS-05: 标签分类系统

**Why：**
标签能够帮助读者快速筛选感兴趣的主题内容，帮助创作者更好地组织文章分类，提升内容发现效率和平台内容结构化。

**What：**
- 文章发布时标签选择：创建新标签或选择已有标签
- 标签筛选页面：点击标签展示该标签下所有文章
- 热门标签展示：首页或侧边栏展示热门标签列表
- 标签管理：创作者可为文章添加 1-5 个标签

**Acceptance：**
- 文章发布时可添加 1-5 个标签，标签名称 2-20 字符
- 标签筛选页面展示该标签下所有文章，按时间倒序排列
- 热门标签根据文章数量动态排序，展示 Top 10

**Scope：**
- 文章标签关联和管理
- 标签筛选和热门标签展示

**Out of Scope：**
- 标签层级和子标签
- 标签推荐系统
- 标签审核机制
- 标签合并和删除

### WWAS-06: 全文搜索功能

**Why：**
搜索是用户快速找到感兴趣内容的关键功能，能够提升内容发现效率和用户满意度，是平台内容导航的重要手段。

**What：**
- 搜索输入框：首页和导航栏的搜索入口
- 搜索结果页面：展示匹配文章列表，包含标题、摘要、作者、时间
- 搜索排序：支持按相关性和时间排序
- 搜索历史：记录用户搜索关键词（可选）

**Acceptance：**
- 搜索支持文章标题和正文关键词匹配
- 搜索结果按相关性或时间排序展示
- 无结果时显示"未找到匹配内容"提示

**Scope：**
- 关键词搜索和结果展示
- 搜索结果排序功能

**Out of Scope：**
- 高级搜索（组合条件）
- 搜索结果推荐
- 搜索 Analytics 统计
- 搜索词联想和补全

### WWAS-07: 内容审核管理（P1）

**Why：**
平台需要管理员审核文章内容以确保合规和质量，防止不当内容进入平台，维护社区秩序和品牌形象。

**What：**
- 管理员后台登录：独立的管理员登录入口
- 待审核列表：展示所有待审核文章，包含标题、作者、提交时间
- 文章审核页面：阅读文章内容，执行通过/拒绝操作
- 审核状态更新：文章状态从"待审核"变为"已发布"或"已拒绝"

**Acceptance：**
- 管理员可查看所有待审核文章列表
- 点击文章进入审核页面，阅读完整内容
- 通过审核后文章状态变为"已发布"，作者收到通知
- 拒绝审核后文章状态变为"已拒绝"，作者收到拒绝原因

**Scope：**
- 管理员后台登录和权限验证
- 待审核文章列表和审核操作
- 审核状态管理和通知（简单版）

**Out of Scope：**
- 审核历史和统计
- 审核规则自动化
- 举报处理系统
- 审核权限分级
