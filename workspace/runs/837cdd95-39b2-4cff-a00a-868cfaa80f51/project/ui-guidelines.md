# UI 设计规范 - 多用户博客平台

> 生成日期：2026-04-27
> 设计风格：简约现代风格（Minimal Modern）
> 产品类型：内容平台 / 博客系统

---

## 1. 设计理念

### 1.1 核心原则

**简约现代风格**强调以下核心特征：

- **内容优先**：设计服务于内容，避免视觉噪音干扰阅读体验
- **大量留白**：通过空间分隔建立清晰的视觉层次和信息分组
- **极简元素**：减少装饰性元素，使用直线条、无圆角或小圆角、清晰边界
- **克制用色**：有限的颜色数量（通常不超过 3-4 种主要色彩），突出内容本身
- **现代字体**：使用无衬线字体，强调清晰度和可读性
- **功能导向**：每个视觉元素必须有明确的功能目的，不做无意义的装饰

### 1.2 设计目标

- **阅读舒适**：优化文章阅读体验，确保长时间阅读不疲劳
- **操作直观**：核心功能（发布、评论、关注）一目了然，操作路径清晰
- **性能流畅**：简洁设计减少渲染负担，提升页面加载速度和交互响应
- **品牌一致**：统一的设计语言贯穿所有页面和组件

## 2. 色彩系统

### 2.1 主色彩体系

采用**克制用色**原则，定义有限但功能明确的色彩体系：

#### 主色 - Primary

| 用途 | 色值 | 使用场景 |
|------|------|---------|
| Primary Brand | `#1a1a1a` (深灰/黑) | 主标题、重要文字、品牌强调 |
| Primary Accent | `#2563eb` (蓝色) | 主要交互按钮、链接、状态指示 |
| Primary Light | `#f5f5f5` (浅灰) | 背景色、分隔区域 |

**为什么选择深灰而非纯黑？**
- 深灰（#1a1a1a）比纯黑更柔和，长时间阅读更舒适
- 符合简约现代风格"不极端"的设计理念
- 与白色背景对比度适中（约 16:1），满足 WCAG AAA 标准

#### 功能色 - Semantic Colors

| 功能 | 色值 | 对比值 | 使用场景 |
|------|------|--------|---------|
| 成功 Success | `#059669` (绿) | 4.5:1 | 成功提示、发布成功状态 |
| 错误 Error | `#dc2626` (红) | 4.5:1 | 错误提示、删除警告 |
| 警告 Warning | `#d97706` (橙) | 4.5:1 | 警告提示、待审核状态 |
| 信息 Info | `#0284c7` (蓝) | 4.5:1 | 信息提示、帮助文本 |

#### 中性色 - Neutral Palette

| 层级 | 色值 | 使用场景 |
|------|------|---------|
| Neutral-900 | `#171717` | 正文标题、重要文字 |
| Neutral-700 | `#404040` | 正文内容、段落文字 |
| Neutral-500 | `#737373` | 辅助文字、标签文字、时间戳 |
| Neutral-300 | `#d4d4d4` | 边框、分隔线 |
| Neutral-100 | `#f5f5f5` | 背景色、卡片背景 |
| Neutral-0 | `#ffffff` | 主背景、白色区域 |

### 2.2 深色模式 Dark Mode

**深色模式设计原则：**
- **不要简单反转**：深色模式不是浅色模式的颜色反转，而是独立的色彩体系
- **降低对比度**：深色模式文字对比度应略低于浅色模式，避免视觉刺眼
- **保持品牌识别**：深色模式下品牌色（蓝色）保持不变，仅调整背景和文字色

#### 深色模式色彩映射

| 元素 | 浅色模式 | 深色模式 |
|------|---------|---------|
| 背景 | `#ffffff` | `#0a0a0a` |
| 卡片背景 | `#f5f5f5` | `#171717` |
| 主标题文字 | `#171717` | `#e5e5e5` |
| 正文文字 | `#404040` | `#a3a3a3` |
| 辅助文字 | `#737373` | `#737373` |
| 边框 | `#d4d4d4` | `#262626` |
| 强调色（蓝） | `#2563eb` | `#2563eb`（保持） |

## 3. 字体系统

### 3.1 字体家族

**主字体选择原则：**
- **无衬线字体**：符合现代简约风格，屏幕阅读清晰
- **清晰可读**：字重变化丰富，适合不同层级文字
- **中文友好**：支持中文显示，字距合理

#### 推荐字体组合

| 用途 | 字体名称 | 使用场景 | 字重 |
|------|---------|---------|------|
| 主标题 | **Inter** | 页面标题、文章标题 | 600 (Semibold) |
| 正文内容 | **Inter** | 文章正文、评论内容 | 400 (Regular) |
| 中文标题 | **思源黑体** / 系统默认 | 中文标题补充 | 500 (Medium) |
| 中文正文 | **思源黑体** / 系统默认 | 中文正文补充 | 400 (Regular) |
| 代码块 | **JetBrains Mono** | 代码内容、技术文章 | 400 (Regular) |

**字体加载策略：**
- 使用 Google Fonts CDN 加载 Inter（支持中文fallback）
- 优先加载 Regular (400) 和 Semibold (600) 字重
- 代码字体仅在实际使用时加载（lazy loading）

### 3.2 字体层级

建立**一致的字体层级系统**，确保视觉秩序清晰：

| 层级 | 字号 | 行高 | 字重 | 使用场景 | 行间距 |
|------|------|------|------|---------|--------|
| H1 - 页面标题 | 48px | 1.2 | 600 | 首页Hero标题、品牌展示 | - |
| H2 - 文章标题 | 32px | 1.3 | 600 | 文章标题、区块标题 | 24px（标题下方） |
| H3 - 小标题 | 24px | 1.4 | 500 | 文章内小标题、卡片标题 | 16px（标题下方） |
| H4 - 辅助标题 | 20px | 1.4 | 500 | 评论标题、侧边栏标题 | 12px（标题下方） |
| Body - 正文 | 16px | 1.75 | 400 | 文章正文、评论正文、描述文字 | 8px（段落间） |
| Body Large | 18px | 1.75 | 400 | 文章正文（大屏）、重点内容 | 8px（段落间） |
| Small - 辅助文字 | 14px | 1.5 | 400 | 标签、时间戳、提示文字 | 4px（元素间） |
| XSmall - 微文字 | 12px | 1.5 | 400 | 极小标签、版权文字、计数器 | 4px（元素间） |

**关键设计要点：**
- **正文最小字号 16px**：避免 iOS 自动缩放，确保阅读舒适
- **行高 1.75**：文章正文行高较高，优化阅读体验
- **层级差异明显**：相邻层级字号差异至少 4px，确保视觉区分

### 3.3 文字排版规范

#### 文章正文排版

| 属性 | 规范 | 说明 |
|------|------|------|
| 最大行宽 | 720px (桌面) / 100% (移动) | 确保阅读舒适，避免视线过长移动 |
| 段落间距 | 16px | 清晰区分段落 |
| 首行缩进 | 0 | 现代风格不使用首行缩进 |
| 字间距 | 0 (默认) | 不调整默认字间距 |
| 对齐方式 | 左对齐 | 不使用两端对齐（避免词间距不均） |

#### 标题与正文间距

| 组合 | 上间距 | 下间距 | 说明 |
|------|--------|--------|------|
| H1 → Body | - | 32px | 页面标题下方大量留白 |
| H2 → Body | - | 24px | 文章标题下方留白 |
| H3 → Body | - | 16px | 小标题下方间距 |
| Body → H3 | 24px | - | 正文到小标题的间距较大 |
| Body → Body | 16px | - | 段落间距 |

## 5. 圆角与边框

### 5.1 圆角规范

**简约现代风格圆角原则：**
- **极小圆角或无圆角**：体现简洁、硬朗的视觉特征
- **一致性**：同一层级元素使用相同圆角值
- **层级递减**：容器圆角 > 组件圆角 > 元素圆角

| 元素类型 | 圆角值 | 使用场景 |
|---------|--------|---------|
| 页面容器 | 0px | 无圆角，体现简洁硬朗 |
| 卡片 | 4px | 极小圆角，轻微柔和 |
| 按钮 | 4px | 与卡片圆角一致 |
| 输入框 | 4px | 与按钮圆角一致 |
| 图片 | 0px | 无圆角，强调内容本身 |
| 标签 | 2px | 极小圆角，标签形态 |
| 头像 | 50% | 圆形，社交元素特例 |

### 5.2 边框规范

| 元素类型 | 边框宽度 | 颜色 | 使用场景 |
|---------|---------|------|---------|
| 卡片边框 | 1px | Neutral-300 | 分隔卡片与背景 |
| 输入框边框 | 1px | Neutral-300 | 默认状态 |
| 输入框聚焦 | 2px | Primary Accent | 聚焦状态强调 |
| 分隔线 | 1px | Neutral-300 | 内容分隔 |
| 导航分隔 | 1px | Neutral-300 | 导航区域分隔 |

## 6. 阴影与层次

### 6.1 阴影系统

**简约现代风格阴影原则：**
- **极简阴影或无阴影**：使用边框而非阴影区分层次
- **功能性阴影**：仅在浮层、模态框等需要视觉分离的场景使用
- **克制深度**：阴影不应过于明显，避免视觉干扰

| 层级 | 阴影参数 | 使用场景 |
|------|---------|---------|
| 无阴影 | - | 卡片、按钮、常规元素（使用边框分隔） |
| 浮层阴影 | `0 4px 12px rgba(0, 0, 0, 0.08)` | 下拉菜单、Tooltip |
| 模态阴影 | `0 8px 24px rgba(0, 0, 0, 0.12)` | 模态框、Dialog |
| 固定栏阴影 | `0 2px 8px rgba(0, 0, 0, 0.04)` | 固定导航栏下方阴影 |

**阴影参数解析：**
- 使用 `rgba(0, 0, 0, 0.08)` 而非纯黑阴影
- 阴影扩散范围较小（12px/24px）
- Y轴偏移为主，X轴偏移为0

### 6.2 Z-Index 层级

定义**明确的 Z-Index 层级系统**，避免层叠混乱：

| 层级 | Z-Index | 使用场景 |
|------|---------|---------|
| 基础层 | 0 | 页面内容、卡片、列表 |
| 固定层 | 10 | 固定导航栏、底部栏 |
| 浮层 | 20 | 下拉菜单、Tooltip、Popover |
| 模态层 | 40 | 模态框、Dialog、全屏遮罩 |
| 最高层 | 100 | Toast通知、系统级提示 |
| 顶层 | 1000 | 紧急警告、系统弹窗 |

## 8. 核心组件规范

### 8.1 导航栏 Navigation

#### 设计特征

- **极简导航**：品牌Logo + 搜索框 + 用户头像，无多余装饰
- **固定顶部**：使用 position: fixed，配合轻微阴影
- **高度固定**：64px（桌面）/ 56px（移动）

#### 响应式变化

| 屏幕 | 导航元素 | 布局 |
|------|---------|------|
| Mobile | Logo + 搜索图标 + 用户头像 | 品牌居左，搜索和用户居右 |
| Tablet | Logo + 搜索框 + 用户头像 | 搜索框展开，宽度增加 |
| Desktop | Logo + 搜索框 + 导航链接 + 用户头像 | 增加导航链接（首页/创作中心/管理） |

#### 状态设计

| 状态 | 样式 | 说明 |
|------|------|------|
| 默认 | 背景 #ffffff，边框 Neutral-300 | 静止状态 |
| 滚动后 | 背景 #ffffff，阴影浮层阴影 | 滚动后增加阴影，强化固定感 |
| 深色模式 | 背景 #0a0a0a，边框 #262626 | 深色模式配色 |

### 8.2 文章卡片 Article Card

#### 设计特征

- **内容优先**：标题 + 摘要 + 作者信息 + 时间，无装饰元素
- **清晰边界**：使用 1px 边框而非阴影
- **高度自适应**：根据内容高度自动调整，不固定高度

#### 响应式变化

| 屏幕 | 卡片布局 | 间距 |
|------|---------|------|
| Mobile | 单列，100%宽度 | 16px 间距 |
| Tablet | 双列，各50%宽度 | 16px 间距 |
| Desktop | 单列或双列，720px 或 50% | 16px 间距 |

#### 状态设计

| 状态 | 样式 | 说明 |
|------|------|------|
| 默认 | 边框 Neutral-300，背景 Neutral-100 | 静止状态 |
| Hover | 边框 Primary Accent，背景 #ffffff | 鼠标悬停（仅桌面） |
| Pressed | 边框 Primary Accent，轻微 scale(0.98) | 点击状态（移动） |

### 8.3 按钮 Button

#### 类型规范

| 类型 | 背景色 | 文字色 | 边框 | 使用场景 |
|------|--------|--------|------|---------|
| Primary | Primary Accent | #ffffff | 无 | 主要操作：发布文章、提交评论 |
| Secondary | Neutral-100 | Neutral-700 | 1px Neutral-300 | 次要操作：取消、返回 |
| Ghost | 透明 | Primary Accent | 无 | 文字链接、取消关注 |
| Danger | Error | #ffffff | 无 | 删除、危险操作 |

#### 尺寸规范

| 尺寸 | Height | Padding | Font Size | 使用场景 |
|------|--------|---------|-----------|---------|
| Large | 48px | 16px × 32px | 16px | 主要CTA、发布按钮 |
| Medium | 40px | 12px × 24px | 14px | 次要按钮、评论提交 |
| Small | 32px | 8px × 16px | 12px | 标签按钮、微型操作 |

#### 状态设计

### 8.4 输入框 Input

#### 设计特征

- **极简设计**：无边框背景，仅底部边框或全边框
- **清晰聚焦**：聚焦时边框加粗并变色
- **足够高度**：48px（确保触摸友好）

#### 类型规范

| 类型 | 高度 | 边框 | 使用场景 |

| 单行输入 | 48px | 1px Neutral-300 | 搜索框、评论输入 |
| 多行输入 | 自适应 | 1px Neutral-300 | 文章正文编辑器 |
| 富文本编辑器 | 自适应 | 1px Neutral-300，工具栏上方 | 文章发布 |

#### 状态设计

| 状态 | 边框样式 | 说明 |
|------|---------|------|

| Focused | 2px Primary Accent | 聚焦状态，边框加粗并变色 |

| Disabled | 1px Neutral-300，背景 Neutral-100 | 禁用状态，灰色背景 |

### 8.5 评论组件 Comment

#### 设计特征

- **层级清晰**：主评论和回复评论使用不同的视觉层级
- **作者信息**：头像 + 昵称 + 时间，水平排列
- **内容优先**：评论内容占据主要空间

#### 响应式变化

| 屏幕 | 评论布局 | 缩进 |
|------|---------|------|
| Mobile | 堆叠布局，回复评论缩进 16px | 回复缩进 |
| Desktop | 水平布局，回复评论缩进 32px | 回应更大缩进 |

#### 状态设计

| 状态 | 样式 | 说明 |
|------|------|------|
| 默认 | 背景 Neutral-100，边框 Neutral-300 | 默认状态 |
| Hover | 背景 #ffffff（仅桌面） | 悬停状态 |
| 用户自己的评论 | 边框 Primary Accent | 高亮用户自己的评论 |

## 9. 动效与交互

### 9.1 动效原则

**简约现代风格动效原则：**
- **功能性优先**：动效表达状态变化和因果关系，不做装饰性动效
- **克制时长**：动效时长控制在 150-300ms，避免过长等待
- **自然曲线**：使用 ease-out（进入）和 ease-in（退出）

### 9.2 核心动效规范

| 动效类型 | 时长 | 曲线 | 使用场景 |
|---------|------|------|---------|
| 状态切换 | 150ms | ease-out | 按钮Hover、开关切换 |
| 内容展开 | 200ms | ease-out | 评论展开、详情展开 |
| 页面过渡 | 300ms | ease-out | 路由切换、页面切换 |
| 模态进入 | 200ms | ease-out | 模态框进入 |
| 模态退出 | 150ms | ease-in | 模态框退出（比进入快） |
| Toast通知 | 200ms | ease-out | Toast显示和隐藏 |

### 9.3 加载状态

| 加载时长 | 反馈方式 | 说明 |
|---------|---------|------|
| < 300ms | 无需反馈 | 用户不会感知延迟 |
| 300ms - 1s | Spinner 加载图标 | 小型加载指示 |
| > 1s | Skeleton Screen | 骨架屏占位，避免布局跳动 |

**Skeleton Screen 设计：**
- 使用灰色矩形（Neutral-200）占位
- 保持与真实内容相同的布局结构
- 不使用动画 shimmer（避免视觉干扰）

### 9.4 Reduced Motion 支持

**遵循 `prefers-reduced-motion` 媒体查询：**

```css
@media (prefers-reduced-motion: reduce) {
  /* 禁用或简化所有动效 */
  * {
    animation-duration: 0.01ms !important;
    transition-duration: 0.01ms !important;
  }
}
```

## 10. 可访问性规范

### 10.1 色彩对比度

**遵循 WCAG 2.1 标准：**
- **正文文字对比度** ≥ 4.5:1（AA标准）
- **大标题对比度** ≥ 3:1
- **交互元素对比度** ≥ 4.5:1

#### 已验证的色彩对比度

| 组合 | 对比值 | 等级 |
|------|--------|------|
| 主标题 (#171717) / 白背景 (#ffffff) | 16:1 | AAA |
| 正文 (#404040) / 白背景 (#ffffff) | 8.6:1 | AAA |
| 辅助文字 (#737373) / 白背景 (#ffffff) | 4.5:1 | AA |
| Primary Accent (#2563eb) / 白背景 (#ffffff) | 4.7:1 | AA |
| 深色模式主标题 (#e5e5e5) / 深背景 (#0a0a0a) | 16:1 | AAA |
| 深色模式正文 (#a3a3a3) / 深背景 (#0a0a0a) | 8.6:1 | AAA |

### 10.2 触摸目标尺寸

**遵循 Apple HIG 和 Material Design 标准：**
- **最小触摸目标** ≥ 44×44px（iOS）/ 48×48dp（Android）
- **触摸目标间距** ≥ 8px

#### 已验证的触摸尺寸

| 元素 | 尺寸 | 是否达标 |
|------|------|---------|
| 主要按钮 | 48px × (动态宽度) | ✅ 达标 |
| 次要按钮 | 40px × (动态宽度) | ❌ 需增加 padding 至 44px |
| 导航链接 | 文字区域 ≥ 44px 高度 | ✅ 达标 |
| 评论回复按钮 | 32px | ❌ 需扩展 hit area 至 44px |
| 标签按钮 | 32px | ❌ 需扩展 hit area 至 44px |

**修正建议：**
- 次要按钮高度增加至 44px（padding: 12px × 24px → 14px × 24px）
- 评论回复按钮、标签按钮使用 `hitSlop` 或 padding 扩展触摸区域至 44px

### 10.3 焦点状态

**遵循 WCAG 2.4.7 焦点可见原则：**
- 所有交互元素必须显示清晰的焦点指示器
- 焦点指示器使用 2-4px 的 ring 或边框变化
- 焦点指示器对比度 ≥ 3:1

#### 焦点样式规范

| 元素类型 | 焦点样式 | 说明 |
|---------|---------|------|
| 按钮 | 2px Primary Accent ring | 聚焦时显示蓝色ring |
| 输入框 | 2px Primary Accent 边框 | 聚焦时边框加粗并变色 |
| 链接 | 2px Primary Accent underline | 聚焦时显示下划线 |
| 卡片 | 2px Primary Accent 边框 | 聚焦时边框变色 |

### 10.4 屏幕阅读器支持

**遵循 WCAG 4.1.2 名称、角色、值原则：**
- 所有交互元素必须有描述性的 accessibilityLabel
- 图标按钮必须有 aria-label 或 accessibilityLabel
- 图片必须有 alt 文字或 accessibilityLabel

#### 示例

| 元素 | accessibilityLabel | 说明 |
|------|---------------------|------|
| 搜索按钮 | "搜索文章" | 明确功能描述 |
| 发布文章按钮 | "发布新文章" | 明确操作描述 |
| 关注按钮 | "关注作者 {作者名}" | 包含上下文信息 |
| 用户头像 | "{用户名}的头像" | 包含用户名 |
| 文章封面图 | "文章封面：{文章标题}" | 包含文章标题 |

### 10.5 键盘导航

**遵循 WCAG 2.1.1 键盘可访问原则：**
- 所有交互元素可通过键盘访问
- Tab 顺序遵循视觉顺序（从上到下，从左到右）
- 提供键盘快捷键（如 Ctrl/Cmd + K 打开搜索）

#### 键盘快捷键建议

| 快捷键 | 功能 | 说明 |
|--------|------|------|
| Ctrl/Cmd + K | 打开搜索框 | 快速搜索 |
| Ctrl/Cmd + N | 新建文章 | 创作者快捷键 |
| Esc | 关闭模态框/搜索框 | 退出浮层 |
| Enter | 提交评论/搜索 | 确认操作 |

## 11. 图标规范

### 11.1 图标风格

**简约现代风格图标原则：**
- **线性图标**：使用 stroke 线条而非 fill 填充
- **一致粗细**：统一使用 1.5px 或 2px stroke-width
- **无装饰**：图标本身极简，无阴影、无渐变
- **SVG 格式**：矢量图标，支持多尺寸和主题切换

### 11.2 图标尺寸

| 尺寸 Token | 值 | 使用场景 |
|------------|-----|---------|
| icon-xs | 16px | 极小图标：时间图标、计数器图标 |
| icon-sm | 20px | 小图标：导航图标、辅助图标 |
| icon-md | 24px | 标准图标：主要操作图标 |
| icon-lg | 32px | 大图标：Hero图标、特色图标 |
| icon-xl | 48px | 巨大图标：Logo、装饰图标 |

### 11.3 图标色彩

| 状态 | 颜色 | 说明 |
|------|------|------|
| 默认 | Neutral-700 | 正文图标颜色 |
| 交互 | Primary Accent | 可点击图标 |
| 禁用 | Neutral-500 | 禁用状态 |
| 深色模式 | Neutral-300 | 深色模式图标颜色 |

### 11.4 推荐图标库

- **Lucide Icons**（推荐）：现代简约风格，stroke 线性图标
- **Heroicons**：Tailwind CSS 官方图标，stroke 版本
- **Feather Icons**：极简线性图标

**禁止使用 Emoji 作为图标！**

## 13. 反模式 Anti-Patterns

### 13.1 禁止的设计

| 反模式 | 问题 | 正确做法 |
|--------|------|---------|
| 使用 Emoji 作为图标 | Emoji 跨平台不一致，无法控制样式 | 使用 SVG 图标库 |
| 过度使用阴影 | 阴影过多干扰阅读，增加视觉噪音 | 仅在浮层、模态使用阴影 |
| 过多圆角 | 圆角过多显得卡通，不现代 | 使用极小圆角（4px）或无圆角 |
| 过多色彩 | 色彩过多视觉混乱 | 限制主要色彩不超过 4 种 |
| 文字过小 | < 12px 文字不可读，iOS 自动缩放 | 最小正文 16px，辅助文字 14px |
| 禁止缩放 | 禁止 viewport 缩放损害可访问性 | 不设置 user-scalable=no |
| 无焦点状态 | 无法识别当前焦点位置 | 所有交互元素显示焦点指示器 |
| 长动画 | 动画超过 500ms 等待过长 | 动画时长 150-300ms |
| 无空状态 | 空列表显示空白或错误 | 显示"暂无内容"提示和引导 |
