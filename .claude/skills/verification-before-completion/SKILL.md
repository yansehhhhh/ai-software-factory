# Verification-Before-Completion Skill

**来源**：superpowers 项目 - 用于验证完成状态。

## 核心原则

**铁律**：NO COMPLETION CLAIMS WITHOUT FRESH VERIFICATION EVIDENCE

**无新鲜验证证据，禁止声称完成。**

## 门控函数

在声称任何结果之前：

1. **IDENTIFY** — 找到证明命令
2. **RUN** — 完整运行该命令
3. **READ** — 阅读完整输出，检查退出码，统计失败数
4. **VERIFY** — 输出确认声明
5. **ONLY THEN** — 才能做出声明

## 红旗信号（立即停止）

出现以下想法时停止：
- "应该可以了"
- "可能工作正常"
- "看起来完成了"
- 在验证前表示满意
- 信任代理的成功报告而无独立验证

## 为什么重要

从 24 个失败案例中总结：
- 信任被打破
- 未定义函数被交付
- 缺失需求被交付
- 虚假完成浪费时间

## 底线

**运行命令。阅读输出。然后再声称结果。这是不可协商的。**
