## 本版范围

本版本面向 HX-Meeting 第一版 MVP，定位为独立会议室预定系统，后续再集成化学通平台。主要输出产品设计阶段文档，包括：

- 正式 PRD 主体；
- UI 设计规范；
- 业务流程图、信息架构图、页面流转图；
- 附录：术语、用户故事、WWAS、测试场景和样例数据。

## 关键确认结论

| 编号 | 事项 | 结论 |
|---|---|---|
| C-001 | 系统形态 | 独立系统，后续集成化学通 |
| C-002 | 身份体系 | 预留化学通 SSO，第一版使用本地/模拟账号 |
| C-003 | 预约粒度 | 30 分钟 |
| C-004 | 预约上限 | 最多提前 30 天，单次不超过 2 小时 |
| C-005 | 可预约时间 | 每天 08:00-22:00 |
| C-006 | 签到窗口 | 会议开始前 15 分钟至开始后 15 分钟 |
| C-007 | 未签到处理 | 自动释放剩余时段，并记录当前会议负责人 1 次违规 |
| C-008 | 违规处罚 | 自然月内累计 3 次违规，禁用预约权限 7 天 |
| C-009 | 转让规则 | 转让需接收人确认，确认后责任转移 |
| C-010 | 日程联动 | 第一版仅系统内日程，不对接化学通日程 |
| C-011 | 会议室管理 | 第一版仅支持基础信息管理 |
| C-012 | 签到权限 | 会议负责人和参会人均可扫码签到 |
| C-013 | 通知方式 | 第一版仅系统内通知/待办 |
| C-014 | 使用终端 | 响应式 Web，PC 与移动端均支持 |
| C-015 | 占用详情可见 | 普通用户仅可见预约人姓名和部门 |
| C-016 | 周期性预约 | 第一版不支持周期性预约 |
