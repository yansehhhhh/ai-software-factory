# 会议室预定系统（HX-Meeting）- 移动端 H5 UI 设计规范

## 1. 设计目标

HX-Meeting 第一版 UI 设计基准调整为**移动端 H5 优先**。本阶段不再交付 PC 设计图作为核心评审依据，后续实现应以 `UI原型/设计稿/index.html`、`UI原型/设计稿/mobile.svg`、移动端组件库和移动端交互说明为准。

移动端 H5 需服务以下核心目标：

- 快速查看首页待办、今日会议和禁用提醒；
- 快速完成“日期选择 + 会议室卡片 + 30 分钟时间段 Chip + 底部预约表单”的预约路径；
- 扫码后直接进入签到落地页，减少移动端跳转层级；
- 通过通知中心和转让确认页快速处理待办；
- 管理员通过移动入口完成会议室基础信息查看、新增和启停等轻量操作。

## 2. 设计原则

| 原则 | 移动端 H5 落点 |
|---|---|
| 移动优先 | 手机竖屏 375px/390px 为主设计视口，单列卡片和底部导航优先 |
| 待办优先 | 首页首屏突出待签到、转让待确认、禁用提醒和未读通知 |
| 触控友好 | 主按钮、时间段 Chip、导航项点击热区不小于 44px |
| 规则前置 | 30 天预约范围、30 分钟粒度、连续选择、最多 2 小时、冲突原因在提交前提示 |
| 隐私克制 | 普通用户查看占用详情时仅展示会议室、占用时间、预约人姓名和部门 |
| 结果明确 | 签到、预约、转让、释放均提供成功/失败结果反馈和下一步动作 |

## 3. 页面结构

### 3.1 移动端页面清单

| 页面/模块 | 主要用户 | H5 页面结构 | 核心动作 |
|---|---|---|---|
| 登录页 | 所有用户 | 手机登录卡片、角色入口 | 模拟登录、进入首页 |
| 首页 | 所有用户 | 顶部问候 + 待办卡片 + 快捷入口 + 今日会议 + 底部导航 | 去预约、去签到、去确认转让、进入通知 |
| 会议预约页 | 普通用户/负责人 | 日期横向选择 + 会议室卡片 + 30 分钟时间段 Chip + 底部预约表单 | 选择连续时段、提交预约、查看冲突 |
| 占用详情抽屉 | 所有用户 | 底部抽屉 | 查看预约人姓名/部门、关闭 |
| 我的预约页 | 普通用户/负责人 | 状态筛选 Tabs + 会议卡片列表 | 去签到、释放、发起转让、查看违规 |
| 扫码签到页 | 负责人/参会人 | 扫码落地卡片 + 匹配结果 + 全宽 CTA | 确认签到、查看失败原因 |
| 转让确认页 | 转让接收人 | 单卡片确认面板 | 确认接收、拒绝 |
| 通知中心 | 所有用户 | 待办优先列表 + 未读标识 | 处理待办、标记已读、跳转详情 |
| 会议室管理入口 | 管理员 | 首页管理入口 + 管理卡片列表 + 底部表单 | 新增、编辑、启用/停用、查看二维码 |

### 3.2 H5 首页信息布局

1. **顶部栏**：系统名称、当前用户/角色、通知角标；不使用 PC 侧边栏。
2. **首屏待办**：优先展示待签到、转让待确认、禁用提醒，支持直接跳转。
3. **快捷入口**：2 列卡片，包括会议预约、扫码签到、我的预约、通知中心；管理员额外显示会议室管理入口。
4. **今日会议**：纵向会议卡片，展示时间、会议室、状态与主操作。
5. **底部导航**：首页、预约、我的预约、核销、通知，最多 5 项，固定在安全区上方。

## 5. 布局规范

### 5.1 H5 容器与安全区

| 项 | 规范 |
|---|---|
| 设计基准 | 375 × 812、390 × 844 手机竖屏 |
| 页面结构 | 顶部栏 + 单列内容流 + 固定底部导航 |
| 内容边距 | 左右 16px，卡片间距 12px-16px |
| 卡片内边距 | 14px-16px |
| 底部安全区 | `padding-bottom: calc(88px + env(safe-area-inset-bottom))` |
| 底部导航 | 高度约 64px，固定在安全区上方，最多 5 项 |

### 5.2 预约页布局

移动端不展示完整横向 PC 看板，采用以下结构：

1. 日期选择：横向日期 Chip，默认今天，最多未来 30 天。
2. 会议室卡片：展示会议室名称、位置、容量、启停状态和当前占用摘要。
3. 时间段 Chip：每个 Chip 对应 30 分钟粒度，如 08:00、08:30、09:00。
4. 选段规则：仅支持同一会议室连续选择；最多连续 4 个 Chip，即 2 小时。
5. 底部预约表单：选中合法区间后从底部抽屉弹出，自动带入会议室、开始时间和结束时间。
6. 冲突提示：点击已占用或非连续/超长时段时，在当前卡片内展示原因与建议时段。

### 5.3 弹层与抽屉

| 场景 | H5 形式 |
|---|---|
| 占用详情 | 底部抽屉，仅展示会议室、时间、预约人姓名和部门 |
| 预约表单 | 底部抽屉/底部固定表单，支持向上展开 |
| 释放确认 | 底部确认面板，明确“不会记违规/会通知参会人” |
| 转让发起 | 独立页或底部抽屉，人员搜索单列展示 |
| 签到结果 | 全屏结果页或高优先级结果卡片 |

## 6. 移动端组件规范

| 组件 | 用途 | 状态/规则 |
|---|---|---|
| MobileTopBar | 页面标题、当前用户、通知角标 | 固定/普通两种形态，避免遮挡浏览器地址栏 |
| BottomNavigation | 顶级入口切换 | 首页、预约、我的预约、核销、通知；管理员入口不强塞底栏 |
| QuickActionCard | 首页快捷入口 | 2 列卡片，图标/标题/说明/角标 |
| MeetingCard | 我的预约、今日会议、参会会议 | 待签到、已签到、违规释放、转让待确认等状态 |
| RoomCard | 预约页会议室容器 | 启用/停用、容量、位置、占用摘要、时间段区 |
| TimeSlotChip | 30 分钟时间段选择 | 空闲、选中、已占用、禁用、冲突；最小触控区 44px |
| StatusTag | 业务状态标签 | 颜色 + 文字，禁止只靠颜色表达 |
| MobileFormItem | 表单项 | 单列展示，错误提示贴近字段 |
| BottomSheet | 占用详情、预约表单、释放确认 | 支持关闭、主按钮固定底部 |
| ConfirmPanel | 转让确认、释放确认 | 清晰展示责任变化和风险 |
| Toast/Result | 成功、失败、规则提示 | 给出下一步动作，如“查看我的预约” |

## 7. 关键交互说明

### 7.1 创建预约

1. 用户从首页或底部导航进入“预约”。
2. 选择日期，系统展示该日期的会议室卡片。
3. 用户在某个会议室卡片内点击 30 分钟时间段 Chip。
4. 系统只允许在同一会议室内选择连续 Chip，最多 4 个（2 小时）。
5. 合法区间形成后，底部预约表单自动带入会议室、开始时间和结束时间。
6. 用户补充会议主题、参会人后提交。
7. 成功后显示结果卡片并提供“查看我的预约”；失败时展示冲突原因、禁用原因或规则提示。

### 7.2 扫码签到

1. 用户扫描会议室二维码或从首页待办点击“去签到”。
2. H5 直接进入签到落地页，展示二维码识别结果、会议匹配结果和签到窗口。
3. 命中唯一待签到会议时，主按钮为“确认签到”。
4. 成功后进入结果页/结果卡片，状态变更为“已签到 / 进行中”。
5. 失败时展示未到窗口、会议室不匹配、无权限、会议已释放等具体原因。

### 7.3 转让确认

1. 接收人在首页待办或通知中心看到“预约转让待确认”。
2. 点击进入转让确认页。
3. 单卡片展示会议室、时间、原负责人、责任说明和禁用校验。
4. 点击“确认接收”后责任转移；点击“拒绝”后回到通知中心。

### 7.4 管理入口

- 管理员在首页快捷入口或“更多管理”进入会议室管理；
- 移动端以会议室卡片替代表格，展示名称、位置、容量、二维码编号、启停状态；
- 新增/编辑使用底部表单，停用/启用需二次确认。

## 8. 可访问性与易用性

- 所有关键点击区域不小于 44px；
- 底部导航和底部抽屉需预留安全区；
- 状态标签必须配合文字；
- 表单错误提示靠近对应字段；
- 时间段选择需有明确选中态、不可选态和冲突态；
- 键盘弹起时底部表单字段不可被遮挡，主按钮可滚动可达；
- 释放预约、确认转让等责任变化操作必须二次确认或显式确认。

## 9. 空状态与异常状态

| 场景 | H5 文案建议 | 操作建议 |
|---|---|---|
| 无预约 | 暂无预约记录 | 去预约会议室 |
| 无通知 | 暂无通知 | 返回首页 |
| 无可用会议室 | 当前日期暂无可预约会议室 | 切换日期 |
| 时间冲突 | 该时段已被预约，请重新选择 | 查看建议时段 |
| 非连续选段 | 仅支持同一会议室连续选择 | 重新选择连续时间段 |
| 超过 2 小时 | 单次最多可预约 2 小时 | 缩短选择区间 |
| 签到失败 | 当前不在签到窗口或会议室不匹配 | 查看我的预约 |
| 权限禁用 | 您的预约权限已禁用至 {日期时间} | 查看违规记录 |
