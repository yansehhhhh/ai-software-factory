# 会议室预定系统（HX-Meeting）- 附录

## 1. 术语表

| 术语 | 定义 |
|---|---|
| HX-Meeting | 会议室预定系统，第一版为独立响应式 Web 系统，后续集成化学通。 |
| 化学通 | 未来计划集成的企业平台，可能提供账号、组织、日程、消息等能力。 |
| 会议负责人 | 当前对预约负责的用户，默认是预约创建人，转让确认后变更为接收人。 |
| 参会人 | 被添加到会议中的参与人员，可在签到窗口内扫码签到。 |
| 会议室占用看板 | 按日期、会议室和时间段展示空闲/占用状态的页面或组件。 |
| 预约粒度 | 可预约时间的最小单位，第一版为 30 分钟。 |
| 签到窗口 | 可扫码签到的时间范围：会议开始前 15 分钟至开始后 15 分钟。 |
| 自动释放 | 未在签到窗口内完成签到时，系统自动释放预约剩余时段的机制。 |
| 违规 | 预约开始后 15 分钟仍无人签到导致自动释放时，对当前负责人的记录。 |
| 预约权限禁用 | 用户自然月内累计 3 次违规后，7 天内不能创建预约或接收转让成为负责人。 |
| 主动释放 | 会议负责人主动取消预约并释放会议室，不计违规。 |
| 预约转让 | 当前负责人将预约责任转移给其他用户，需接收人确认后生效。 |
| 系统内日程 | HX-Meeting 内部生成的预约/参会记录，不等同于化学通日程。 |
| 系统内通知/待办 | HX-Meeting 内展示的消息和待处理事项，第一版不依赖短信、邮件或化学通消息。 |
| 停用会议室 | 管理员停用后的会议室，不允许创建新预约，但保留历史记录。 |

## 3. WWAS（Who / What / Acceptance / Scenario）

| 编号 | Who | What | Acceptance | Scenario |
|---|---|---|---|---|
| WWAS-001 | 普通用户 | 创建会议室预约 | 合法时间段可创建成功，冲突或违规规则被拒绝 | 用户需要为明天下午会议预约会议室 |
| WWAS-002 | 普通用户 | 查看占用详情 | 仅展示预约人姓名和部门 | 用户发现目标时段被占用，希望私下协调 |
| WWAS-003 | 参会人 | 扫码签到 | 签到窗口内扫描正确会议室二维码可成功 | 负责人临时不在，由参会人完成签到 |
| WWAS-004 | 系统任务 | 自动释放未签到预约 | 开始后 15 分钟仍无人签到时释放剩余时段并记违规 | 预约人占而不用，释放会议室给其他人 |
| WWAS-005 | 会议负责人 | 主动释放预约 | 未结束预约可被负责人释放且不记违规 | 会议取消后负责人释放会议室 |
| WWAS-006 | 会议负责人/接收人 | 转让预约 | 接收人确认后责任转移 | 原负责人出差，将会议转给同事负责 |
| WWAS-007 | 管理员 | 管理会议室 | 可新增、编辑、启用、停用会议室 | 新增一间会议室并生成二维码 |
| WWAS-008 | 被禁用用户 | 尝试预约 | 系统阻止创建预约并提示禁用截止时间 | 用户当月违规达到 3 次后再次预约 |

## 6. 后续集成扩展备忘

| 方向 | 第一版处理 | 后续可能扩展 |
|---|---|---|
| 化学通账号 | 预留用户字段 | SSO 登录、组织架构同步 |
| 化学通日程 | 仅系统内日程 | 预约成功后同步主持人和参会人日程 |
| 化学通建日程预约会议室 | 不支持 | 在化学通建日程时调用 HX-Meeting 可用会议室接口 |
| 通知渠道 | 系统内通知 | 邮件、短信、化学通消息 |
| 会议室运营 | 基础信息管理 | 设备标签、开放时间、维护时段、使用统计 |
