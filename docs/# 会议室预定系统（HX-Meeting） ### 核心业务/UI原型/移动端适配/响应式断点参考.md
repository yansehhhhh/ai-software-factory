# HX-Meeting 移动端 H5 适配参考

## 1. 适配目标

HX-Meeting 第一版 UI 以**移动端 H5**为设计和评审基准，重点适配手机浏览器、企业微信/微信 WebView 或类似移动 WebView 环境。

本阶段不再以 PC 断点、桌面侧边栏或桌面横向看板作为主要展示内容。保留“响应式断点参考”文件名仅为兼容既有产物路径，正文聚焦 H5 手机宽度、安全区、触控热区、键盘弹起、底部导航和 WebView 适配。

## 2. 推荐视口

| 视口 | 用途 | 适配重点 |
|---|---|---|
| 360 × 780 | 小屏 Android WebView | 内容压缩、按钮换行、底部安全区 |
| 375 × 812 | iPhone 常见 H5 场景 | 主设计基准、首页待办、扫码签到 |
| 390 × 844 | 大屏手机 | 卡片间距、底部抽屉、预约表单 |
| 414 × 896 | 大屏手机/微信环境 | 长列表滚动、键盘弹起 |
| 768 × 1024 | 平板/横屏补充 | 可作为双列增强，不是核心交付 |

## 3. H5 页面级适配策略

### 3.1 首页

- 顶部栏展示系统名称、用户和通知角标；
- 首屏优先展示待签到、转让待确认、禁用提醒；
- 快捷入口使用 2 列卡片；
- 今日会议使用纵向卡片；
- 管理员入口以首页卡片出现，不强塞进底部导航。

### 3.2 会议预约页

- 日期选择使用横向滚动 DateStrip；
- 会议室以单列 RoomCard 展示；
- 每个时间段 Chip 对应 30 分钟；
- 仅支持同一会议室连续选择，最多 4 个 Chip（2 小时）；
- 合法区间形成后展示底部预约表单；
- 冲突、非连续、超长在当前会议室卡片内提示。

### 3.3 我的预约 / 通知中心

- 使用状态 Tabs 或筛选 Chip；
- 会议和通知均以卡片列表呈现；
- 主操作按钮靠近卡片底部，优先展示“去签到”“去确认”；
- 次操作如释放、转让可并排或折叠。

### 3.4 扫码签到页

- 扫码后直接进入签到落地页；
- 首屏展示二维码识别结果、会议室、命中会议、签到窗口；
- 全宽主按钮“确认签到”；
- 成功/失败使用结果卡片或全屏结果页。

### 3.5 转让确认页

- 单卡片展示会议室、时间、原负责人和责任说明；
- “确认接收”和“拒绝”按钮清晰分组；
- 禁用期用户不可确认接收新的转让责任。

### 3.6 会议室管理入口

- 管理员从首页入口进入；
- 会议室以卡片呈现名称、位置、容量、二维码编号、启停状态；
- 新增/编辑用底部表单；
- 停用/启用需二次确认。

## 4. 安全区与固定元素

| 元素 | 适配要求 |
|---|---|
| 底部导航 | 使用 `padding-bottom: env(safe-area-inset-bottom)`，内容区底部预留 88px 以上 |
| 底部抽屉 | 底部按钮避开手势区，支持内容滚动 |
| 顶部栏 | 不与浏览器地址栏或 WebView 顶部胶囊冲突 |
| Toast | 不遮挡底部导航和主按钮 |
| 扫码结果页 | 主要按钮位于拇指可达区域，避免贴近手势条 |

## 5. 触控热区

- 主按钮、导航项、时间段 Chip、快捷入口卡片点击区域不小于 `44px × 44px`；
- 时间段 Chip 横纵间距至少 `8px`；
- 危险操作按钮与普通按钮保持足够距离，避免误触；
- 卡片主操作应全宽或具有明显边界；
- 状态标签不可替代可点击控件，若可点击必须具备按钮语义。

## 6. 键盘弹起与表单

- 预约表单和会议室管理表单以单列展示；
- 键盘弹起时，当前输入框和提交按钮应可滚动到可视区域；
- 底部抽屉高度可随键盘调整，必要时转为整屏表单；
- 错误提示贴近字段，不集中堆叠在页面顶部；
- 人员选择支持搜索结果滚动，已选人员以标签展示。

## 7. WebView 适配

- 避免依赖浏览器后退作为唯一返回路径，页面内需提供返回或关闭；
- 避免固定高度死布局，允许卡片和抽屉内容随字号增长；
- 弱网时显示骨架或局部加载，不阻塞底部导航；
- 扫码参数异常时展示“二维码无效/会议室不匹配”结果页；
- 使用系统字体，避免外部字体加载失败影响首屏。

## 8. 组件级适配规则

| 组件 | H5 规则 |
|---|---|
| MobileTopBar | 高度 56px 左右，标题不超过一行，右侧保留未读角标 |
| BottomNavigation | 最多 5 项，固定底部，必须有文字标签 |
| QuickActionCard | 首页 2 列，整卡可点击 |
| RoomCard | 单列展示，时间段区可换行，不横向拖拽长看板 |
| TimeSlotChip | 30 分钟粒度，选中态连续高亮，已占用置灰/标注 |
| BottomSheet | 占用详情、预约表单、释放确认、编辑会议室 |
| MeetingCard | 标题最多 2 行，状态标签靠右或标题下方 |
| AdminRoomCard | 移动管理以卡片为主，不以表格为主 |

## 9. 信息裁剪规则

| 内容 | H5 展示规则 |
|---|---|
| 欢迎语 | 保留姓名和关键待办数量 |
| 占用详情 | 仅展示会议室、时间、预约人姓名和部门 |
| 会议卡片 | 标题、会议室、时间、状态、主操作；详情进入展开/详情页 |
| 通知内容 | 摘要优先，待办按钮外露 |
| 管理信息 | 名称、位置、容量、二维码编号、状态优先 |

## 10. 验证清单

- [ ] 375px 视口下底部导航不遮挡内容；
- [ ] 预约页时间段 Chip 点击热区不小于 44px；
- [ ] 连续选择最多 4 个 Chip，超过 2 小时有提示；
- [ ] 底部预约表单键盘弹起后可继续操作；
- [ ] 扫码签到成功和失败均有明确结果反馈；
- [ ] 转让确认责任说明完整；
- [ ] 管理员会议室管理以移动卡片展示；
- [ ] `desktop.svg` 不再作为核心设计依据。
