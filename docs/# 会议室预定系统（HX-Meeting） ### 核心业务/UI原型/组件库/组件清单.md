# HX-Meeting 移动端 H5 组件清单

## 1. 组件库目标

本组件库服务于 HX-Meeting 第一版**移动端 H5**落地。组件体系不再以桌面表格、桌面看板或侧边栏导航为核心，而围绕手机竖屏、底部导航、单列卡片、触控选择和底部抽屉组织。

核心原则：

- 首页待办、预约、扫码签到、通知、转让确认和会议室管理入口可在手机上快速完成；
- 预约流程使用“日期选择 + 会议室卡片 + 30 分钟时间段 Chip + 底部预约表单”；
- 主操作触控热区不小于 `44px`；
- 状态表达必须使用颜色 + 文案；
- 普通用户占用详情仅展示预约人姓名和部门。

## 3. 导航与框架组件

### 3.1 MobileTopBar

| 项 | 说明 |
|---|---|
| 用途 | 页面标题、用户头像、未读角标 |
| 结构 | 左侧页面标题，中间可选副标题，右侧用户/通知 |
| 状态 | 普通、带未读、禁用提示 |
| 交互 | 页面切换后标题同步更新；不承载复杂二级导航 |

### 3.2 BottomNavigation

| 项 | 说明 |
|---|---|
| 用途 | 手机端顶级页面切换 |
| 推荐项 | 首页、预约、我的预约、核销、通知 |
| 状态 | 默认、选中、未读角标、禁用入口提示 |
| 交互 | 固定在底部安全区上方，最多 5 项，文字标签必须保留 |

### 3.3 QuickActionCard

| 项 | 说明 |
|---|---|
| 用途 | 首页 2 列快捷入口 |
| 示例 | 会议预约、扫码签到、我的预约、通知中心、会议室管理入口（管理员） |
| 状态 | 默认、强调、带角标、无权限隐藏 |
| 交互 | 整卡可点击，点击热区不小于 44px |

## 4. 预约业务组件

### 4.1 DateStrip

| 项 | 说明 |
|---|---|
| 用途 | 预约页选择日期 |
| 结构 | 横向滚动日期 Chip，显示今天/周几/月日 |
| 规则 | 仅展示今天至未来 30 天 |
| 状态 | 默认、选中、不可选 |

### 4.2 RoomCard

| 项 | 说明 |
|---|---|
| 用途 | 承载会议室信息与时间段选择 |
| 字段 | 会议室名称、位置、容量、二维码编号、启停状态、占用摘要 |
| 状态 | 可预约、部分占用、停用、无可用时段 |
| 交互 | 展开时间段 Chip；点击占用摘要打开底部抽屉 |

### 4.3 TimeSlotChip

| 项 | 说明 |
|---|---|
| 用途 | 30 分钟时间段选择 |
| 字段 | 时间点、状态、是否属于当前选中区间 |
| 状态 | 空闲、选中、已占用、禁用、冲突提示 |
| 规则 | 同一会议室内连续选择，最多 4 个 Chip（2 小时）；非连续或超长即时阻止 |

### 4.4 BookingBottomSheet

| 项 | 说明 |
|---|---|
| 用途 | 选中合法时间段后填写预约信息 |
| 字段 | 会议室与时间（自动带入）、会议主题、参会人 |
| 状态 | 默认、校验错误、提交中、成功、失败 |
| 交互 | 底部抽屉，可上滑展开；主按钮固定在抽屉底部或可视区域内 |

### 5.1 MeetingCard

| 项 | 说明 |
|---|---|
| 用途 | 首页今日会议、我的预约、我的参会 |
| 字段 | 会议主题、会议室、日期时间、负责人、状态、签到窗口 |

| 交互 | 主操作优先展示“去签到”；释放、转让可作为次操作 |

### 5.2 TodoCard

| 项 | 说明 |
|---|---|
| 用途 | 首页待办和通知中心待办置顶 |
| 类型 | 待签到、转让待确认、权限禁用、违规提醒 |
| 状态 | 待办、未读、已完成、失败 |
| 交互 | 主按钮直达目标页，如“去签到”“去确认” |

### 5.3 TransferConfirmPanel

| 项 | 说明 |
|---|---|
| 用途 | 转让接收人确认责任转移 |
| 字段 | 原负责人、接收人、会议室、日期时间、责任说明 |
| 状态 | 待确认、已确认、已拒绝、不可接收 |
| 交互 | 确认接收/拒绝；确认前说明签到、释放和违规责任 |

## 6. 签到与反馈组件

### 6.1 CheckInLandingCard

| 项 | 说明 |
|---|---|
| 用途 | 扫码后落地签到页 |
| 字段 | 二维码识别结果、会议室、当前用户、命中会议、签到窗口 |
| 状态 | 可签到、未到时间、超时、会议室不匹配、无权限、已释放 |
| 交互 | 命中唯一会议时展示全宽“确认签到”按钮 |

### 6.2 ResultFeedback

| 项 | 说明 |
|---|---|
| 用途 | 预约、签到、释放、转让后的结果反馈 |
| 类型 | 成功结果页、失败结果卡片、Toast、InlineAlert |
| 字段 | 结果标题、原因、下一步动作 |
| 交互 | 提供“查看我的预约”“返回首页”“重新选择时间”等恢复路径 |

### 6.3 Toast / InlineAlert

| 项 | 说明 |
|---|---|
| 用途 | 轻量提示和规则阻断 |
| 类型 | 信息、成功、警告、错误 |
| 示例 | “继续选择将超过 2 小时”“该时段已被预约”“签到成功” |
| 交互 | 靠近操作上下文展示，重要失败保留在页面内 |

## 8. 状态映射建议

| 业务状态 | 组件表现 |
|---|---|
| 空闲 | `TimeSlotChip` 浅蓝底，可点击 |
| 已选中 | `TimeSlotChip` 主色实心，区间连续高亮 |
| 已占用 | 灰蓝/浅黄标签 + 预约人姓名/部门 |
| 待签到 | `StatusTag` 黄色 + 签到窗口说明 |
| 已签到/进行中 | `StatusTag` 绿色 + 已核销说明 |
| 已取消/已释放 | 灰色标签 |
| 违规释放 | 红色标签 + 违规原因 |
| 转让待确认 | 橙色标签 + “去确认”按钮 |
| 权限禁用 | 红色 `InlineAlert` + 禁用截止时间 |

## 9. 移动端适配注意事项

- 底部导航和底部抽屉使用 `env(safe-area-inset-bottom)`；
- 所有按钮、Chip、导航项最小触控区不小于 `44px`；
- 预约表单字段单列展示，不使用桌面双列表格；
- 管理员会议室管理使用卡片，不以桌面表格作为主组件；
- `desktop.svg` 已降级为取消说明，不作为组件库评审依据。
