# HX-Meeting 移动端 H5 交互说明

## 1. 说明范围

本交互说明覆盖 HX-Meeting 第一版**移动端 H5**核心操作路径。当前阶段不再以 PC 设计图或桌面端横向看板作为核心交互依据，评审和后续实现应以移动端 H5 主原型、移动组件库、移动适配文档为准。

覆盖路径：

- H5 首页待办与底部导航；
- 移动端会议预约；
- 占用详情底部抽屉；
- 扫码签到落地页；
- 我的预约与释放/转让；
- 通知中心与转让确认；
- 管理员会议室管理入口；
- 结果反馈、错误态和空状态。

## 2. 交互设计原则

| 原则 | H5 交互落点 |
|---|---|
| 待办优先 | 首页首屏展示待签到、转让待确认、禁用提醒和未读通知 |
| 底部导航直达 | 首页、预约、我的预约、核销、通知固定在底部安全区上方 |
| 触控优先 | 主按钮、导航项、时间段 Chip 点击热区不小于 44px |
| 规则前置 | 30 天范围、30 分钟粒度、连续选择、最多 2 小时在选择阶段提示 |
| 抽屉承载轻流程 | 占用详情、预约表单、释放确认、编辑会议室使用底部抽屉或确认面板 |
| 结果可恢复 | 成功/失败均展示下一步动作，如查看我的预约、重新选择时间、返回首页 |

## 3. 页面与模块总览

| 页面/模块 | 主要交互目标 | 关键动作 |
|---|---|---|
| 首页 | 查看待办、进入核心功能 | 去签到、去预约、去确认转让、进入通知、进入管理入口 |
| 会议预约页 | 找空闲并提交预约 | 切换日期、选择会议室、选择连续时间段、展开底部预约表单 |
| 占用详情抽屉 | 查看有限占用信息 | 查看预约人姓名和部门、关闭抽屉 |
| 我的预约 | 管理负责会议 | 去签到、发起转让、释放预约、查看违规 |
| 扫码签到页 | 完成会议核销 | 识别二维码、确认签到、查看失败原因 |
| 转让确认页 | 处理责任转移 | 确认接收、拒绝 |
| 通知中心 | 处理待办和未读消息 | 处理转让、去签到、查看禁用/违规 |
| 会议室管理入口 | 管理员维护会议室 | 新增、编辑、启用/停用、查看二维码 |

## 4. 关键交互路径

### 4.1 首页 → 会议预约 → 底部表单提交

1. 用户进入 H5 首页。
2. 首页显示顶部栏、待办卡片、快捷入口、今日会议和底部导航。
3. 用户点击快捷入口或底部导航“预约”。
4. 预约页顶部展示日期横向 Chip，默认今天，可切换未来 30 天内日期。
5. 页面按单列展示会议室卡片，每张卡片显示会议室名称、位置、容量、启停状态和占用摘要。
6. 用户点击某会议室下的 30 分钟时间段 Chip。
7. 系统只允许在同一会议室内连续选择时间段，最多 4 个 Chip（2 小时）。
8. 形成合法区间后，底部预约表单展开，自动带入会议室、开始时间、结束时间。
9. 用户补充会议主题和参会人后点击“提交预约”。
10. 成功后显示结果反馈并提供“查看我的预约”；失败时在当前上下文展示冲突、禁用或规则错误。

#### 预约路径反馈要求

| 场景 | H5 反馈 |
|---|---|
| 超过未来 30 天 | 日期 Chip 不可选或提示“最多提前 30 天预约” |
| 非连续选择 | 当前会议室卡片内提示“仅支持连续时间段” |
| 超过 2 小时 | 阻止追加 Chip，提示“单次最多 2 小时” |
| 已占用时段 | Chip 置灰/标注占用，点击打开占用详情抽屉 |
| 时间冲突 | 显示被占用时段和预约人姓名/部门，并建议改选 |
| 用户禁用 | 顶部红色横幅，禁用提交预约能力 |

### 4.2 占用详情 → 底部抽屉

- 用户在会议室卡片内点击“查看占用详情”或已占用 Chip；
- 底部抽屉从屏幕底部弹出；
- 普通用户仅可见：会议室名称、占用时间、预约人姓名、预约人部门；
- 不展示会议主题和参会人；
- 用户点击关闭、蒙层或下滑返回预约页，保留日期与滚动位置。

### 4.3 首页/我的预约/扫码 → 签到结果

1. 用户从首页待办、我的预约卡片、通知提醒或扫码入口进入签到页。
2. 扫码场景直接落地 H5 签到页，不要求用户二次搜索会议。
3. 页面展示二维码识别结果、会议室、当前用户、命中会议和签到窗口。
4. 若当前时间在会议开始前 15 分钟至开始后 15 分钟内，显示全宽“确认签到”按钮。
5. 点击确认后进入成功结果卡片/结果页，状态变为“已签到 / 进行中”。
6. 若失败，展示具体原因和恢复动作。

| 失败状态 | 展示文案 | 恢复动作 |
|---|---|---|
| 未到签到时间 | 当前未到签到窗口，请稍后重试 | 返回我的预约 |
| 超过签到窗口 | 签到窗口已结束，会议可能已释放 | 查看我的预约/通知 |
| 会议室不匹配 | 当前二维码与预约会议室不一致 | 重新扫描正确二维码 |
| 无签到权限 | 当前用户不是负责人或参会人 | 联系负责人 |
| 会议已释放 | 当前会议已不可签到 | 返回首页 |

### 4.4 我的预约 → 释放确认

1. 负责人在“我的预约”会议卡片点击“释放”。
2. H5 展示底部确认面板。
3. 面板说明：释放后会议室时段立即回到可预约状态，主动释放不计违规，并通知参会人。
4. 用户二次确认后，会议状态变为“已释放/已取消”。
5. 页面显示成功 Toast 或结果卡片，并刷新会议卡片状态。

### 4.5 我的预约/通知 → 转让确认

1. 发起人从我的预约卡片点击“发起转让”，使用底部抽屉搜索接收人。
2. 系统校验接收人是否有效且不处于预约权限禁用期。
3. 接收人在首页待办或通知中心看到“预约转让待确认”。
4. 点击后进入转让确认页，单卡片展示会议室、日期时间、原负责人和责任说明。
5. 接收人点击“确认接收”后责任转移；点击“拒绝”后原负责人不变。
6. 确认/拒绝后返回通知中心并更新状态。

### 4.6 首页 → 会议室管理入口

- 管理员在首页看到“会议室管理”快捷入口；
- 进入后以会议室卡片展示名称、位置、容量、二维码编号和启停状态；
- 新增/编辑使用底部表单；
- 停用/启用需底部确认面板二次确认；
- 普通用户不显示该入口。

## 5. 系统自动交互

### 5.1 自动释放

- 会议开始后 15 分钟仍无人签到时，系统自动释放剩余时段；
- 当前负责人增加 1 次违规；
- 我的预约中该记录显示“违规释放”；
- 通知中心生成违规通知；
- 若自然月累计达到 3 次，生成权限禁用通知。

### 5.2 权限禁用

- 预约权限禁用持续 7 天；
- H5 首页、预约页、通知中心均展示禁用横幅或禁用通知；
- 禁用期用户仍可查看会议、完成签到、确认已有转让；
- 禁用期用户不可创建预约或接收新的转让责任。

## 6. 页面状态设计

| 页面 | 状态 | 展示方式 |
|---|---|---|
| 首页 | 普通 | 待办卡 + 快捷入口 + 今日会议 |
| 首页 | 管理员 | 增加会议室管理入口卡片 |
| 首页 | 禁用 | 顶部/首屏红色禁用提醒 |
| 预约页 | 默认 | 日期 Chip + 会议室卡片 + 时间段 Chip |
| 预约页 | 已选合法区间 | 底部预约表单展开 |
| 预约页 | 冲突/超长 | 卡片内 InlineAlert，阻止提交 |
| 我的预约 | 待签到 | 黄色状态 + 去签到主按钮 |
| 我的预约 | 违规释放 | 红色状态 + 查看通知 |
| 通知中心 | 待办 | 列表置顶 + 主按钮 |
| 签到页 | 成功 | 结果图标 + 查看我的预约 |
| 签到页 | 失败 | 原因说明 + 恢复动作 |

## 7. 移动端适配与无障碍

- 底部导航固定在安全区上方，内容区底部留出足够 padding；
- 底部抽屉打开时主按钮不被系统手势区遮挡；
- 键盘弹起时，预约表单字段可滚动可达；
- 所有状态使用颜色 + 文案，不只依赖颜色；
- 所有主操作按钮、时间段 Chip、导航项触控区不小于 44px；
- 页面跳转后标题同步变化，便于读屏和用户定位；
- 重要操作（释放、转让确认、停用会议室）必须显式确认。
