# 技术选型

| 层级 | 技术 | 说明 |
| --- | --- | --- |
| 前端 | Vue 3 / Vite | 快速构建交互式工作台 |
| 后端 | Spring Boot | 提供清晰 controller-service-workflow 分层 |
| Runner | Node.js | 封装 Claude Code CLI 调用和文件落盘 |
| 测试 | JUnit / Playwright | 覆盖服务逻辑和端到端流程 |
| 产物 | Markdown / PlantUML / OpenAPI / SQL | 便于审阅、版本管理和后续生成 |

## 依据
公司内部投票网页 - 架构设计 1. 架构目标 面向公司内部员工建设一个安全、易用、可扩展的投票网页系统，支持管理员创建多场景投票，员工登录后参与投票，并在投票结束后查看图表化统计结果。 核心架构目标： - 支持邮箱/工号 + 密码登录。 - 支持员工与管理员角色权限隔离。 - 支持管理员创建和管理投票。 - 支持单题、多题投票。 - 支持单选、多选、评分题、文本反馈。 - 支持实名/匿名投票模式。 - 支持员工账号 Excel 批量导入。 - 支持提交后锁定，不允许修改。 - 支持投票结束后面向所有员工公开统计结
