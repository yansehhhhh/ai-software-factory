# 接口清单

| 方法 | 路径 | 用途 | 请求 | 响应 |
| --- | --- | --- | --- | --- |
| POST | /api/requirements | 提交需求并创建任务 | 需求文本 | 任务 ID 和初始状态 |
| GET | /api/workflow/status | 查询工作流状态 | 任务 ID | 阶段、进度、Agent 状态 |
| GET | /api/artifacts | 访问生成产物 | 文件路径 | 文件内容或下载流 |
| POST | /api/tests/playwright | 触发端到端测试 | 测试用例 ID | 测试运行结果 |

## 设计依据
公司内部投票网页 - 接口设计产物 1. 接口设计目标 本接口设计基于以下业务约束： - 员工与管理员使用邮箱/工号 + 密码登录。 - 只有管理员可创建和管理投票。 - 管理员创建投票时选择实名或匿名。 - 支持单题或多题投票。 - 题型支持：单选、多选、评分题、文本反馈。 - 投票创建后立即开始，只设置结束时间。 - 所有员工都可以参与每个投票。 - 员工提交后不允许修改，提交后即锁定。 - 投票结束后所有员工可查看图表化统计结果。 - 管理员可通过 Excel 批量导入员工账号。 - 匿名投票不得暴露投票人与
