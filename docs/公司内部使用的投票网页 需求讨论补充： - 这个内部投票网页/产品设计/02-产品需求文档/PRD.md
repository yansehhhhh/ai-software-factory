# 公司内部投票网页 - 产品需求文档

## 1. 目标

### 1.1 产品目标

建设一个面向公司内部员工使用的网页投票系统，支持管理员快速创建多种投票活动，员工通过统一账号登录后参与投票，并在投票结束后查看图表化统计结果。

### 1.2 业务目标

- 降低内部意见征集、评选、满意度反馈、方案选择等投票活动的组织成本。
- 统一公司内部投票入口，减少使用外部表单或临时工具带来的数据分散与权限风险。
- 支持实名与匿名两类投票模式，兼顾追溯性与表达安全感。
- 提供结构化、图表化的结果展示，帮助管理者和员工快速理解投票结果。

### 1.3 成功指标

| 指标 | 目标 |
|---|---|
| 投票创建效率 | 管理员可在 5 分钟内完成一个标准投票活动创建 |
| 员工参与效率 | 员工登录后可在 3 步内完成投票提交 |
| 数据准确性 | 同一员工同一投票仅允许成功提交一次 |
| 结果可读性 | 结束后的投票结果以图表方式呈现核心统计信息 |
| 账号导入效率 | 支持通过 Excel 批量导入员工账号 |

## 2. 角色

| 角色 | 说明 | 核心权限 |
|---|---|---|
| 员工 | 公司内部普通用户，参与所有已发布且未结束的投票 | 登录、查看投票列表、参与投票、查看已结束投票结果 |
| 管理员 | 负责创建、管理投票和维护员工账号的内部管理角色 | 员工权限 + 创建投票、管理投票、导入员工账号、查看投票明细与统计 |
| 系统 | 投票网页及后端服务 | 身份认证、权限校验、投票提交锁定、结果统计、数据存储 |

## 3. 范围

### 3.1 本期范围

| 模块 | 范围说明 |
|---|---|
| 登录认证 | 支持邮箱/工号 + 密码登录；区分员工与管理员角色 |
| 投票列表 | 展示进行中、已结束投票；员工可进入投票或查看结果 |
| 投票创建 | 仅管理员可创建；创建后立即开始；只设置结束时间 |
| 题目配置 | 管理员可选择单题或多题；题型支持单选、多选、评分题、文本反馈 |
| 投票模式 | 管理员创建时选择实名或匿名，创建后不可随意变更 |
| 投票参与 | 所有员工均可参与每个投票；每人每个投票仅允许提交一次；提交后不允许修改 |
| 结果展示 | 投票结束后所有员工可查看图表化统计结果 |
| 投票管理 | 管理员可查看、编辑未结束投票的基础信息、提前结束或查看结果统计 |
| 员工账号管理 | 管理员通过 Excel 批量导入员工账号 |

### 3.2 非本期范围

| 模块 | 暂不包含说明 |
|---|---|
| 指定人员/部门范围 | 本期所有员工均可参与所有投票，不支持按部门或人员限制参与范围 |
| 投票提交后修改 | 本期提交后即锁定，不支持截止前修改答案 |
| 第三方单点登录 | 本期采用邮箱/工号 + 密码登录，不接入 SSO、LDAP、企业微信或飞书登录 |
| 复杂问卷逻辑 | 不支持跳题、题目分支、条件显示、随机题序 |
| 投票模板市场 | 不提供模板库或模板复用能力，可作为后续迭代 |
| 移动端原生 App | 本期提供响应式网页，不开发 iOS/Android 原生应用 |

## 4. 功能需求

### 4.1 登录与权限

#### 4.1.1 功能描述

系统应支持员工和管理员使用邮箱或工号加密码登录。登录成功后，系统根据账号角色展示对应菜单和操作权限。

#### 4.1.2 业务规则

- 邮箱或工号均可作为登录账号。
- 密码不能为空，应在服务端进行加密存储。
- 管理员可访问投票创建、投票管理、员工账号导入等管理功能。
- 员工不可访问投票创建和管理页面。
- 未登录用户访问业务页面时，应跳转至登录页。

#### 4.1.3 异常处理

- 账号不存在、密码错误时，提示“账号或密码错误”。
- 账号被禁用时，提示“账号已被禁用，请联系管理员”。
- 登录态失效时，跳转登录页并提示重新登录。

### 4.2 员工账号批量导入

#### 4.2.1 功能描述

管理员可通过上传 Excel 文件批量导入员工账号，系统解析员工信息并创建或更新员工账号。

#### 4.2.2 业务规则

- 仅管理员可执行导入。
- Excel 至少包含工号、姓名、邮箱、初始密码、角色字段。
- 工号和邮箱应具备唯一性。
- 支持导入前校验，展示成功条数、失败条数和失败原因。
- 对已存在员工账号的处理策略应明确，可采用按工号更新基础信息的方式。

#### 4.2.3 异常处理

- 文件格式不正确时，应提示仅支持 Excel 文件。
- 必填字段缺失时，应标记对应行失败并说明原因。
- 重复工号或重复邮箱应阻止导入对应行。

### 4.3 投票创建

#### 4.3.1 功能描述

管理员可创建投票活动，配置标题、说明、结束时间、实名/匿名模式、单题或多题结构，以及题目内容和选项。

#### 4.3.2 业务规则

- 只有管理员可创建投票。
- 投票创建成功后立即开始。
- 管理员只设置结束时间，不设置开始时间。
- 结束时间必须晚于当前时间。
- 投票可配置为实名或匿名，投票模式在已有提交后不可修改。
- 投票可配置为单题或多题。
- 每个投票至少包含 1 道题。
- 题型支持：单选、多选、评分题、文本反馈。
- 所有员工默认都可参与每个投票。

#### 4.3.3 题型规则

| 题型 | 配置规则 | 提交规则 | 统计规则 |
|---|---|---|---|
| 单选 | 至少 2 个选项 | 员工选择 1 个选项 | 按选项统计票数与占比 |
| 多选 | 至少 2 个选项，可设置最少/最多选择数量 | 员工选择多个选项 | 按选项统计被选择次数与占比 |
| 评分题 | 设置评分范围，如 1-5 分或 1-10 分 | 员工选择一个评分值 | 统计平均分、分布、参与人数 |
| 文本反馈 | 设置题目标题和填写提示 | 员工输入文本内容 | 统计提交人数，管理员可查看文本列表；员工结果页仅展示汇总信息 |

### 4.4 投票参与

#### 4.4.1 功能描述

员工登录后可查看所有进行中的投票，进入投票详情页填写答案并提交。

#### 4.4.2 业务规则

- 所有员工均可参与每个投票。
- 已结束投票不可提交。
- 同一员工对同一投票仅允许提交一次。
- 员工提交后不允许修改答案。
- 提交时必须完成所有必答题。
- 匿名投票下，结果展示不得暴露投票人与答案的对应关系。
- 实名投票下，管理员可在管理端查看投票人及其答案；普通员工只查看统计结果。

#### 4.4.3 异常处理

- 重复提交时，系统应拒绝并提示“你已提交该投票，无法重复提交”。
- 投票已结束时，提交按钮不可用；若提交瞬间过期，服务端应拒绝提交。
- 网络异常时，应提示用户稍后重试，避免产生重复提交。

### 4.5 投票列表与详情

#### 4.5.1 功能描述

员工和管理员可查看投票列表，根据状态进入投票详情、结果页或管理页。

#### 4.5.2 业务规则

- 投票状态至少包括：进行中、已结束。
- 进行中的投票显示标题、说明摘要、结束时间、是否已提交。
- 已结束的投票显示标题、结束时间、结果入口。
- 管理员额外可看到管理入口和提交进度。

### 4.6 投票结果展示

#### 4.6.1 功能描述

投票结束后，所有员工可查看该投票的图表化统计结果。管理员可查看更完整的统计信息和必要的明细数据。

#### 4.6.2 业务规则

- 投票未结束前，普通员工不可查看结果。
- 投票结束后，所有员工可查看统计结果。
- 单选、多选题应展示图表化选项分布。
- 评分题应展示平均分和评分分布。
- 文本反馈题对普通员工展示提交数量与可公开摘要；对管理员展示文本反馈明细。
- 匿名投票不得向任何结果查看者展示投票人与答案的映射。
- 实名投票下，管理员可查看投票人答案明细，普通员工仍只看汇总结果。

### 4.7 投票管理

#### 4.7.1 功能描述

管理员可管理自己或全部管理员创建的投票活动，包括查看状态、查看提交进度、提前结束、查看结果。

#### 4.7.2 业务规则

- 管理员可查看所有投票活动。
- 管理员可提前结束进行中的投票。
- 已有提交记录的投票，不应允许修改题目结构、实名匿名模式和题型。
- 已结束投票不可重新开启，除非后续迭代明确支持。
- 删除投票如需支持，应避免误删统计数据；本期建议先不提供物理删除，仅保留归档或隐藏作为后续能力。

## 5. 非功能需求

### 5.1 安全要求

- 密码必须加密存储，不得明文保存。
- 登录接口应具备基础防暴力破解能力，例如失败次数限制或验证码策略。
- 所有管理接口必须进行管理员权限校验。
- 匿名投票必须从产品和数据访问层面避免泄露投票人与答案映射。
- Excel 导入应校验文件类型和内容，避免恶意文件风险。

### 5.2 性能要求

- 常规列表页在 1000 个投票活动以内应在 2 秒内完成加载。
- 单个投票在 5000 名员工参与规模下，结果统计页应在 3 秒内返回核心统计数据。
- Excel 导入 5000 条员工数据时，应提供进度或结果反馈，不应导致页面无响应。

### 5.3 可用性要求

- 员工投票流程应简洁，减少干扰项。
- 提交前应提供确认提示，明确“提交后不可修改”。
- 管理员创建投票时，应对关键配置提供即时校验和说明。
- 图表应具备清晰标签、数值和占比。

### 5.4 兼容性要求

- 支持主流桌面浏览器最近两个大版本：Chrome、Edge、Safari、Firefox。
- 支持移动端浏览器访问核心参与与结果查看流程。
- 页面应具备响应式布局，适配桌面、平板和手机宽度。

### 5.5 可维护性要求

- 题型应采用可扩展设计，便于后续增加排序题、矩阵题等类型。
- 统计逻辑应与展示层解耦，便于未来支持导出或更多图表。
- 权限判断应集中封装，避免散落在页面逻辑中。

## 6. 验收标准

### 6.1 登录与权限验收

- Given 未登录用户访问投票列表，When 页面加载，Then 系统跳转至登录页。
- Given 员工账号登录成功，When 访问投票创建页，Then 系统拒绝访问并提示无权限。
- Given 管理员账号登录成功，When 访问员工导入页，Then 系统允许进入并显示上传入口。

### 6.2 员工导入验收

- Given 管理员上传符合模板的 Excel，When 系统解析完成，Then 成功创建或更新员工账号并展示导入结果。
- Given Excel 中存在缺失工号的行，When 执行导入，Then 该行导入失败并展示失败原因。
- Given 非管理员用户，When 请求导入接口，Then 系统拒绝操作。

### 6.3 投票创建验收

- Given 管理员填写标题、结束时间、题目和选项，When 点击创建，Then 系统创建投票并立即置为进行中。
- Given 管理员设置的结束时间早于当前时间，When 点击创建，Then 系统阻止提交并提示结束时间必须晚于当前时间。
- Given 管理员创建匿名投票，When 员工提交答案，Then 结果数据不得向任何页面暴露投票人与答案映射。

### 6.4 投票参与验收

- Given 员工首次进入进行中的投票，When 填写必答题并提交，Then 系统保存答案并标记该员工已提交。
- Given 员工已提交某投票，When 再次进入该投票，Then 系统显示已提交状态且不可再次提交。
- Given 员工在结束时间之后提交，When 请求到达服务端，Then 系统拒绝提交并提示投票已结束。

### 6.5 结果查看验收

- Given 投票仍在进行中，When 普通员工尝试查看结果，Then 系统不展示结果并提示结束后可查看。
- Given 投票已结束，When 员工进入结果页，Then 系统展示图表化统计结果。
- Given 实名投票已结束，When 管理员查看结果明细，Then 系统可展示投票人及答案明细。
- Given 匿名投票已结束，When 管理员查看结果，Then 系统只展示汇总统计，不展示投票人答案对应关系。

## 7. 约束

- 本期必须使用邮箱/工号 + 密码登录，不引入第三方登录。
- 本期投票创建后立即开始，只配置结束时间。
- 本期所有员工都能参与每个投票，不做部门或指定人员范围控制。
- 本期员工提交后不允许修改答案。
- 本期结果仅在投票结束后对所有员工公开。
- PRD 主体不承载完整 UI 设计规范、完整流程图源码、术语全集、用户故事全集、测试场景全集或样例数据全集；这些内容拆分至对应文件。

|---|---|---|---|
| Q1 | 管理员是否分为超级管理员和普通管理员？ | 影响账号管理与投票管理权限边界 | 本期先采用单一管理员角色 |
| Q2 | Excel 导入账号时，初始密码如何通知员工？ | 影响账号启用流程与安全策略 | 本期在导入结果中生成初始密码状态，通知方式后续确认 |

| Q6 | 密码找回和重置由谁执行？ | 影响账号管理闭环 | 本期建议管理员重置密码 |
