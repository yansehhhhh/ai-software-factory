## 3. 本次交付范围

本次交付包括以下产品设计产物：

1. 产品需求文档主体：目标、角色、范围、功能需求、非功能需求、验收标准、约束与待确认问题。

4. 附录：术语、用户故事、WWAS、测试场景、样例数据。

## 4. 需求来源摘要

需求来自用户输入与需求讨论补充：

- 产品目标：做一个公司内部使用的投票网页。
- 投票场景：支持多种内部投票场景。
- 权限：只有管理员可创建和管理投票；员工参与投票。
- 登录：员工与管理员使用邮箱/工号加密码登录。
- 投票设置：创建后立即开始，只设置结束时间。
- 投票形式：支持单题或多题；题型包括单选、多选、评分题、文本反馈。
- 实名性：管理员创建投票时选择实名或匿名。
- 参与范围：所有员工都可以参与每个投票。
- 提交规则：提交后不允许修改，答案锁定。
- 结果公开：投票结束后所有员工可查看图表化统计结果。
- 账号管理：通过 Excel 批量导入员工账号。
