# 公司内部投票网页 - UI 设计规范

## 1. 设计定位

公司内部投票网页应体现“清晰、高效、可信、低学习成本”的设计定位。产品主要用于员工参与投票和管理员创建管理投票，因此界面应兼顾普通员工的快速参与体验与管理员的配置效率。

## 2. 设计原则

### 2.1 清晰优先

- 员工端突出“待投票、已提交、已结束、查看结果”等关键状态。
- 管理端突出创建进度、题目配置、结束时间和实名/匿名等不可轻易修改的关键字段。
- 结果页优先展示图表和核心指标，减少复杂表格对普通员工的干扰。

### 2.2 操作可预期

- 投票提交前必须明确提示“提交后不可修改”。
- 已结束投票和已提交投票应禁用提交入口，并说明原因。
- 创建投票时对结束时间、题目选项数量、必填字段进行即时校验。

### 2.3 内部系统可信感

- 采用简洁、稳定的企业级视觉语言。
- 避免过度娱乐化样式，确保正式投票场景可用。
- 对匿名投票明确展示隐私提示，增强员工信任。

### 2.4 响应式可用

- 桌面端适合管理员配置和结果分析。
- 移动端优先保障员工参与投票和查看结果。

## 3. 页面结构

### 3.1 登录页

| 区域 | 内容 |
|---|---|
| 品牌区 | 公司名称或系统名称“内部投票系统” |
| 登录表单 | 邮箱/工号输入框、密码输入框、登录按钮 |
| 辅助信息 | 错误提示、忘记密码提示或联系管理员说明 |

### 3.2 员工投票列表页

| 区域 | 内容 |
|---|---|
| 顶部导航 | 系统名称、当前用户、退出登录 |
| 筛选区 | 状态筛选：全部、进行中、已结束、我已提交 |
| 投票卡片列表 | 标题、说明摘要、状态、结束时间、是否已提交、操作按钮 |
| 空状态 | 无投票时展示说明文案 |

### 3.3 投票详情页

| 区域 | 内容 |
|---|---|
| 投票头部 | 标题、说明、结束时间、实名/匿名标识 |
| 隐私提示 | 匿名投票提示“系统不会展示投票人与答案对应关系”；实名投票提示“管理员可查看提交明细” |
| 题目区 | 单选、多选、评分、文本反馈题按顺序展示 |
| 提交区 | 提交按钮、提交后不可修改提示 |
| 确认弹窗 | 二次确认提交 |

### 3.4 投票结果页

| 区域 | 内容 |
|---|---|
| 结果概览 | 参与人数、总员工数、参与率、结束时间 |
| 图表区 | 各题统计图表 |
| 文本反馈区 | 普通员工展示汇总，管理员展示明细入口 |
| 权限提示 | 未结束时提示结果暂不可见 |

### 3.5 管理员工作台

| 区域 | 内容 |
|---|---|
| 侧边导航 | 投票管理、创建投票、员工账号导入 |
| 概览卡片 | 进行中投票数、已结束投票数、总参与次数 |
| 最近投票 | 最近创建投票、状态、参与进度、管理入口 |

### 3.6 创建投票页

| 区域 | 内容 |
|---|---|
| 基础信息 | 标题、说明、结束时间 |
| 投票设置 | 实名/匿名、单题/多题 |
| 题目编辑器 | 题型选择、题干、选项、评分范围、文本提示 |
| 预览区 | 显示员工端投票预览 |
| 操作区 | 创建投票、取消 |

### 3.7 员工账号导入页

| 区域 | 内容 |
|---|---|
| 模板下载 | Excel 模板下载入口 |
| 文件上传 | 拖拽上传或点击上传 |
| 校验结果 | 总行数、成功数、失败数、失败原因 |
| 导入结果 | 成功提示、失败明细下载或展示 |

## 4. 视觉规范

| 成功色 | `#16A34A` | 已提交、导入成功、进行完成 |
| 警告色 | `#F59E0B` | 截止提醒、重要提示 |
| 错误色 | `#DC2626` | 表单错误、导入失败、权限拒绝 |
| 信息色 | `#0EA5E9` | 说明提示、匿名/实名提示 |
| 文字主色 | `#111827` | 一级文本、标题 |
| 文字次色 | `#4B5563` | 正文、说明 |
| 文字弱色 | `#9CA3AF` | 占位符、辅助信息 |
| 边框色 | `#E5E7EB` | 分割线、输入框边框 |
| 页面背景 | `#F9FAFB` | 页面底色 |

## 5. 组件规范

### 5.1 按钮

| 状态 | 样式 | 使用场景 |
|---|---|---|
| Primary 默认 | 主色背景、白色文字 | 创建投票、提交投票、确认导入 |
| Primary Hover | 主色加深 `#1D4ED8` | 鼠标悬停 |
| Primary Disabled | 灰色背景 `#D1D5DB`、白色文字 | 已提交、已结束、表单未完成 |
| Secondary | 白底、主色文字、主色边框 | 取消、返回、下载模板 |
| Danger | 错误色背景、白色文字 | 提前结束确认等高风险操作 |

### 5.2 表单输入

- 输入框高度：40px。
- 默认边框：`#D1D5DB`。
- 聚焦边框：主色 `#2563EB`，可配合 2px 浅色外发光。
- 错误态：边框错误色，下方展示 12px 错误说明。
- 必填项：标签右侧显示红色星号。

### 5.3 投票卡片

投票卡片用于列表页展示投票活动。

| 元素 | 规则 |
|---|---|
| 标题 | 最多两行，超出省略 |
| 状态标签 | 进行中、已结束、已提交 |
| 时间 | 展示结束时间；临近结束可用警告色提示 |
| 操作按钮 | 进行中且未提交显示“去投票”；已提交显示“已提交”；已结束显示“查看结果” |

### 5.4 题目组件

#### 单选题

- 使用 Radio 组件。
- 每个选项独立成行，支持较长文本换行。
- 选中态使用主色圆点和浅蓝背景。

#### 多选题

- 使用 Checkbox 组件。
- 可展示“最多选择 N 项”提示。
- 超过最多选择数量时阻止继续选择并提示。

#### 评分题

- 可使用数字评分按钮或星级评分。
- 企业内部场景优先使用数字评分，便于表达清晰。
- 选中评分高亮主色。

#### 文本反馈

- 使用多行文本域。
- 默认高度不少于 120px。
- 可展示字数限制与剩余字数。

### 5.5 状态标签

| 状态 | 文案 | 颜色 |
|---|---|---|
| 进行中 | 进行中 | 信息色或主色 |
| 已提交 | 已提交 | 成功色 |
| 已结束 | 已结束 | 中性色 |
| 匿名 | 匿名投票 | 信息色浅底 |
| 实名 | 实名投票 | 警告色浅底 |

### 5.6 弹窗

| 弹窗 | 触发时机 | 内容要求 |
|---|---|---|
| 提交确认弹窗 | 员工点击提交投票 | 明确提示“提交后不可修改”，提供取消和确认提交 |
| 提前结束确认弹窗 | 管理员点击提前结束 | 提示结束后所有员工可查看结果，提供取消和确认结束 |
| 导入结果弹窗 | 员工导入完成 | 展示成功、失败统计和失败原因入口 |

### 5.7 图表组件

| 题型 | 推荐图表 | 展示内容 |
|---|---|---|
| 单选 | 横向条形图 / 饼图 | 选项、票数、占比 |
| 多选 | 横向条形图 | 选项、被选择次数、占比 |
| 评分题 | 柱状图 + 平均分卡片 | 分值分布、平均分、参与人数 |
| 文本反馈 | 指标卡 + 列表 | 提交数量、文本明细或摘要 |

## 6. 交互状态

### 6.1 投票提交状态

| 状态 | 页面表现 |
|---|---|
| 未提交且进行中 | 表单可填写，提交按钮可用 |
| 未提交但已结束 | 表单只读，提交按钮禁用，提示已结束 |
| 已提交且进行中 | 展示已提交提示，表单只读，不允许修改 |
| 已提交且已结束 | 可进入结果页，表单只读 |

### 6.2 创建投票校验状态

| 字段 | 校验规则 | 错误提示示例 |
|---|---|---|
| 标题 | 必填，建议 2-50 字 | 请输入投票标题 |
| 结束时间 | 必填，晚于当前时间 | 结束时间必须晚于当前时间 |
| 题目 | 至少 1 道 | 请至少添加一道题目 |
| 单选/多选选项 | 至少 2 个 | 请至少添加两个选项 |
| 评分范围 | 最小值小于最大值 | 请设置有效评分范围 |

### 6.3 加载与空状态

- 列表加载：使用骨架屏或轻量 Loading。
- 无投票：显示空状态插画或图标 + “暂无投票”。
- 无结果：投票未结束时说明“投票结束后可查看结果”。
- 导入中：展示上传进度或解析中状态。

## 7. 响应式要求

### 7.1 断点设置

| 断点 | 宽度 | 布局策略 |
|---|---|---|
| Mobile | `< 768px` | 单列布局，隐藏复杂侧边栏，按钮全宽优先 |
| Tablet | `768px - 1199px` | 内容区单列或双列混合，导航可折叠 |
| Desktop | `>= 1200px` | 管理端侧边栏 + 主内容区，结果图表可多列展示 |

### 7.2 移动端适配

- 投票详情页题目采用单列纵向排列。
- 提交按钮可固定在底部，但需避免遮挡题目内容。
- 图表需支持横向滚动或自适应缩放。
- 文本反馈输入框在移动端保持足够高度。

### 7.3 桌面端适配

- 管理端创建投票页可采用左侧表单、右侧预览布局。
- 结果页可采用概览卡片 + 多图表网格布局。
- 列表页支持更丰富的筛选和状态信息。

## 8. 可访问性要求

- 所有表单控件必须具备明确 label。
- 错误提示不能只依赖颜色表达。
- 图表需展示数值和文字标签，不能仅依赖颜色区分。
- 键盘可操作核心表单和按钮。
- 文本与背景对比度应满足 WCAG AA 基础要求。

## 9. 页面文案规范

| 场景 | 推荐文案 |
|---|---|
| 匿名提示 | 本投票为匿名投票，结果中不会展示投票人与答案的对应关系。 |
| 实名提示 | 本投票为实名投票，管理员可查看你的提交明细。 |
| 提交确认 | 提交后不可修改，请确认你的答案无误。 |
| 已提交提示 | 你已提交该投票，无法再次提交或修改。 |
| 未结束结果提示 | 投票结束后，所有员工可查看统计结果。 |
| 导入成功 | 员工账号导入完成，请查看导入结果。 |
