# 公司内部投票网页 - 响应式断点参考

## 1. 适配目标

本响应式设计用于公司内部投票网页的 Web 原型，重点保障：

- 员工在手机上快速查看投票、填写题目并提交。
- 员工在手机上查看已结束投票的图表化结果。
- 管理员在桌面端高效创建、管理投票和导入员工账号。
- 平板端在员工参与和轻量管理场景下保持可用。

## 2. 断点定义

| 断点 | 宽度范围 | 主要设备 | 布局策略 |
|---|---|---|---|
| Mobile S | `320px - 374px` | 小屏手机 | 单列布局，压缩非关键说明，按钮全宽 |
| Mobile M/L | `375px - 767px` | 主流手机 | 单列布局，底部固定主操作，图表单列 |
| Tablet | `768px - 1199px` | 平板、小屏笔记本 | 内容单列或双列混合，导航可折叠 |
| Desktop | `>= 1200px` | 桌面浏览器 | 管理端侧边栏 + 主内容，多列卡片与图表 |

与上一阶段 UI 设计规范保持一致：

- Mobile：`< 768px`
- Tablet：`768px - 1199px`
- Desktop：`>= 1200px`

## 3. 全局响应式规则

### 3.1 页面容器

| 设备 | 规则 |
|---|---|
| Mobile | 页面左右边距 16px，主内容单列，底部预留固定按钮或底部导航空间 |
| Tablet | 页面左右边距 24px，可使用 2 列信息卡片 |
| Desktop | 页面左右边距 24-32px，最大宽度 1280px，管理端展示侧边栏 |

### 3.2 导航

| 设备 | 员工端 | 管理端 |
|---|---|---|
| Mobile | 顶部标题 + 底部导航，最多 3-4 个核心入口 | 顶部返回 + 折叠菜单，复杂管理优先桌面使用 |
| Tablet | 顶部导航，可折叠侧边栏 | 可折叠侧边栏或顶部菜单 |
| Desktop | 顶部导航 + 页面内筛选 | 固定侧边栏 + 顶部用户信息 |

### 3.3 字体

| 层级 | Mobile | Tablet/Desktop |
|---|---|---|
| 页面标题 | 22px / 30px | 24px / 32px |
| 区块标题 | 17-18px / 28px | 18px / 28px |
| 正文 | 14-16px / 22-24px | 14px / 22px |
| 辅助说明 | 12px / 18px | 12px / 18px |
| 数据指标 | 24-26px / 34px | 28px / 36px |

注意：移动端正文输入控件建议不小于 16px，避免 iOS 自动放大。

## 4. 页面级适配

### 4.1 登录页

Mobile：

- 登录卡片宽度为 `calc(100% - 32px)`。
- 表单垂直排列。
- 登录按钮全宽，最小高度 44px。
- 错误提示显示在密码字段下方。

Desktop：

- 登录卡片最大宽度 440px。
- 居中显示，可使用浅蓝渐变背景增强可信感。

### 4.2 员工投票列表

Mobile：

- 投票卡片单列展示。
- 筛选 Tabs 横向排列，超出可横向滚动。
- 投票卡片内操作按钮靠下，全宽或半宽。
- 状态标签必须换行后仍清晰可读。

Desktop：

- 投票卡片可单列宽卡或双列卡片。
- 顶部筛选区与页面标题并列。
- 可展示更多摘要与截止时间。

### 4.3 投票详情

Mobile：

- 题目单列纵向排列。
- 单选/多选选项高度不小于 44px。
- 评分按钮为 44px 正方形，支持换行。
- 文本反馈框高度不小于 120px。
- 提交按钮固定在底部，内容区底部增加 88px padding，避免遮挡。

Desktop：

- 题目区域可居中显示，最大宽度 800-900px。
- 提交区可 sticky 在底部或页面末尾。
- 隐私提示靠近标题区。

### 4.4 创建投票页

Mobile：

- 不强制完整复杂编辑体验，建议按区块折叠：基础信息、投票设置、题目编辑、预览。
- 预览区放在题目编辑之后。
- 创建按钮全宽。

Tablet：

- 基础信息与题目编辑单列，预览可折叠。

Desktop：

- 左侧表单编辑区，右侧员工端预览区。
- 表单区宽度约 60%，预览区约 40%。
- 侧边栏固定显示。

### 4.5 投票结果页

Mobile：

- 指标卡片 2 列或横向滚动，小屏可单列。
- 图表采用横向条形图，减少轴标签拥挤。
- 每个图表下方展示文字摘要。
- 文本反馈仅展示汇总和主题摘要。

Desktop：

- 指标卡片 4 列展示。
- 图表 2 列网格展示。
- 管理员明细可通过表格展示。

### 4.6 员工账号导入页

Mobile：

- 上传区单列。
- 失败原因表格可改为卡片列表。
- 文件选择按钮全宽。

Desktop：

- 上传区和结果区双列展示。
- 失败原因使用表格展示。

## 5. 触摸与可访问性

- 移动端所有按钮、选项、评分按钮触摸区域不小于 44px。
- 相邻触摸目标间距不小于 8px。
- 不依赖 hover 才能显示关键信息。
- 底部固定按钮需避开系统手势区域。
- 表单错误不只用红色表达，必须显示文字。
- 图表不只用颜色表达，需要标签、数值和占比。

## 6. 图表响应式规则

| 图表 | Mobile | Desktop |
|---|---|---|
| 单选题统计 | 横向条形图，标签换行，显示票数/占比 | 横向条形图或饼图 |
| 多选题统计 | 横向条形图，避免多色过载 | 横向条形图 |
| 评分题统计 | 平均分卡片 + 简化柱状图 | 平均分卡片 + 完整分布柱状图 |
| 文本反馈 | 反馈数量 + 摘要卡片 | 摘要 + 管理员明细表 |

## 7. 固定区域与安全距离

| 固定元素 | Mobile 处理 |
|---|---|
| 顶部栏 | 高度 56-60px，内容从安全区下方开始 |
| 底部导航 | 高度 56-64px，正文底部增加对应 padding |
| 底部提交按钮 | 高度 64-76px，正文底部增加 88px padding |
| 弹窗 | 宽度 `calc(100% - 32px)`，底部 Sheet 或居中弹窗均可 |

## 8. 横屏与窄屏处理

- 横屏手机不应出现不可操作的固定区域遮挡。
- 小屏宽度下，表格改为卡片列表。
- 复杂筛选可折叠，不挤占首屏核心操作。
- 长标题允许换行，不强制单行省略影响理解。
