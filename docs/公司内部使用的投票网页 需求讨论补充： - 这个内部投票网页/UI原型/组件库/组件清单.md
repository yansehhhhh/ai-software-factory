# 公司内部投票网页 - 组件清单

## 1. 组件库设计依据

本组件库基于上一阶段产品设计产物生成，重点遵循：

- PRD 中的角色权限：员工、管理员、系统。
- PRD 中的核心流程：登录、投票列表、投票详情、提交锁定、结果展示、投票管理、Excel 导入。
- UI 设计规范中的企业级视觉语言：清晰、高效、可信、低学习成本。
- UI 设计规范中的颜色、字体、间距、圆角、阴影、响应式和可访问性要求。
- 流程图中的页面流转：登录页、员工投票列表、投票详情、提交确认弹窗、已提交状态、结果页、管理员工作台、创建投票页、投票管理页、员工账号导入页。

## 3. 基础组件

### 3.1 Button 按钮

| 变体 | 用途 | 状态 |
|---|---|---|
| Primary | 创建投票、提交投票、确认导入 | 默认、Hover、Focus、Disabled、Loading |
| Secondary | 返回、取消、下载模板、查看结果 | 默认、Hover、Focus、Disabled |
| Ghost | 退出、取消、返回列表 | 默认、Hover、Focus |
| Danger | 提前结束投票等高风险操作 | 默认、Hover、Focus、Disabled |

要求：

- 最小高度 40px，移动端主要操作建议 44px。
- 每个页面仅保留一个主要 CTA。
- Disabled 状态需要降低视觉强调并设置不可点击语义。

### 3.2 Form Field 表单字段

| 字段类型 | 使用位置 | 规则 |
|---|---|---|
| 文本输入 | 登录账号、投票标题 | 必须有可见 label |
| 密码输入 | 登录页 | 支持密码自动填充，后续可加入显示/隐藏密码 |
| 日期时间输入 | 创建投票结束时间 | 必填，晚于当前时间 |
| 文本域 | 投票说明、文本反馈题 | 最小高度 108-120px |
| 下拉选择 | 题型选择 | 支持单选、多选、评分、文本反馈 |

要求：

- 错误提示显示在字段下方。
- 必填字段使用星号标记。
- 错误提示必须说明原因和修正方式。

### 3.3 Status Tag 状态标签

| 状态 | 文案 | 颜色 |
|---|---|---|
| 进行中 | 进行中 | 主色浅底 + 主色文字 |
| 已提交 | 已提交 | 成功浅底 + 成功文字 |
| 已结束 | 已结束 | 中性色浅底 + 次级文字 |
| 匿名投票 | 匿名投票 | 信息浅底 + 信息文字 |
| 实名投票 | 实名投票 | 警告浅底 + 深色警告文字 |
| 不可修改 | 不可修改 | 中性色浅底 + 次级文字 |

## 4. 业务组件

### 4.1 Login Card 登录卡片

用于邮箱/工号 + 密码登录。

组成：

- 品牌标识。
- 邮箱/工号输入框。
- 密码输入框。
- 登录按钮。
- 错误提示。
- 联系管理员说明。

状态：

- 默认。
- 输入错误。
- 账号禁用。
- 登录中。

### 4.2 Vote Card 投票卡片

用于员工投票列表和管理员最近投票列表。

组成：

- 投票标题。
- 投票说明摘要。
- 状态标签。
- 实名/匿名标签。
- 结束时间。
- 主操作按钮。

操作规则：

- 进行中且未提交：显示“去投票”。
- 进行中且已提交：显示“已提交”，按钮禁用。
- 已结束：显示“查看结果”。
- 管理员列表额外显示“管理”“提前结束”。

### 4.3 Question Block 题目块

用于投票详情与创建投票预览。

支持题型：

| 题型 | 子组件 | 交互 |
|---|---|---|
| 单选题 | Radio Option | 只能选择一个选项 |
| 多选题 | Checkbox Option | 支持最少/最多选择数量限制 |
| 评分题 | Score Button Group | 数字评分按钮，选中高亮 |
| 文本反馈 | Textarea | 展示字数统计 |

状态：

- 可填写。
- 只读。
- 错误。
- 已提交锁定。

### 4.4 Privacy Notice 隐私提示

用于投票详情页顶部。

| 模式 | 文案 |
|---|---|
| 匿名投票 | 本投票为匿名投票，结果中不会展示投票人与答案的对应关系。 |
| 实名投票 | 本投票为实名投票，管理员可查看你的提交明细。 |

要求：

- 不能只通过颜色表达实名/匿名，必须显示明确文字。
- 匿名投票提示应靠近投票标题和题目区域。

### 4.5 Result Metric Card 结果指标卡

用于结果页概览。

字段：

- 参与人数。
- 总员工数。
- 参与率。
- 结束时间。
- 平均分。

要求：

- 数字使用 `font-size-metric`。
- 数据单位和说明需清晰。

### 4.6 Chart Block 图表区

用于图表化统计。

| 数据类型 | 推荐展示 |
|---|---|
| 单选题 | 横向条形图，显示票数和占比 |
| 多选题 | 横向条形图，显示被选择次数和占比 |
| 评分题 | 柱状图 + 平均分卡片 |
| 文本反馈 | 指标卡 + 摘要列表，管理员可见明细入口 |

可访问性要求：

- 图表必须有文字摘要。
- 每个数据项同时显示数值和占比。
- 不依赖颜色作为唯一含义表达。

### 4.7 Admin Sidebar 管理端侧边导航

用于桌面端管理页面。

导航项：

- 工作台。
- 创建投票。
- 投票管理。
- 员工导入。
- 结果查看。

要求：

- 当前页面高亮。
- 移动端隐藏侧边栏，切换为底部导航或顶部菜单。

### 4.8 Upload Panel Excel 上传面板

用于员工账号导入页。

组成：

- 模板下载按钮。
- 拖拽上传区域。
- 文件类型说明。
- 解析中状态。
- 导入校验结果。
- 失败原因表格。

状态：

- 默认等待上传。
- 上传中。
- 校验成功。
- 部分失败。
- 文件格式错误。

### 4.9 Confirm Modal 确认弹窗

用于提交投票和提前结束投票。

| 弹窗 | 触发 | 文案重点 |
|---|---|---|
| 提交确认 | 员工点击提交 | 提交后不可修改 |
| 提前结束确认 | 管理员点击提前结束 | 结束后所有员工可查看结果，且无法继续提交 |

要求：

- 提供取消和确认。
- 支持 Esc 关闭。
- 打开后聚焦弹窗标题或首个可交互控件。
- 危险操作使用 Danger 按钮。

### 4.10 Empty / Error State 空状态与错误状态

| 场景 | 文案 | 操作 |
|---|---|---|
| 无投票 | 当前筛选条件下没有投票活动。 | 切换筛选 / 返回全部 |
| 未结束结果不可见 | 投票结束后，所有员工可查看统计结果。 | 返回列表 |
| 登录失败 | 账号或密码错误，请检查后重试。 | 重新输入 |
| 文件格式错误 | 仅支持 Excel 文件，请上传 .xlsx 文件。 | 重新选择文件 |
| 无权限 | 当前账号无权限访问该页面。 | 返回投票列表 |

## 5. 页面级组件组合

| 页面 | 组件组合 |
|---|---|
| 登录页 | Login Card + Error State |
| 员工投票列表 | Topbar + Filter Tabs + Vote Card + Empty State |
| 投票详情 | Topbar + Privacy Notice + Question Block + Sticky Submit + Confirm Modal |
| 已提交状态 | Submitted State + Return Button |
| 结果页 | Result Metric Card + Chart Block + Text Summary + Admin Detail Entry |
| 管理员工作台 | Admin Sidebar + Metric Cards + Recent Vote Table |
| 创建投票页 | Base Info Form + Vote Setting Segment + Question Editor + Preview Panel |
| 投票管理页 | Admin Sidebar + Filter + Vote Management Table + End Confirm Modal |
| 员工账号导入页 | Template Download + Upload Panel + Import Result Table |

## 6. 可访问性检查清单

- 所有表单控件必须有可见 label。
- 所有按钮具备明确可读文案。
- 图表必须提供文字数据和摘要。
- 状态不能只依赖颜色，应配合文字标签。
- 键盘 Tab 顺序应与视觉顺序一致。
- 弹窗支持 Esc 关闭，并提供明确取消按钮。
- 移动端触摸目标不小于 44px。
- 正文与背景对比度满足 WCAG AA。
