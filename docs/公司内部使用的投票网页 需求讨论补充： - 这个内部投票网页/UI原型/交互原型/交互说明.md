# 公司内部投票网页 - 交互说明

## 1. 交互设计依据

本交互原型基于上一阶段产品设计产物继续设计，覆盖 PRD、UI 设计规范、业务流程图、信息架构图、页面流转图和术语表中的核心路径。

核心约束：

- 员工与管理员均通过邮箱/工号 + 密码登录。
- 系统根据角色进入员工端或管理员端。
- 只有管理员可创建和管理投票。
- 投票创建后立即开始，只设置结束时间。
- 所有员工都可以参与每个投票。
- 投票由管理员选择实名或匿名。
- 同一员工对同一投票只能提交一次，提交后不可修改。
- 投票结束后所有员工可查看图表化统计结果。
- 管理员通过 Excel 批量导入员工账号。

## 2. 页面与路由

| 页面 | 建议路由 | 角色 | 说明 |
|---|---|---|---|
| 登录页 | `/Login` | 未登录用户 | 邮箱/工号 + 密码登录 |
| 员工投票列表 | `/Votes` | 员工、管理员 | 查看全部、进行中、已结束、我已提交投票 |
| 投票详情页 | `/Votes/{id}` | 员工、管理员 | 查看投票规则、填写题目、提交投票 |
| 提交确认弹窗 | `SubmitConfirm` | 员工、管理员 | 二次确认“提交后不可修改” |
| 已提交状态 | `SubmittedState` | 员工、管理员 | 展示已提交，不允许重复提交或修改 |
| 投票结果页 | `/Votes/{id}/Result` | 员工、管理员 | 投票结束后查看图表化统计结果 |
| 管理员工作台 | `/Admin/Dashboard` | 管理员 | 查看投票概览、最近投票和管理入口 |

| 投票管理页 | `/Admin/Votes` | 管理员 | 查看进度、提前结束、查看结果 |
| 管理端投票详情 | `/Admin/Votes/{id}` | 管理员 | 查看投票配置与提交进度 |

## 3. 核心交互路径

### 3.1 登录与角色分流

1. 用户进入登录页。
2. 输入邮箱/工号和密码。
3. 点击登录。
4. 系统校验账号有效性：
   - 校验成功：识别角色。
   - 校验失败：展示“账号或密码错误，请检查后重试”。
   - 账号禁用：展示“账号已被禁用，请联系管理员”。
5. 角色分流：
   - 员工：进入员工投票列表。
   - 管理员：进入管理员工作台。

交互要求：

- 登录按钮在请求中进入 Loading 状态，防止重复点击。
- 错误提示显示在密码字段下方或登录按钮上方。
- 未登录访问业务页时，统一跳转登录页。

### 3.2 员工参与投票路径

1. 员工登录后进入投票列表。
2. 使用筛选项查看全部、进行中、已结束、我已提交。
3. 点击进行中且未提交投票的“去投票”。
4. 进入投票详情页。
5. 页面展示：
   - 投票标题。
   - 投票说明。
   - 结束时间。
   - 实名/匿名标识。
   - 隐私提示。
   - 题目列表。
6. 员工填写单选、多选、评分和文本反馈。
7. 点击“提交投票”。
8. 打开提交确认弹窗，提示“提交后不可修改，请确认你的答案无误”。
9. 点击确认提交。
10. 系统校验：
    - 必答题已完成：保存答案并锁定提交记录。
    - 必答题未完成：定位到第一个错误题目并提示。
    - 投票已结束：拒绝提交并提示“投票已结束”。
    - 已提交：拒绝重复提交并提示“你已提交该投票，无法重复提交”。
11. 提交成功后进入已提交状态。

关键状态：

| 状态 | 页面表现 |
|---|---|
| 未提交且进行中 | 表单可编辑，提交按钮可用 |
| 已提交且进行中 | 表单只读，提交按钮禁用，显示已提交提示 |
| 未提交但已结束 | 表单只读，显示投票已结束 |
| 已提交且已结束 | 可进入结果页，表单只读 |

### 3.3 结果查看路径

1. 员工或管理员进入已结束投票。
2. 点击“查看结果”。
3. 系统判断投票状态：
   - 已结束：展示结果页。
   - 未结束：展示“投票结束后，所有员工可查看统计结果”。
4. 结果页展示：
   - 参与人数。
   - 总员工数。
   - 参与率。
   - 结束时间。
   - 单选/多选横向条形图。
   - 评分题平均分与分布图。
   - 文本反馈汇总。
5. 如果当前用户是管理员且投票为实名投票，则展示“查看明细”入口。
6. 如果投票为匿名投票，不展示投票人与答案映射。

图表交互要求：

- 图表应同时展示标签、票数和占比。
- 鼠标悬停或键盘聚焦时可展示详细说明。
- 移动端图表可单列展示，必要时横向滚动。
- 空数据时展示“暂无统计数据”。

### 3.4 管理员创建投票路径

1. 管理员登录进入工作台。
2. 点击“创建投票”。
3. 填写基础信息：标题、说明、结束时间。
4. 选择投票模式：实名或匿名。
5. 选择题目结构：单题或多题。
6. 配置题目：
   - 单选题：至少两个选项。
   - 多选题：至少两个选项，可设置最多选择数量。
   - 评分题：设置评分范围。
   - 文本反馈：填写题目标题和提示。
7. 右侧预览区实时展示员工端效果。
8. 点击“创建投票并立即开始”。
9. 系统校验：
   - 标题必填。
   - 结束时间晚于当前时间。
   - 至少一道题。
   - 单选/多选至少两个选项。
10. 校验通过后创建投票，状态为进行中，并向所有员工开放。
11. 跳转投票管理页。

错误处理：

- 结束时间无效：字段下方提示“结束时间必须晚于当前时间”。
- 题目不足：题目编辑器区域提示“请至少添加一道题目”。
- 选项不足：对应题目下方提示“请至少添加两个选项”。

### 3.5 管理员投票管理路径

1. 管理员进入投票管理页。
2. 查看投票标题、模式、状态、参与进度、操作。
3. 对进行中投票可点击“提前结束”。
4. 系统打开提前结束确认弹窗。
5. 弹窗提示“结束后所有员工可查看统计结果，进行中的员工将无法继续提交”。
6. 点击确认后，投票状态变为已结束。
7. 管理员可进入结果页查看统计。
8. 对实名投票，管理员可查看投票人及答案明细。
9. 对匿名投票，管理员只能查看汇总统计，不可查看投票人与答案映射。

### 3.6 员工账号 Excel 导入路径

1. 管理员进入员工账号导入页。
2. 点击“下载模板”。
3. 根据模板准备 Excel 文件。
4. 上传 Excel 文件。
5. 系统校验文件类型与字段：
   - 工号。
   - 姓名。
   - 邮箱。
   - 初始密码。
   - 角色。
6. 展示导入校验结果：总行数、成功数、失败数。
7. 对失败行展示行号和失败原因。
8. 管理员确认导入或重新上传。
9. 导入完成后展示结果提示。

错误处理：

- 文件格式错误：提示“仅支持 Excel 文件，请上传 .xlsx 文件”。
- 必填字段缺失：在失败表格中说明字段名称。
- 工号或邮箱重复：标记对应行失败或按产品策略更新。

## 4. 导航与返回规则

### 4.1 桌面端

- 管理端使用左侧侧边导航。
- 员工端使用顶部导航 + 页面内返回按钮。
- 深层页面保留返回列表或返回管理页按钮。
- 当前导航项必须高亮。

### 4.2 移动端

- 隐藏复杂侧边栏。
- 员工端提供底部导航：投票列表、投票详情、结果查看。
- 管理端优先保留顶部返回和关键操作，复杂管理功能可折叠菜单。
- 底部固定提交按钮需预留底部安全距离，不遮挡内容。

## 5. 反馈与状态规则

| 类型 | 触发 | 反馈方式 |
|---|---|---|
| Loading | 登录、提交、导入、创建投票 | 按钮 Loading + 禁用重复点击 |
| Success | 提交成功、创建成功、导入成功 | Toast 或状态页提示 |
| Error | 表单校验失败、登录失败、导入失败 | 字段级错误 + 必要时错误摘要 |
| Empty | 无投票、无结果、无导入记录 | 空状态说明 + 可执行动作 |
| Confirm | 提交投票、提前结束 | 弹窗二次确认 |

## 6. 可访问性交互要求

- 所有可点击元素可通过键盘访问。
- 弹窗打开后焦点进入弹窗，关闭后回到触发按钮。
- 弹窗支持 Esc 关闭。
- 错误信息使用 `aria-live` 或 `role="alert"`。
- 图表需提供文字摘要或数据表替代。
- 状态标签必须使用文字说明，不只依赖颜色。
- 移动端触摸目标不小于 44px。
